# Closeout Loop - 关闭循环

## 概述

Closeout Loop 确保任务完成、文档同步、知识传递，形成完整的闭环。

## 关闭循环目标

- 确认任务完成
- 同步相关文档
- 记录经验教训
- 更新知识库
- 通知相关角色

## 关闭循环检查清单

### 1. 任务完成确认

确认任务是否真正完成：

- [ ] 所有验收标准通过
- [ ] 所有验证通过
- [ ] 所有 Bug 已修复
- [ ] 所有文档已更新
- [ ] 无阻塞问题

### 2. 代码提交

确认代码已正确提交：

- [ ] 代码已提交到版本控制
- [ ] 提交信息清晰
- [ ] 代码已合并到主分支
- [ ] PR/MR 已关闭

### 3. 文档同步

同步相关文档：

- [ ] 设计文档已更新
- [ ] API 文档已更新
- [ ] 用户文档已更新
- [ ] 变更日志已更新
- [ ] 架构文档已更新

### 4. 测试更新

更新测试相关：

- [ ] 单元测试已更新
- [ ] 集成测试已更新
- [ ] 测试文档已更新
- [ ] 测试覆盖率已更新

### 5. 知识传递

传递相关知识：

- [ ] 技术决策已记录（ADR）
- [ ] 经验教训已记录
- [ ] 最佳实践已更新
- [ ] 常见问题已更新

### 6. 通知相关角色

通知相关角色：

- [ ] 通知项目负责人
- [ ] 通知相关开发人员
- [ ] 通知 QA 团队
- [ ] 通知文档团队

## 关闭循环流程

### 1. 自我检查

执行自我检查：

- 检查验收标准
- 检查验证结果
- 检查文档状态
- 检查代码提交

### 2. 同行评审

请求同行评审：

- 代码评审
- 设计评审
- 文档评审
- 测试评审

### 3. 修复问题

修复评审中发现的问题：

- 修复代码问题
- 修复文档问题
- 修复测试问题
- 重新验证

### 4. 最终确认

最终确认任务完成：

- 所有评审通过
- 所有问题已修复
- 所有文档已同步
- 所有通知已发送

### 5. 归档

归档任务相关材料：

- 归档设计文档
- 归档测试报告
- 归档评审记录
- 归档变更记录

## 关闭循环最佳实践

1. **完整性**：确保所有检查项都完成
2. **及时性**：及时关闭循环，不要拖延
3. **记录性**：记录所有重要信息
4. **沟通性**：及时通知相关角色
5. **改进性**：持续改进关闭循环流程

## 关闭循环常见问题

### 问题 1：文档未同步

**原因**：忘记更新文档

**解决**：
- 在任务计划中包含文档更新
- 使用文档同步检查清单
- 在关闭循环前检查文档状态

### 问题 2：验证未完成

**原因**：验证被跳过

**解决**：
- 确保所有验证都执行
- 记录验证结果
- 处理验证失败

### 问题 3：通知未发送

**原因**：忘记通知相关角色

**解决**：
- 在任务计划中包含通知
- 使用通知检查清单
- 在关闭循环前确认通知

## 关闭循环改进

持续改进关闭循环流程：

- 收集反馈
- 分析问题
- 优化流程
- 更新检查清单
