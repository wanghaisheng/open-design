# OpenSpec Sync - OpenSpec 同步

## 概述

OpenSpec Sync 用于同步 OpenSpec 文档和变更记录，确保文档与实现保持一致。

## OpenSpec 结构

### 目录结构

```
openspec/
├── changes/
│   ├── change-name-1/
│   │   ├── README.md
│   │   ├── proposal.md
│   │   ├── tasks.md
│   │   └── design.md (可选)
│   └── change-name-2/
│       └── ...
├── decisions/
│   ├── adr-001.md
│   └── adr-002.md
└── specs/
    ├── spec-001.md
    └── spec-002.md
```

### 变更记录（Changes）

每个变更记录包含：

- **README.md**：变更概述
- **proposal.md**：变更提案
- **tasks.md**：任务列表
- **design.md**：架构设计（可选）

### 架构决策记录（ADRs）

每个 ADR 包含：

- **背景**：决策背景
- **决策**：决策内容
- **后果**：决策后果
- **替代方案**：考虑的替代方案

### 规范文档（Specs）

每个规范文档包含：

- **需求**：功能需求
- **设计**：系统设计
- **实现**：实现细节
- **测试**：测试计划

## 同步流程

### 1. 创建变更记录

在开始变更前创建变更记录：

```bash
mkdir -p openspec/changes/change-name
cd openspec/changes/change-name
```

创建以下文件：

- `README.md`：变更概述
- `proposal.md`：变更提案
- `tasks.md`：任务列表

### 2. 更新变更记录

在变更过程中更新变更记录：

- 更新 `tasks.md`：记录任务进度
- 更新 `proposal.md`：记录变更调整
- 更新 `design.md`：记录设计变更

### 3. 记录架构决策

对于架构变更，记录 ADR：

```bash
mkdir -p openspec/decisions
cd openspec/decisions
```

创建 ADR 文件：

- `adr-001.md`：架构决策记录

### 4. 同步规范文档

更新相关规范文档：

- 更新需求
- 更新设计
- 更新实现
- 更新测试

### 5. 归档变更记录

变更完成后归档变更记录：

- 标记变更状态为已完成
- 移动到归档目录（可选）
- 更新变更索引

## 变更记录模板

### README.md 模板

```markdown
# 变更名称

## 概述
[变更概述]

## 目标
- [目标 1]
- [目标 2]

## 范围
- [包含内容]
- [不包含内容]

## 状态
- 当前状态：进行中/已完成
- 开始时间：YYYY-MM-DD
- 完成时间：YYYY-MM-DD

## 相关文档
- [提案](proposal.md)
- [任务](tasks.md)
- [设计](design.md)
```

### proposal.md 模板

```markdown
# 变更提案

## 背景
[变更背景]

## 问题
[要解决的问题]

## 解决方案
[解决方案描述]

## 影响
- [影响 1]
- [影响 2]

## 风险
- [风险 1] - [缓解措施]
- [风险 2] - [缓解措施]

## 替代方案
- [替代方案 1]
- [替代方案 2]
```

### tasks.md 模板

```markdown
# 任务列表

## 里程碑 1
- [ ] 任务 1.1
- [ ] 任务 1.2
- [ ] 任务 1.3

## 里程碑 2
- [ ] 任务 2.1
- [ ] 任务 2.2

## 任务详情

### 任务 1.1
- **ID**：TASK-001
- **描述**：任务描述
- **负责人**：角色名称
- **状态**：待完成/进行中/已完成
- **验收标准**：
  - [ ] 验收标准 1
  - [ ] 验收标准 2
```

```markdown
## 决策
[决策内容]

## 后果
### 正面影响
- [正面影响 1]
- [正面影响 2]

### 负面影响
- [负面影响 1]
- [负面影响 2]

- [替代方案 1]：[为什么不选择]

- [ADR-XXX]：相关决策
```

## 同步检查清单

### 变更开始前

- [ ] 创建变更记录目录
- [ ] 创建 README.md
- [ ] 创建 proposal.md
- [ ] 创建 tasks.md

### 变更过程中

- [ ] 更新 tasks.md 任务状态
- [ ] 更新 proposal.md 变更调整
- [ ] 记录架构决策（如需要）
- [ ] 更新相关规范文档

### 变更完成后

- [ ] 标记变更状态为已完成
- [ ] 更新 README.md 完成时间
- [ ] 归档变更记录
- [ ] 更新变更索引

## 同步最佳实践

1. **及时同步**：变更过程中及时更新文档
2. **版本控制**：所有文档纳入版本控制
3. **清晰命名**：使用清晰的命名约定
4. **完整记录**：记录所有重要决策
5. **定期审查**：定期审查文档准确性
