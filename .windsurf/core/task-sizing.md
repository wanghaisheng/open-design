# Task Sizing - 任务规模

## 概述

Task Sizing 用于评估和分类任务规模，确保任务在合理范围内，便于规划和执行。

## 任务规模分类

### 小型任务（Small）

- **时间范围**：1-4 小时
- **复杂度**：低
- **风险**：低
- **示例**：
  - 修复简单 Bug
  - 添加单个功能
  - 更新文档

### 中型任务（Medium）

- **时间范围**：1-3 天
- **复杂度**：中
- **风险**：中
- **示例**：
  - 实现一个系统
  - 重构模块
  - 添加多个相关功能

### 大型任务（Large）

- **时间范围**：1-2 周
- **复杂度**：高
- **风险**：高
- **示例**：
  - 架构变更
  - 新系统开发
  - 跨模块重构

### 超大型任务（X-Large）

- **时间范围**：2 周以上
- **复杂度**：极高
- **风险**：极高
- **示例**：
  - 引擎迁移
  - 大规模重构
  - 新平台支持

## 任务规模评估

### 评估因素

1. **时间估算**：预计完成时间
2. **复杂度**：技术复杂度
3. **风险**：潜在风险
4. **依赖**：外部依赖
5. **影响范围**：受影响的代码和系统

### 评估方法

使用以下方法评估任务规模：

1. **分解法**：将任务分解为子任务，估算每个子任务
2. **类比法**：与类似任务比较
3. **专家判断**：基于经验判断
4. **三点估算**：乐观、悲观、最可能估算

## 任务规模调整

### 调整原则

- 如果任务超过中型，考虑分解
- 如果任务风险高，增加缓冲时间
- 如果依赖复杂，调整优先级

### 分解策略

将大型任务分解为中型任务：

1. 按功能分解
2. 按系统分解
3. 按里程碑分解
4. 按依赖关系分解

## 任务规模记录

在任务文档中记录：

```yaml
task_size: medium
estimated_hours: 16
complexity: medium
risk: medium
dependencies:
  - task-1
  - task-2
```

## 任务规模验证

在执行过程中验证任务规模：

- 定期检查进度
- 调整估算
- 记录实际时间
- 更新任务规模知识库
