# RAMS 框架文件规范

本文档目录定义了 RAMS（Role-Action Model System）框架中三种核心文件的规范：SOUL.md、ROLE.md、SKILL.md。

---

## 三种文件的关系

```
┌─────────────────────────────────────────────────────────┐
│  SOUL.md                                                │
│  "这个人是什么样的人"                                    │
│  • 性格、价值观、沟通风格                                │
│  • 跨项目复用                                           │
└───────────────────┬─────────────────────────────────────┘
                    │ 决定
                    ▼
┌─────────────────────────────────────────────────────────┐
│  ROLE.md                                                │
│  "这个人要做什么事"                                    │
│  • 职责、技能列表、工作流程                              │
│  • 引用 SKILL.md                                       │
└───────────────────┬─────────────────────────────────────┘
                    │ 引用
                    ▼
┌─────────────────────────────────────────────────────────┐
│  SKILL.md                                               │
│  "如何做这件事"                                        │
│  • 操作步骤、检查清单、输出格式                          │
│  • 被 ROLE.md 引用                                     │
└─────────────────────────────────────────────────────────┘
                    │
                    ▼
          ┌─────────────────────┐
          │   角色实例            │
          │  (Role Instance)     │
          │  = Soul + Skills     │
          └─────────────────────┘
                    │
                    ▼
          ┌─────────────────────┐
          │   角色变体            │
          │  (Role Variant)      │
          │  = 不同技能组合       │
          └─────────────────────┘
```

## 快速对比

| 维度 | SOUL.md | ROLE.md | SKILL.md |
|------|---------|---------|----------|
| **核心问题** | 是什么样的人？ | 要做什么事？ | 如何做这件事？ |
| **内容示例** | 注重用户体验 | 设计经济系统 | 列出货币流入流出渠道 |
| **类比** | NPC角色设定 | 岗位JD | 操作手册/SOP |
| **稳定性** | 跨项目稳定 | 随项目调整 | 跨项目复用 |
| **关键词** | 性格、价值观、风格 | 职责、技能、流程 | 步骤、检查清单、输出 |

## 职责边界红线

### SOUL.md 禁止包含
- ❌ 具体职责（应在 ROLE.md）
- ❌ 操作步骤（应在 SKILL.md）
- ❌ 技能列表

### ROLE.md 禁止包含
- ❌ 性格描述（应在 SOUL.md）
- ❌ 操作细节（应在 SKILL.md）
- ❌ 检查清单

### SKILL.md 禁止包含
- ❌ 角色设定（应在 SOUL.md）
- ❌ 职责描述（应在 ROLE.md）
- ❌ 理论解释（"是什么/为什么"）
- ❌ 工具列表
- ❌ "要素/原则/流程"等概念性章节

---

## 使用流程

### 创建新角色

1. **选择或创建 SOUL.md** —— 定义性格
2. **创建 ROLE.md** —— 定义职责和技能列表
3. **选择或创建 SKILL.md** —— 定义具体操作
4. **实例化角色** —— Soul + Skills

### 修正现有文件

1. 检查是否包含违规章节（见各规范文档的"禁止章节"）
2. 对照质量检查清单验证
3. 参考模板文件进行重构
