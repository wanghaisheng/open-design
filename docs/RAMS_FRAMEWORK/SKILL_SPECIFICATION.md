# SKILL.md 规范定义

SKILL.md 是 RAMS 框架中定义"如何做"的文件，提供具体可执行的操作步骤、方法、技巧与最佳实践。

---

## 核心定位

**SKILL.md = "如何做这件事"**

- **类比**：操作手册、SOP（标准操作程序）、技术指南
- **内容**：具体的操作步骤、检查清单、输出格式、协作协议
- **禁止**：角色设定、概念定义、理论解释、工具列表

---

## 文件结构规范

### 技能目录结构

```
skill-name/
  ├── SKILL.md              # 技能描述（必须）
  ├── implementation.yaml   # 实现配置（可选）
  ├── schema.yaml           # 输入输出Schema（可选）
  ├── dependencies.yaml     # 依赖关系（可选）
  ├── version.yaml          # 版本管理（可选）
  ├── cache.yaml            # 缓存配置（可选）
  ├── monitoring.yaml       # 监控配置（可选）
  └── scripts/              # 脚本文件（可选）
      └── skill-script.py
```

## 必须包含的章节

| 章节 | 说明 | 必须/可选 |
|------|------|-----------|
| `适用场景` | 列出何时使用此技能 | 必须 |
| `操作步骤` | 详细的操作流程，每步包含目标、操作方法、检查清单 | 必须 |
| `操作注意事项` | 执行时的关键注意事项 | 必须 |
| `输出格式` | 交付物的具体格式 | 必须 |
| `协作协议` | 与谁汇报、与谁协调 | 必须 |
| `常见错误` | 常见错误及修正方法 | 必须 |

---

## 禁止出现的章节

| 违规章节 | 原因 | 应移至 |
|----------|------|--------|
| `Context` | 角色设定 | soul.md |
| `核心职责` | 角色职责 | role.md |
| `Domain Context` | 概念定义 | role.md 或独立文档 |
| `XX要素` | 概念枚举 | role.md 或删除 |
| `XX原则` | 理论原则 | role.md 或删除 |
| `XX流程` | 概念性流程 | 改为具体操作步骤 |
| `设计工具` / `创作工具` / `参考工具` | 工具列表 | 删除（工具不是技能） |
| `参考资源` | 外部链接 | 删除或移至附录 |
| `Psychology Principles Integration` | 理论解释 | 删除或融入操作步骤 |

## 操作步骤写作规范

### 每个步骤必须包含：

1. **目标**：这一步要达成什么（可验收标准）
2. **操作方法**：
   - 使用动词开头（列出、写、画、设计、输出）
   - 具体可执行（避免抽象概念）
   - 有明确的完成标志
3. **检查清单**：
   - 可勾选（`- [ ]`）
   - 可验证（是/否问题）
   - 完备（覆盖所有关键点）

### 示例对比：

❌ **错误示例**（概念性）：
```markdown
### 1. 故事概念
- 定义故事主题
- 确定故事目标
- 设计故事结构
```

✅ **正确示例**（可执行）：
```markdown
### 步骤1：把叙事目标写成"可验收"的一句话

**目标**：明确叙事服务什么体验目标、成功标准是什么

**操作方法**：
1. 写叙事主题与一句话目标（面向玩家情感体验）
2. 列出叙事输入/输出（玩家行为、选择、状态变化）
3. 写 3-5 条成功标准（例如：玩家理解角色动机；选择有可见后果）
4. 写 3-5 条失败信号（例如：玩家不知道为什么选；剧情前后矛盾）

**检查清单**：
- [ ] 有清晰边界：叙事"包含什么/不包含什么"
- [ ] 有成功标准：可观测、可测试
- [ ] 有失败信号：能快速触发调整
```

## 质量检查清单

创建或修改 SKILL.md 后，使用此清单验证：

- [ ] 没有 `Context` / `核心职责` / `Domain Context` 章节
- [ ] 没有 `XX要素` / `XX原则` / `XX流程` / `参考资源` 章节
- [ ] 没有独立的 `设计工具` / `创作工具` / `AI工具` / `传统工具` 章节
- [ ] 包含 `适用场景` 章节
- [ ] 包含 `操作步骤` 章节，每步有目标、操作方法、检查清单
- [ ] 包含 `操作注意事项` 章节
- [ ] 包含 `输出格式` 章节
- [ ] 包含 `协作协议` 章节（向谁汇报、协调对象）
- [ ] 包含 `常见错误` 章节
- [ ] 所有步骤都是"如何做"而非"是什么/为什么"
- [ ] 所有检查清单项都是可验证的

## 与其他文件的关系

```
soul.md (性格/价值观)
    ↓
role.md (职责/技能列表) ←→ SKILL.md (如何做) ←→ 其他SKILL.md
    ↓
角色实例 (Role Instance)
    ↓
角色变体 (Role Variant) = 不同技能组合
```

- **soul.md** 决定角色"是什么样的人"
- **role.md** 决定角色"要做什么事"（使用哪些 skills）
- **SKILL.md** 决定"如何做这件事"
- **角色实例** = soul + 选定的 skills

## 技能配置文件

### implementation.yaml

定义技能的实现方式，支持多种实现类型（AI模型、API、脚本等）。详见 [SKILL_IMPLEMENTATION.md](SKILL_IMPLEMENTATION.md)。

### schema.yaml

定义技能的输入输出Schema，用于数据验证和类型提示。详见 [SKILL_SCHEMA.md](SKILL_SCHEMA.md)。

### dependencies.yaml

定义技能的依赖关系，包括技能依赖、工具依赖、数据依赖等。详见 [SKILL_DEPENDENCIES.md](SKILL_DEPENDENCIES.md)。

### version.yaml

定义技能的版本信息，包括版本号、变更日志、兼容性等。详见 [SKILL_VERSION.md](SKILL_VERSION.md)。

### cache.yaml

定义技能的缓存配置，包括缓存策略、失效条件等。详见 [SKILL_CACHE.md](SKILL_CACHE.md)。

### monitoring.yaml

定义技能的监控配置，包括指标收集、日志记录、告警通知等。详见 [SKILL_MONITORING.md](SKILL_MONITORING.md)。
