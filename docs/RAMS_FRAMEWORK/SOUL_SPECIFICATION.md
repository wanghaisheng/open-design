# SOUL.md 规范定义

SOUL.md 是 RAMS 框架中定义"这个人是什么样的人"的文件，描述角色的性格、价值观、沟通风格与行为准则。

---

## 核心定位

**SOUL.md = "这个人是什么样的人"**

- **类比**：NPC的角色设定、演员的表演风格、人物小传
- **内容**：性格特征、价值观、沟通风格、行为准则、决策偏好
- **禁止**：具体职责（应在 role.md）、操作步骤（应在 skill.md）

---

## 章节说明

### 核心性格

- 描述这个灵魂的基本性格特征
- 用具体行为表现来说明，而非抽象标签
- 可引用心理学模型（如 MBTI、大五人格）作为参考，但不必严格对应

### 价值观

- **核心价值观**：驱动决策的根本信念
- **决策优先级**：面对冲突时的取舍顺序

### 沟通风格

- **语言特征**：用词偏好、句式习惯、语气特征
- **沟通偏好**：主动/被动、直接/委婉、详细/简洁

### 协作偏好

- **理想协作方式**：喜欢如何与他人合作
- **冲突处理**：面对分歧时的处理方式

### 适用角色

- 列出与此 soul 匹配的角色类型
- 说明为什么适合

### 不兼容场景

- 列出此 soul 不适合的场景
- 说明原因

## 与 ROLE.md 的区别

| 方面 | SOUL.md | ROLE.md |
|------|---------|---------|
| **聚焦** | 是什么样的人 | 要做什么事 |
| **内容** | 性格、价值观、风格 | 职责、技能、流程 |
| **稳定性** | 跨项目稳定 | 随项目变化 |
| **数量** | 少量灵魂（可复用） | 多种角色 |
| **示例** | "注重用户体验" | "设计经济系统" |

---

## 灵魂复用

灵魂是跨项目复用的，一个 soul 可以搭配不同的 role：

```
soul: "注重用户体验的设计灵魂"
    ├─→ + role: "UX设计师" = UX设计师实例
    ├─→ + role: "UI设计师" = UI设计师实例
    └─→ + role: "产品经理" = 产品经理实例
```

---

## 角色实例化公式

```
角色实例 = Soul + Role + 选定的 Skills
```

示例：
- **Soul**: "注重用户体验的设计灵魂"
- **Role**: "经济设计师"
- **Skills**: economy-design, core-loop-design, balance-adjustment
- **结果**: 一个注重用户体验的经济设计师

## 质量检查清单

创建或修改 SOUL.md 后，使用此清单验证：

- [ ] 有清晰的 `核心性格` 章节
- [ ] 有 `价值观` 章节，包含核心价值观和决策优先级
- [ ] 有 `沟通风格` 章节
- [ ] 有 `协作偏好` 章节
- [ ] 有 `适用角色` 章节
- [ ] 没有具体职责描述（应在 role.md）
- [ ] 没有操作步骤（应在 skill.md）
- [ ] 性格描述用具体行为说明，而非抽象标签

---

## 与其他文件的关系

```
SOUL.md (性格/价值观)
    ↓
ROLE.md (职责/技能列表)
    ↓
SKILL.md (如何做)
    ↓
角色实例 (Soul + Role + Skills)
    ↓
角色变体 (不同技能组合)
```
