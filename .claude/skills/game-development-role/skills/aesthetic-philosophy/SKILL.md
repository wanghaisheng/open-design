## 操作注意事项

### 哲学定义

1. **清晰明确**：美学哲学要清晰明确
2. **可执行**：哲学要能指导实践
3. **一致性**：哲学要与游戏愿景一致
4. **灵活性**：哲学要有一定灵活性

### 框架建立

1. **全面覆盖**：框架要覆盖所有美学方面
2. **层次清晰**：框架要有清晰的层次
3. **可测量**：标准要可测量
4. **可迭代**：框架要可迭代改进

### 决策指导

1. **依据充分**：决策要有充分依据
2. **流程透明**：决策流程要透明
3. **记录完整**：决策记录要完整
4. **及时反馈**：及时提供决策反馈

## 常见错误

### 错误1：哲学过于抽象

**问题**：美学哲学过于抽象，难以执行

**修正**：将抽象哲学转化为可执行原则

### 错误2：框架不完整

**问题**：美学框架不完整，遗漏重要方面

**修正**：确保框架覆盖所有美学维度

### 错误3：决策无依据

**问题**：美学决策缺乏依据，随意性强

**修正**：建立明确的决策依据和流程

### 错误4：忽视一致性

**问题**：忽视美学一致性，导致风格混乱

**修正**：建立一致性检查机制

### 错误5：不记录决策

**问题**：美学决策不记录，难以追溯

**修正**：记录所有美学决策和理由
