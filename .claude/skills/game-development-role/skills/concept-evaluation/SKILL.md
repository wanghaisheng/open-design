# Concept Evaluation

评估概念质量，从创意性、可行性、一致性和沟通性等多个维度评估视觉概念的质量。

## 适用场景

- 需要评估概念创意性
- 需要评估概念可行性
- 需要评估概念一致性
- 需要评估概念沟通性
- 需要提供改进建议

## 操作步骤

### 步骤1：建立标准

**目标**：建立评估标准

**操作方法**：
1. 建立评估标准（创意性、可行性、一致性、沟通性）
2. 建立评分体系（1-5分或1-10分）
3. 建立权重分配（各维度权重）

**检查清单**：
- [ ] 评估标准已建立
- [ ] 评分体系已建立
- [ ] 权重分配已建立

### 步骤2：执行评估

**目标**：执行概念评估

**操作方法**：
1. 创意性评估（独特性、创新性、视觉冲击、记忆点）
2. 可行性评估（技术可行性、资源可行性、时间可行性、成本可行性）
3. 一致性评估（风格一致性、色彩一致性、氛围一致性、整体一致性）
4. 沟通性评估（清晰度、易理解性、表达力、传达效果）

**检查清单**：
- [ ] 创意性已评估
- [ ] 可行性已评估
- [ ] 一致性已评估
- [ ] 沟通性已评估

### 步骤3：综合评分

**目标**：计算综合评分

**操作方法**：
1. 综合评分计算（根据权重计算）
2. 评估结果分析（分析强项和弱项）
3. 评估结果排序（按综合评分排序）

**检查清单**：
- [ ] 综合评分已计算
- [ ] 评估结果已分析
- [ ] 评估结果已排序

### 步骤4：反馈改进

**目标**：提供改进建议

**操作方法**：
1. 提供评估报告（各维度评分、综合评分）
2. 提供改进建议（针对弱项提供具体建议）
3. 跟踪改进进度（跟踪改进实施）

**检查清单**：
- [ ] 评估报告已提供
- [ ] 改进建议已提供
- [ ] 改进进度已跟踪

## 输出格式

### 评估报告
- 评估维度评分
- 综合评估结果
- 改进建议
- 优先级排序

## 常见错误

### 错误1：不建立评估标准

**问题**：不建立评估标准，直接评估

**修正**：必须先建立评估标准、评分体系、权重分配

### 错误2：不全面评估

**问题**：只评估单一维度，不全面评估

**修正**：必须全面评估所有维度（创意性、可行性、一致性、沟通性）

### 错误3：不提供改进建议

**问题**：只提供评分，不提供改进建议

**修正**：必须提供改进建议，针对弱项提供具体建议

### 错误4：不跟踪改进进度

**问题**：提供评估报告后不跟踪改进进度

**修正**：必须跟踪改进进度，验证改进效果

### 错误5：主观评估

**问题**：评估主观，不数据驱动

**修正**：必须客观评估，数据驱动，避免主观偏见
