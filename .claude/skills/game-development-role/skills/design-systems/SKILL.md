# Design Systems

将游戏概念分解为独立系统，映射依赖关系，优先确定设计顺序，指导创建每个系统的GDD。

## 适用场景

- 需要将游戏概念分解为独立系统
- 需要映射系统依赖关系
- 需要确定设计优先级
- 需要指导创建系统GDD
- 需要检测循环依赖

## 操作步骤

### 步骤1：读取概念

**目标**：读取必需上下文

**操作方法**：
1. 读取游戏概念
2. 读取游戏支柱（如果存在）
3. 读取系统索引（如果存在）
4. 检查现有GDD

**检查清单**：
- [ ] 游戏概念已读取
- [ ] 游戏支柱已读取（如存在）
- [ ] 系统索引已读取（如存在）
- [ ] 现有GDD已检查

### 步骤2：系统枚举

**目标**：枚举所有系统

**操作方法**：
1. 提取显式系统（从概念文档直接提到的系统）
2. 识别隐式系统（每个显式系统隐含的隐藏系统）
3. 系统分类（按类别组织系统）
4. 系统描述（每个系统的简要描述）

**检查清单**：
- [ ] 显式系统已提取
- [ ] 隐式系统已识别
- [ ] 系统已分类
- [ ] 系统已描述

### 步骤3：依赖映射

**目标**：映射系统依赖关系

**操作方法**：
1. 映射输入/输出依赖（系统A产生系统B需要的数据）
2. 映射结构依赖（系统A提供系统B插入的框架）
3. 映射UI依赖（每个游戏玩法系统都有对应的UI系统）
4. 按依赖排序（将系统排列到层级中）
5. 检测循环依赖

**检查清单**：
- [ ] 输入/输出依赖已映射
- [ ] 结构依赖已映射
- [ ] UI依赖已映射
- [ ] 系统已排序
- [ ] 循环依赖已检测

### 步骤4：优先级分配

**目标**：分配设计优先级

**操作方法**：
1. 基于MVP分配（概念中MVP部分提到的系统）
2. 基于垂直切片分配（在一个区域提供完整体验所需的系统）
3. 基于Alpha分配（所有剩余的游戏玩法系统）
4. 基于完整愿景分配（润色、元系统和nice-to-have系统）
5. 确定设计顺序（MVP基础系统 → MVP核心系统 → MVP功能系统 → 垂直切片 → 依此类推）

**检查清单**：
- [ ] MVP优先级已分配
- [ ] 垂直切片优先级已分配
- [ ] Alpha优先级已分配
- [ ] 完整愿景优先级已分配
- [ ] 设计顺序已确定

### 步骤5：创建系统索引

**目标**：创建系统索引文档

**操作方法**：
1. 起草文档（系统枚举表、依赖映射、推荐设计顺序）
2. 用户批准
3. 更新会话状态
4. 写入文件

**检查清单**：
- [ ] 文档已起草
- [ ] 用户已批准
- [ ] 会话状态已更新
- [ ] 文件已写入

### 步骤6：设计单个系统

**目标**：设计单个系统（可选，迭代）

**操作方法**：
1. 选择系统
2. 委托给代理
3. 编写GDD
4. 运行设计审查
5. 更新系统索引
6. 循环或停止

**检查清单**：
- [ ] 系统已选择
- [ ] 已委托给代理
- [ ] GDD已编写
- [ ] 设计审查已运行
- [ ] 系统索引已更新

## 常见错误

### 错误1：不识别隐式系统

**问题**：只提取显式系统，不识别隐式系统

**修正**：必须识别隐式系统（物品栏、战斗、开放世界、多人游戏、制作、对话、进度等）

### 错误2：不映射依赖关系

**问题**：不映射系统依赖关系，导致设计顺序错误

**修正**：必须映射输入/输出依赖、结构依赖、UI依赖

### 错误3：不检测循环依赖

**问题**：不检测循环依赖，导致系统无法实现

**修正**：必须检测循环依赖并解决

### 错误4：不分配优先级

**问题**：不分配设计优先级，导致开发混乱

**修正**：必须基于MVP、垂直切片、Alpha、完整愿景分配优先级

### 错误5：不运行设计审查

**问题**：不运行设计审查，导致质量问题

**修正**：必须运行设计审查，每个GDD通过设计审查
