# Team UI

协调UI团队：协调UX设计师、UI程序员和艺术总监，从线框图到最终设计、实施和完善用户界面功能。

## UI团队组成

### 1. UX设计师 (UX Designer)
- 用户流程
- 线框图
- 可访问性
- 输入处理

### 2. UI程序员 (UI Programmer)
- UI框架
- 屏幕
- 小部件
- 数据绑定
- 实施

### 3. 艺术总监 (Art Director)
- 视觉风格
- 布局润色
- 与艺术圣经一致性

## UI团队原则

### 1. 用户中心
- 以用户为中心的设计
- 清晰的用户流程
- 直观的交互

### 2. 可访问性
- 文本大小
- 对比度
- 色盲安全
- 输入支持

### 3. 视觉一致性
- 统一视觉风格
- 与艺术圣经一致
- 品牌一致性

### 4. 技术可行性
- 性能优化
- 平台支持
- 分辨率适配

## UI团队流程

### 阶段1：UX设计
委托给UX设计师：
- 定义功能的用户流程（入口点、状态、出口点）
- 为每个屏幕/状态创建线框图
- 指定交互模式：键盘/鼠标和手柄如何导航？
- 定义可访问性要求：文本大小、对比度、色盲安全
- 识别UI需要显示的数据（读取什么游戏状态？）
- 输出：带线框图和交互图的UX规格

### 阶段2：视觉设计
委托给艺术总监：
- 根据艺术圣经审查线框图
- 定义视觉处理：颜色、排版、间距、动画
- 指定资产需求（图标、背景、装饰元素）
- 确保与现有UI屏幕一致
- 输出：带样式说明的视觉设计规格

### 阶段3：实施
委托给UI程序员：
- 按照UX规格和视觉设计实施UI
- 确保UI从不拥有或修改游戏状态——仅显示，动作通过事件
- 所有文本通过本地化系统——无硬编码字符串
- 支持两种输入方法（键盘/鼠标+手柄）
- 实施可访问性功能（文本缩放、色盲模式支持）
- 将数据绑定连接到游戏状态
- 输出：实施的UI功能

### 阶段4：审查（并行）
并行委托：
- UX设计师：验证实施与线框图和交互规格匹配。测试仅键盘和仅手柄导航。检查可访问性。
- 艺术总监：验证与艺术圣经的视觉一致性。检查最低和最高支持分辨率。

### 阶段5：润色
- 解决审查反馈
- 验证动画可跳过并尊重运动偏好
- 确认UI声音通过音频事件系统触发
- 在所有支持的分辨率和宽高比下测试

## 委托方法

使用Task工具生成每个团队成员作为子代理：
- ux-designer——用户流程、线框图、可访问性、输入处理
- ui-programmer——UI框架、屏幕、小部件、数据绑定
- art-director——视觉风格、布局润色、艺术圣经一致性

始终在每个代理的提示中提供完整上下文（功能要求、现有UI模式、平台目标）。在管道允许的情况下并行启动独立代理（例如，阶段4审查代理可同时运行）。

## 决策点

在每个阶段转换时，使用AskUserQuestion向用户呈现子代理的提案作为可选项。在对话中编写代理的完整分析，然后用简洁标签捕获决策。用户必须批准才能进入下一阶段。

## 输出

涵盖以下内容的摘要报告：UX规格状态、视觉设计状态、实施状态、可访问性合规、输入方法支持、任何未决问题。
