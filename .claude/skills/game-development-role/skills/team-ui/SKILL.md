# Team UI

协调UI团队：协调UX设计师、UI程序员和艺术总监，从线框图到最终设计、实施和完善用户界面功能。

## 适用场景

- 需要设计UX流程
- 需要设计视觉UI
- 需要实施UI功能
- 需要审查UI质量
- 需要完善UI体验

## 操作步骤

### 步骤1：UX设计

**目标**：设计UX

**操作方法**：
1. 委托给UX设计师（定义功能的用户流程、为每个屏幕/状态创建线框图、指定交互模式、定义可访问性要求、识别UI需要显示的数据）
2. 输出带线框图和交互图的UX规格

**检查清单**：
- [ ] 用户流程已定义
- [ ] 线框图已创建
- [ ] 交互模式已指定
- [ ] 可访问性要求已定义
- [ ] UX规格已输出

### 步骤2：视觉设计

**目标**：设计视觉UI

**操作方法**：
1. 委托给艺术总监（根据艺术圣经审查线框图、定义视觉处理、指定资产需求、确保与现有UI屏幕一致）
2. 输出带样式说明的视觉设计规格

**检查清单**：
- [ ] 线框图已审查
- [ ] 视觉处理已定义
- [ ] 资产需求已指定
- [ ] 视觉一致性已确保
- [ ] 视觉设计规格已输出

### 步骤3：实施

**目标**：实施UI

**操作方法**：
1. 委托给UI程序员（按照UX规格和视觉设计实施UI、确保UI不拥有或修改游戏状态、所有文本通过本地化系统、支持两种输入方法、实施可访问性功能、将数据绑定连接到游戏状态）
2. 输出实施的UI功能

**检查清单**：
- [ ] UI已实施
- [ ] 游戏状态未修改
- [ ] 本地化系统已使用
- [ ] 输入方法已支持
- [ ] 可访问性功能已实施
- [ ] 数据绑定已连接

### 步骤4：审查

**目标**：审查UI

**操作方法**：
1. 并行委托（UX设计师、艺术总监）
2. UX设计师：验证实施与线框图和交互规格匹配、测试仅键盘和仅手柄导航、检查可访问性
3. 艺术总监：验证与艺术圣经的视觉一致性、检查最低和最高支持分辨率

**检查清单**：
- [ ] 实施已验证
- [ ] 导航已测试
- [ ] 可访问性已检查
- [ ] 视觉一致性已验证
- [ ] 分辨率支持已检查

### 步骤5：润色

**目标**：润色UI

**操作方法**：
1. 解决审查反馈
2. 验证动画可跳过并尊重运动偏好
3. 确认UI声音通过音频事件系统触发
4. 在所有支持的分辨率和宽高比下测试

**检查清单**：
- [ ] 审查反馈已解决
- [ ] 动画已验证
- [ ] UI声音已确认
- [ ] 分辨率和宽高比已测试

## 常见错误

### 错误1：不设计UX

**问题**：不定义用户流程、不创建线框图

**修正**：必须委托给UX设计师，设计UX

### 错误2：不设计视觉UI

**问题**：不定义视觉处理、不指定资产需求

**修正**：必须委托给艺术总监，设计视觉UI

### 错误3：不实施UI

**问题**：不按照规格实施UI、不确保游戏状态未被修改

**修正**：必须委托给UI程序员，实施UI

### 错误4：不审查UI

**问题**：不验证实施、不检查可访问性

**修正**：必须并行委托给UX设计师和艺术总监，审查UI

### 错误5：不润色UI

**问题**：不解决审查反馈、不验证动画

**修正**：必须解决审查反馈，验证动画、UI声音、分辨率和宽高比
