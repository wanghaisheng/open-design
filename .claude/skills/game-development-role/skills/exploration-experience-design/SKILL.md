# Exploration Experience Design

设计游戏探索体验，创造引人入胜的探索机制和玩家故事生成系统。

## 适用场景

- 设计开放世界游戏的探索系统
- 设计好奇心驱动的探索机制
- 设计自叙事和玩家故事生成系统
- 优化现有游戏的探索体验
- 设计3A开放世界的探索标准

## 操作步骤

### 步骤1：读取PRD和探索需求

**目标**：读取游戏PRD，理解探索需求和目标

**操作方法**：
1. 读取游戏PRD文档
2. 提取游戏类型（开放世界、线性、半开放）
3. 提取目标用户和探索偏好
4. 提取现有探索系统设计
5. 分析探索的核心价值主张
6. 提取商业化目标和探索付费点

**检查清单**：
- [ ] PRD文档已读取
- [ ] 游戏类型已提取
- [ ] 探索需求已提取
- [ ] 探索系统已提取
- [ ] 核心价值主张已提取
- [ ] 商业化目标已提取

### 步骤2：设计好奇心驱动探索机制

**目标**：基于五维心理学模型设计好奇心驱动的探索系统

**操作方法**：
1. **信息缺口（Information Gap）设计**
   - 创造玩家"知道存在但不知道详情"的信息缺口
   - 通过视觉线索、环境叙事、NPC对话暗示未知内容
   - 设计渐进式信息揭示，保持探索动力
2. **新奇性（Novelty）设计**
   - 设计多样化的探索目标和奖励
   - 避免重复的探索模板，创造独特的奇遇
   - 设计随机事件和动态变化，增加世界新鲜感
3. **挑战性（Challenge）设计**
   - 设计适度的探索难度（物理障碍、谜题、敌人）
   - 提供可克服的挑战，增强探索成就感
   - 设计不同难度层级的探索内容
4. **自主性（Autonomy）设计**
   - 提供多条探索路径和选择
   - 允许玩家自主决定探索顺序和方式
   - 设计非线性探索结构
5. **社会性（Sociality）设计**
   - 设计可分享的探索发现
   - 设计合作探索机制
   - 设计探索成就和排行榜

**检查清单**：
- [ ] 信息缺口已设计
- [ ] 新奇性已设计
- [ ] 挑战性已设计
- [ ] 自主性已设计
- [ ] 社会性已设计

### 步骤3：设计自叙事和玩家故事生成

**目标**：设计让玩家创造自己故事的机制

**操作方法**：
1. **自叙事设计（Self-Narrative Design）**
   - 设计玩家可自定义的叙事元素（角色背景、选择、行为）
   - 设计玩家行为对世界的影响和反馈
   - 设计可回忆和分享的"玩家故事"时刻
2. **玩家故事生成机制**
   - 设计动态事件系统，创造独特的玩家经历
   - 设计玩家选择和行为的持久影响
   - 设计玩家故事的记录和分享功能
3. **叙事与探索结合**
   - 设计探索中的叙事碎片和环境故事
   - 设计玩家通过探索逐步揭示的叙事
   - 设计玩家探索行为对叙事的影响

**检查清单**：
- [ ] 自叙事设计已设计
- [ ] 玩家故事生成已设计
- [ ] 叙事与探索结合已设计

### 步骤4：设计3A开放世界探索原则

**目标**：应用3A开放世界设计标准

**操作方法**：
1. **视觉引导设计**
   - 设计清晰的地标和视觉焦点
   - 设计引导玩家探索的视觉线索
   - 设计远距离可见的兴趣点
2. **可达性设计**
   - 设计多样的移动方式（步行、骑乘、传送）
   - 设计合理的探索节奏和体力系统
   - 设计可探索的垂直空间
3. **奖励分布设计**
   - 设计分散的探索奖励
   - 设计不同价值层级的探索奖励
   - 设计与探索难度匹配的奖励
4. **世界鲜活度设计**
   - 设计动态的世界事件和NPC行为
   - 设计昼夜循环和天气系统
   - 设计生态系统和动物行为

**检查清单**：
- [ ] 视觉引导已设计
- [ ] 可达性已设计
- [ ] 奖励分布已设计
- [ ] 世界鲜活度已设计

### 步骤5：设计探索节奏和情绪曲线

**目标**：控制探索体验的节奏和情绪波动

**操作方法**：
1. 设计探索强度曲线（高强度探索与低强度探索交替）
2. 设计情绪兴奋度曲线（好奇-发现-惊喜-满足）
3. 设计探索缓冲和恢复空间
4. 设计探索高潮节点（重大发现、剧情转折、Boss战）

**检查清单**：

- [ ] 缓冲空间已设计

## 操作注意事项

### 核心设计原则

1. **好奇心驱动**：探索的核心动力应来自玩家的好奇心和信息缺口
2. **自叙事生成**：设计应鼓励玩家创造自己的故事和经历
3. **开放世界标准**：应用3A开放世界的设计原则和标准
4. **情绪节奏**：控制探索体验的情绪波动和节奏

### 好奇心驱动设计原则

1. **信息缺口**：创造"知道存在但不知道详情"的信息缺口
2. **新奇性**：提供多样化的探索目标和奖励
3. **挑战性**：设计适度的探索难度和成就感
4. **自主性**：提供多条探索路径和选择
5. **社会性**：设计可分享和合作的探索机制

### 自叙事设计原则

1. **玩家选择**：允许玩家自定义叙事元素和行为
2. **持久影响**：设计玩家行为对世界的持久影响
3. **故事记录**：设计玩家故事的记录和分享功能
4. **探索结合**：将叙事与探索深度结合

### 3A开放世界设计原则

1. **视觉引导**：设计清晰的地标和视觉线索
2. **可达性**：设计多样的移动方式和合理的节奏
3. **奖励分布**：设计分散且匹配难度的奖励
4. **世界鲜活度**：设计动态的世界事件和生态系统

## 协作协议

### 向谁汇报

- **lead-designer**：探索体验设计方案和体验目标
- **creative-director**：探索体验与游戏风格和主题的一致性
- **technical-director**：探索系统的技术实现可行性

### 协调对象

- **level-design**：协调探索与关卡设计的结合
- **narrative-design**：协调探索与叙事的结合
- **ui-design**：协调探索系统的UI/UX设计
- **qa-lead**：协调探索系统的测试和质量保证

## 常见错误

### 错误1：忽视好奇心驱动

**问题**：探索设计缺乏好奇心驱动机制，玩家缺乏探索动力

**修正**：基于五维心理学模型设计好奇心驱动探索机制

### 错误2：自叙事设计不足

**问题**：探索体验缺乏自叙事和玩家故事生成机制

**修正**：设计自叙事和玩家故事生成系统，鼓励玩家创造自己的故事

### 错误3：未应用3A开放世界标准

**问题**：开放世界探索未达到3A游戏的设计标准

**修正**：应用3A开放世界设计原则，包括视觉引导、可达性、奖励分布和世界鲜活度

### 错误4：探索节奏不合理

**问题**：探索体验缺乏节奏控制，情绪波动过大或过小

**修正**：设计探索节奏和情绪曲线，控制探索体验的节奏和情绪波动

### 错误5：探索奖励不合理

**问题**：探索奖励与探索难度不匹配，或奖励分布不合理

**修正**：设计与探索难度匹配的奖励，合理分布探索奖励
