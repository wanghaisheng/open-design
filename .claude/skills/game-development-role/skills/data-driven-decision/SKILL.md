# Data Driven Decision

数据驱动决策技能，基于可量化数据做出"继续投入"或"果断放弃"的客观决策。

## 适用场景

- 原型测试决策
- 项目继续/放弃决策
- 数据分析决策
- 资源分配决策
- 风险控制决策

## 操作步骤

### 步骤1：设定决策标准

**目标**：在测试前设定清晰的量化决策标准

**操作方法**：
1. **定义中止标准**
   - 设定CPI上限（如<$0.8）
   - 设定留存下限（如D1>45%, D7>15%）
   - 设定时长下限（如首日>50分钟）
   - 设定ARPU目标

2. **定义成功标准**
   - 设定CPI目标（如<$0.35）
   - 设定留存目标（如D1>50%, D7>20%）
   - 设定时长目标（如首日>60分钟）
   - 设定ARPU目标

3. **定义测试周期**
   - 设定测试时长
   - 设定样本量要求
   - 设定数据收集周期
   - 设定评估时间点

**检查清单**：
- [ ] 中止标准已定义
- [ ] 成功标准已定义
- [ ] 测试周期已设定
- [ ] 标准已团队确认

### 步骤2：收集测试数据

**目标**：收集完整准确的测试数据

**操作方法**：
1. **数据收集**
   - 收集CPI数据
   - 收集留存数据（D1, D7, D30）
   - 收集游戏时长数据
   - 收集ARPU数据
   - 收集用户行为数据

2. **数据验证**
   - 验证数据完整性
   - 验证数据准确性
   - 识别异常数据
   - 处理数据缺失

3. **数据标准化**
   - 标准化数据格式
   - 标准化计算方法
   - 标准化时间窗口
   - 标准化用户分群

**检查清单**：
- [ ] 数据已收集
- [ ] 数据已验证
- [ ] 数据已标准化
- [ ] 数据已准备好

### 步骤3：对比基准线

**目标**：将测试数据与品类基准线对比

**操作方法**：
1. **获取基准线**
   - 获取品类CPI基准
   - 获取品类留存基准
   - 获取品类时长基准
   - 获取品类ARPU基准

2. **数据对比**
   - 对比CPI数据
   - 对比留存数据
   - 对比时长数据
   - 对比ARPU数据

3. **差异分析**
   - 分析差异原因
   - 识别优势指标
   - 识别劣势指标
   - 评估改进空间

**检查清单**：
- [ ] 基准线已获取
- [ ] 数据已对比
- [ ] 差异已分析
- [ ] 原因已识别

### 步骤4：评估结构性潜力

**目标**：评估项目是否具备结构性潜力

**操作方法**：
1. **四维评估**
   - 评估趋势维度
   - 评估受众维度
   - 评估核心玩法维度
   - 评估变现维度

2. **潜力判断**
   - 判断结构性潜力
   - 识别结构性短板
   - 评估改进可行性
   - 判断长期潜力

3. **风险评估**
   - 评估市场风险
   - 评估技术风险
   - 评估竞争风险
   - 评估资源风险

**检查清单**：
- [ ] 四维已评估
- [ ] 潜力已判断
- [ ] 风险已评估
- [ ] 结构性潜力已确认

### 步骤5：做出决策

**目标**：基于数据做出客观决策

**操作方法**：
1. **决策树应用**
   - 数据是否达到最低及格线？
   - 是否具备结构性潜力？
   - 是否允许修复性测试？

2. **决策制定**
   - **继续开发**：数据达标，继续投入
   - **修复性测试**：数据不达标但有结构性潜力，允许一次修复
   - **立即中止**：数据不达标且无结构性潜力，立即中止

3. **决策记录**
   - 记录决策过程
   - 记录决策理由
   - 记录数据依据
   - 记录后续行动

**检查清单**：
- [ ] 决策树已应用
- [ ] 决策已制定
- [ ] 决策已记录
- [ ] 后续行动已规划

## 操作注意事项

### 决策原则

1. **客观决策**：基于数据客观决策
2. **果断中止**：果断中止无潜力项目
3. **提前定义**：提前定义决策标准
4. **记录完整**：完整记录决策过程

### 数据分析

1. **数据质量**：确保数据质量可靠
2. **基准对比**：与品类基准对比
3. **差异分析**：深入分析差异原因
4. **趋势分析**：分析数据趋势

### 决策执行

1. **坚决执行**：坚决执行决策
2. **及时沟通**：及时沟通决策结果
3. **资源调整**：及时调整资源分配
4. **经验总结**：总结决策经验

## 协作协议

### 向谁汇报

- creative-director：决策结果和理由
- producer：资源调整建议

### 协调对象

- data-analyst：数据分析支持
- team-leads：团队沟通

## 常见错误

### 错误1：不提前定义标准

**问题**：不提前定义决策标准，决策随意

**修正**：在测试前就明确定义决策标准

### 错误2：只看单一指标

**问题**：只看单一指标，忽视整体表现

**修正**：综合评估多个指标，全面判断

### 错误3：不舍得中止

**问题**：对项目不舍得中止，继续投入

**修正**：果断中止无潜力项目，加速止损

### 错误4：决策不记录

**问题**：不记录决策过程，难以追溯

**修正**：完整记录决策过程和理由

### 错误5：不执行决策

**问题**：做出决策但不执行

**修正**：坚决执行决策，及时调整资源
