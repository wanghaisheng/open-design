### 步骤3：设计抽卡概率模型

**目标**：设计抽卡的概率模型和保底机制

**操作方法**：
1. 确定抽卡模型类型（简单抽卡模型、简单保底抽卡模型）
2. **简单保底抽卡模型设计**（基于GGanalysis理论）
   - **综合概率（Comprehensive Probability）**：所有保底机制折算后的最终玩家期望概率
   - **期望抽数 E(X)**：1 / 综合概率（例如：1 / 0.016 ≈ 62.5 抽）
   - **基础概率与保底**：
     - 基础概率（p_i for small i）：例如0.6%
     - 硬保底（Hard Pity / N）：第90抽时，获得最高稀有度道具的条件概率p_90为100%
     - 软保底（Soft Pity）：从第74抽开始，条件概率p_i开始线性或指数级提升
   - **大保底机制（50/50 Pity System）**：针对限定UP卡池
     - 首次获得最高稀有度时：有50%的概率获得当期UP角色，50%的概率获得常驻角色
     - 触发"大保底"：如果首次未获得UP角色，则下一次获得的最高稀有度角色必定为当期UP角色
3. 设计各稀有度的基础概率
4. 设计概率递增机制（如连续未获得高稀有度时概率提升）
5. 设计硬保底机制（如90抽必得SSR）
6. 设计软保底机制（如大小保底）
7. 计算期望抽数和综合概率
8. 验证概率模型的数学合理性

**检查清单**：
- [ ] 抽卡模型类型已确定
- [ ] 基础概率已设计

## 操作注意事项

### 概率设计原则

1. **数学合理性**：确保概率模型的数学正确性，避免逻辑错误
2. **透明度**：概率和保底机制应清晰公示
3. **无偏统计**：使用无偏统计量进行数据监控，避免幸存者偏差
4. **极端情况**：考虑极端情况对玩家体验的影响

### 玩家体验

1. **奖励感**：抽卡动画和音效应增强奖励感
2. **保底透明**：保底机制应清晰展示给玩家
3. **玩家教育**：教育玩家理解概率和保底机制
4. **投诉处理**：建立玩家投诉处理机制

### 经济平衡

1. **产出控制**：控制高稀有度道具的产出率
2. **返还合理**：返还比例应合理，避免经济通胀
3. **定价匹配**：抽卡定价应与目标用户付费能力匹配
4. **监控完善**：建立完善的数据监控体系

### 法律合规

1. **概率公示**：按照法规要求公示抽卡概率
2. **数据保护**：保护玩家抽卡数据隐私
3. **玩家权益**：保障玩家合法权益
4. **风险提示**：提供适当的风险提示

## 常见错误

### 错误1：概率模型数学错误

**问题**：概率模型设计存在数学错误，导致期望抽数计算错误

**修正**：严格验证概率模型的数学正确性，使用正确的统计方法计算期望抽数和综合概率

### 错误2：保底机制设计不合理

**问题**：保底抽数设置过高或过低，影响玩家体验

**修正**：基于目标用户付费能力和游戏商业化目标，合理设计保底抽数

### 错误3：返还机制不完善

**问题**：重复道具的返还机制不完善，导致玩家不满

**修正**：设计合理的返还比例和兑换机制，确保重复道具有价值

### 错误4：缺乏透明度

**问题**：概率和保底机制不透明，导致玩家质疑

**修正**：清晰公示概率和保底机制，提供详细的说明

### 错误5：监控指标设计不当

**问题**：监控指标设计不当，导致数据偏差

**修正**：使用无偏统计量，避免幸存者偏差，设计合理的监控指标

### 错误6：忽视玩家感受

**问题**：忽视抽卡系统对玩家心理的影响

**修正**：设计抽卡动画和音效，增强奖励感，建立玩家教育和投诉处理机制

### 错误7：法律合规不足

**问题**：未考虑法律合规要求，存在合规风险

**修正**：按照法规要求公示概率，保护玩家数据隐私，保障玩家合法权益
