## 适用场景

- 优化游戏的爽感体验
- 提升游戏的核心爽点循环
- 设计浓缩策略（提升频率、加强强度、降低门槛）
- 跨界机制融合创新
- 构建双循环结构（核心循环+元循环）
- 构想游戏终极形态与市场定位

---

## 操作步骤

### 步骤1：读取PRD和核心玩法

**目标**：读取游戏PRD，理解核心玩法和当前设计

**操作方法**：
1. 读取游戏PRD文档
2. 提取游戏愿景和目标
3. 提取核心玩法循环
4. 提取主要游戏系统
5. 提取当前设计的优势和短板
6. 分析当前爽感体验的问题

**检查清单**：
- [ ] PRD文档已读取
- [ ] 游戏愿景已提取
- [ ] 核心玩法已提取
- [ ] 主要系统已提取
- [ ] 优势短板已分析
- [ ] 爽感问题已识别

---

### 步骤2：定义核心爽点循环

**目标**：精准识别游戏中最核心的、可被无限重复的"动作-反馈"循环

**操作方法**：
1. 分析当前核心玩法循环
2. 识别核心动作（玩家执行的操作）
3. 识别核心反馈（系统给予的反馈）
4. 提炼核心爽点循环（2-4个关键动词）
5. 确保循环可被无限重复
6. 验证循环的爽感强度

**检查清单**：
- [ ] 当前循环已分析
- [ ] 核心动作已识别
- [ ] 核心反馈已识别
- [ ] 核心爽点循环已提炼
- [ ] 循环可重复性已验证
- [ ] 爽感强度已验证

---

### 步骤3：实施浓缩策略

**目标**：针对核心爽点循环，提出具体的浓缩方案

**操作方法**：
1. **提升频率**
   - 分析当前循环发生的频率
   - 提出提升频率的具体方案
   - 评估频率提升对爽感的影响
2. **加强强度**
   - 分析当前反馈的强度
   - 提出加强强度的具体方案
   - 评估强度加强对爽感的影响
3. **降低门槛**
   - 分析当前循环启动的门槛
   - 提出降低门槛的具体方案
   - 评估门槛降低对爽感的影响
4. 整合浓缩策略方案

**检查清单**：
- [ ] 频率提升方案已提出
- [ ] 强度加强方案已提出
- [ ] 门槛降低方案已提出
- [ ] 方案影响已评估
- [ ] 浓缩策略已整合

---

### 步骤4：跨界机制融合

**目标**：从爆款游戏中借鉴成熟机制，解决当前设计短板或放大优点

**操作方法**：
1. **诊断当前设计短板**
   - 分析长线留存问题
   - 分析玩法单一问题
   - 分析重玩价值问题
   - 分析其他潜在短板
2. **提出嫁接方案**
   - 从海量游戏库中选择爆款游戏
   - 识别该游戏的成功机制
   - 提出将该机制嫁接到当前项目的方案
3. **阐述融合效果**
   - 说明该机制如何解决短板
   - 说明该机制带来的新化学反应
   - 说明该机制对游戏体验的影响

**检查清单**：
- [ ] 当前设计短板已诊断
- [ ] 爆款游戏已选择
- [ ] 成功机制已识别
- [ ] 嫁接方案已提出
- [ ] 融合效果已阐述

---

### 步骤5：构建双循环结构

**目标**：设计无缝衔接的"双循环"系统（核心循环+元循环）

**操作方法**：
1. **设计核心循环**
   - 基于步骤2-4强化的核心玩法循环
   - 定义循环产出的资源（资源A）
   - 确保循环的爽感和可重复性
2. **设计长线/元循环**
   - 基于步骤4融合的机制
   - 设计局外成长或收集系统
   - 定义循环消耗的资源（资源A）
   - 确保循环反哺核心循环
3. **验证双循环衔接**
   - 确保两个循环无缝衔接
   - 确保资源流动合理
   - 确保循环相互促进

**检查清单**：
- [ ] 核心循环已设计
- [ ] 资源产出已定义
- [ ] 元循环已设计
- [ ] 资源消耗已定义
- [ ] 双循环衔接已验证
- [ ] 循环相互促进已验证

### 步骤6：构想终极形态与市场定位

**目标**：描述"浓缩+融合"后的新物种

**操作方法**：
1. **描述终极体验**
   - 生动描述玩家的"上头"感受
   - 描述核心爽感和策略深度的结合
   - 描述长期留存和重玩价值
2. **定义新品类/标签**
   - 结合融合的机制定义新品类
   - 使用"A + B = C"的格式
   - 确保标签准确反映游戏特色
3. **分析市场定位**
   - 分析目标用户群体
   - 分析竞争优势
   - 分析市场机会

**检查清单**：

## 操作注意事项

### 设计哲学

1. **直觉优先于思考**：唤醒玩家的原始本能，让肌肉记忆取代深思熟虑
2. **爽感就是一切**：剔除一切阻碍核心爽感的"杂质"
3. **创新源于融合**：最强大的创新往往来自于将一个领域成熟的、被验证过的成功模式，"跨界移植"到另一个领域

### 浓缩策略原则

1. **提升频率**：让核心爽点循环发生得更多
   - 缩短循环时间
   - 增加循环触发条件
   - 减少循环间隔
2. **加强强度**：让反馈更具冲击力
   - 增强视觉反馈
   - 增强听觉反馈
   - 增强数值反馈
3. **降低门槛**：让循环更容易启动
   - 简化操作
   - 降低难度
   - 提供引导

### 跨界机制融合原则

1. **有据可依**：基于已被市场验证的爆款游戏模式
2. **解决短板**：针对当前设计的具体短板进行融合
3. **创造新意**：融合后产生新的化学反应
4. **保持一致性**：确保融合机制与游戏整体风格一致

### 双循环结构原则

1. **无缝衔接**：两个循环之间无缝衔接
2. **资源流动**：资源在两个循环之间合理流动
3. **相互促进**：两个循环相互促进，形成正向循环
4. **长期留存**：元循环驱动玩家进行"下一局"

## 常见错误

### 错误1：核心爽点循环识别不准确

**问题**：未能准确识别游戏的核心爽点循环

**修正**：深入分析核心玩法，识别玩家执行的核心动作和系统给予的核心反馈，提炼出2-4个关键动词组成的循环

### 错误2：浓缩策略过于激进

**问题**：过度提升频率或强度，导致游戏失去平衡

**修正**：平衡浓缩策略和游戏平衡，确保优化后的游戏仍然有挑战性和深度

### 错误3：跨界机制融合不恰当

**问题**：选择的机制与游戏风格不匹配，或融合后产生冲突

**修正**：选择与游戏风格匹配的机制，确保融合后产生正向的化学反应

### 错误4：双循环结构衔接不紧密

**问题**：核心循环和元循环之间衔接不紧密，资源流动不合理

**修正**：确保两个循环无缝衔接，资源流动合理，循环相互促进

### 错误5：忽视游戏平衡性

**问题**：过度追求爽感，忽视游戏平衡性

**修正**：在优化爽感的同时，确保游戏平衡性，避免Pay-to-Win或其他平衡问题

### 错误6：融合机制缺乏创新

**问题**：只是简单复制其他游戏的机制，缺乏创新

**修正**：在借鉴成功机制的基础上，进行创新和改进，创造出独特的游戏体验
