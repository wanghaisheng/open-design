## 适用场景

- 制定游戏7天和30天运营目标
- 设计用户习惯养成策略
- 分析用户行为障碍和激励方案
- 设计运营活动和产品功能优化
- 提升用户留存率和付费转化率
- 设计新手引导和用户激活策略

---

## 操作步骤

### 步骤1：读取参数和PRD

**目标**：读取游戏PRD和运营目标参数

**操作方法**：
1. 读取游戏PRD文档
2. 读取游戏类型（角色扮演、策略卡牌、休闲益智等）
3. 读取核心玩法简介
4. 读取目标用户画像
5. 读取当前运营阶段（早期增长期、成熟期等）
6. 读取核心运营目标（留存、付费、活跃度、社交连接等）

**检查清单**：
- [ ] PRD文档已读取
- [ ] 游戏类型已确定
- [ ] 核心玩法已理解
- [ ] 目标用户已分析
- [ ] 运营阶段已确认
- [ ] 核心运营目标已明确

---

### 步骤2：定义7天和30天运营目标

**目标**：设定清晰的7天和30天核心运营目标

**操作方法**：
1. 设定7天运营目标
   - 次日留存率 (Day 1 Retention Rate)
   - 7日留存率 (Day 7 Retention Rate)
   - 新手引导完成率 (New User Tutorial Completion Rate)
   - 首次付费转化率 (First Time Purchase Conversion Rate)
   - 核心玩法参与度 (Engagement with Core Gameplay)
2. 设定30天运营目标
   - 30日留存率 (Day 30 Retention Rate)
   - 月活跃用户数 (MAU)
   - 付费用户比例 (Paying User Percentage)
   - 平均每付费用户收入 (ARPPU)
   - 社交互动指标 (Social Interaction Metrics)
   - 深度玩法参与度 (Engagement with Advanced Gameplay)
3. 为每个目标设定具体指标数值
4. 初步识别与目标相关的关键用户行为

**检查清单**：
- [ ] 7天运营目标已设定
- [ ] 30天运营目标已设定
- [ ] 目标指标已量化
- [ ] 关键用户行为已初步识别

---

### 步骤3：精确化关键行为

**目标**：将关键用户行为用"不舒服地具体"的方式描述

**操作方法**：
1. 选择1-2个核心运营目标
2. 将相关的关键用户行为精确化
3. 使用"谁、做什么、何时、如何、期望频率"的描述方式
4. 例如："新用户在注册后的前7天内，每天至少登录游戏1次，并在登录后30分钟内完成至少3个标记为'日常'的引导任务"
5. 确保行为描述具体、可测量、可执行

**检查清单**：
- [ ] 核心目标已选择
- [ ] 关键行为已精确化
- [ ] 行为描述包含5W1H要素
- [ ] 行为可测量可执行

---

### 步骤4：分析核心障碍

**目标**：分析用户在执行关键行为时可能遇到的障碍

**操作方法**：
1. 分析后勤障碍
   - 任务入口难找？
   - 任务描述不清晰？
   - 操作繁琐？
   - 游戏加载慢导致中断？
   - 奖励领取不方便？
2. 分析认知障碍
   - 觉得任务无聊/重复？
   - 不明白任务的价值？
   - 害怕失败/难度过高？
   - 忘记登录（缺乏提醒）？
   - 有其他更有吸引力的游戏/事情？
   - 初期挫败感强？
3. 针对每个障碍提出具体描述和影响

**检查清单**：
- [ ] 后勤障碍已分析
- [ ] 认知障碍已分析
- [ ] 障碍描述已具体化
- [ ] 障碍影响已评估

---

### 步骤5：设计即时收益

**目标**：构思能够驱动用户完成关键行为的即时收益

**操作方法**：
1. 针对识别出的核心障碍，设计即时收益
2. 思考方向：
   - 完成任务后的即时奖励（游戏货币、道具、经验值）
   - 清晰的进度反馈和正向激励（动画、音效）
   - 解锁新内容/功能的兴奋感
   - 社交炫耀/认可
   - 每日登录的惊喜感（连续登录奖励递增）
   - 缓解焦虑（任务未完成时有清晰的提示和低门槛的帮助）
3. 确保即时收益能够克服对应障碍
4. 设计收益的即时性和可见性

**检查清单**：
- [ ] 即时收益已设计
- [ ] 收益与障碍对应
- [ ] 收益即时性已确保
- [ ] 收益可见性已设计

---

### 步骤6：整合3B分析

**目标**：将Behavior、Barriers、Benefits整合成完整的3B分析

**操作方法**：
1. 整理精确化的关键行为描述
2. 整理核心障碍列表（分类并详细描述）
3. 整理即时收益设计思路列表（详细说明如何克服障碍）
4. 验证3B框架的逻辑一致性
5. 确保每个障碍都有对应的收益解决方案

**检查清单**：
- [ ] 关键行为已整理
- [ ] 核心障碍已整理
- [ ] 即时收益已整理
- [ ] 3B逻辑一致性已验证
- [ ] 障碍收益对应关系已确认

---

### 步骤7：提出运营策略建议

**目标**：基于3B分析提出具体的运营活动或产品功能优化建议

**操作方法**：
1. 基于3B分析，提出3-5条具体的、可操作的运营活动建议
2. 明确每个建议针对的目标（7天目标或30天目标）
3. 清晰描述活动内容或功能优化点
4. 解释建议如何通过解决特定"障碍"并提供"即时收益"来驱动用户完成"关键行为"
5. 说明预期效果
6. 提供执行要点

**检查清单**：
- [ ] 运营策略建议已提出
- [ ] 建议针对目标已明确
- [ ] 活动内容已清晰描述
- [ ] 3B框架联系已说明
- [ ] 预期效果已说明
- [ ] 执行要点已提供

## 操作注意事项

### 3B框架应用原则

1. **行为精确化**：关键行为必须用"不舒服地具体"的方式描述，包含5W1H要素
2. **障碍分类**：区分后勤障碍和认知障碍，针对不同类型设计不同解决方案
3. **收益即时性**：即时收益必须真正即时，能够立即反馈给用户
4. **逻辑一致性**：确保每个障碍都有对应的收益解决方案

### 运营目标设定

1. **目标量化**：运营目标必须设定具体的指标数值
2. **目标合理性**：目标必须基于游戏类型和运营阶段设定，避免过高或过低
3. **目标关联性**：7天目标和30天目标之间应有逻辑关联和递进关系

### 运营策略设计

1. **可操作性**：运营策略建议必须具体可操作，避免泛泛而谈
2. **目标明确**：每个策略建议必须明确针对哪个运营目标
3. **3B联系**：每个策略建议必须说明如何通过3B框架驱动用户行为
4. **效果预期**：每个策略建议必须说明预期效果

### 数据驱动

1. **数据验证**：运营策略上线后必须通过数据验证效果
2. **A/B测试**：对于关键策略建议进行A/B测试
3. **迭代优化**：根据数据反馈持续优化运营策略

## 常见错误

### 错误1：关键行为描述不够具体

**问题**：关键行为描述过于笼统，缺乏具体性和可测量性

**修正**：使用"不舒服地具体"的方式描述行为，包含5W1H要素（谁、做什么、何时、如何、期望频率）

### 错误2：障碍分析不够深入

**问题**：障碍分析停留在表面，未区分后勤障碍和认知障碍

**修正**：深入分析后勤障碍和认知障碍，针对不同类型设计不同解决方案

### 错误3：即时收益不够即时

**问题**：设计的即时收益不够即时，无法立即反馈给用户

**修正**：确保即时收益真正即时，能够立即反馈给用户，增强激励效果

### 错误4：运营策略过于泛泛

**问题**：运营策略建议过于泛泛，缺乏具体可操作性

**修正**：运营策略建议必须具体可操作，明确针对哪个运营目标，说明如何通过3B框架驱动用户行为

### 错误5：目标设定不合理

**问题**：运营目标设定过高或过低，与游戏类型和运营阶段不匹配

**修正**：基于游戏类型和运营阶段设定合理的运营目标，确保目标具有挑战性但可实现

### 错误6：缺乏数据验证

**问题**：运营策略上线后缺乏数据验证，无法评估效果

**修正**：建立完善的数据监控体系，通过数据验证运营策略效果，持续迭代优化
