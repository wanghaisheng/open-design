# Balance Check

分析游戏平衡数据文件、公式和配置，识别异常值、破坏的进度曲线、退化策略和经济失衡。在修改任何平衡相关数据或设计后使用。

## 适用场景

- 修改平衡相关数据后需要验证
- 修改平衡设计后需要验证
- 需要识别游戏中的异常值
- 需要检测退化策略
- 需要分析经济失衡

## 操作步骤

### 步骤1：识别平衡领域

**目标**：确定分析范围和重点

**操作方法**：
1. 从参数识别平衡领域（战斗/经济/进度/掉落）
2. 确定分析范围（全系统或特定系统）
3. 确定分析重点（如特定武器或经济循环）

**检查清单**：
- [ ] 平衡领域已识别
- [ ] 分析范围已确定
- [ ] 分析重点已明确

### 步骤2：读取相关数据文件

**目标**：获取平衡数据

**操作方法**：
1. 读取 assets/data/ 中的数据文件
2. 读取 design/balance/ 中的数据文件
3. 读取配置文件
4. 记录所有数据源

**检查清单**：
- [ ] 数据文件已读取
- [ ] 配置文件已读取
- [ ] 数据源已记录

### 步骤3：读取设计文档

**目标**：理解设计意图和平衡目标

**操作方法**：
1. 读取 design/gdd/ 中的系统设计文档
2. 理解设计意图
3. 理解平衡目标

**检查清单**：
- [ ] 设计文档已读取
- [ ] 设计意图已理解
- [ ] 平衡目标已理解

### 步骤4：执行战斗平衡分析

**目标**：分析战斗系统平衡

**操作方法**：
1. **DPS计算**：计算每个武器/能力在每个力量等级的DPS
2. **击杀时间**：计算每个等级的击杀时间
3. **支配选项**：识别支配所有其他选项的选项
4. **防御状态**：检查防御选项是否能创造不可杀状态
5. **伤害交互**：验证伤害类型/抗性交互是否平衡

**检查清单**：
- [ ] DPS已计算
- [ ] 击杀时间已计算
- [ ] 支配选项已识别
- [ ] 防御状态已检查
- [ ] 伤害交互已验证

### 步骤5：执行经济平衡分析

**目标**：分析经济系统平衡

**操作方法**：
1. **资源源头和汇**：映射所有资源源头和汇及其流速
2. **资源积累**：随时间预测资源积累
3. **无限循环**：检查无限资源循环
4. **黄金汇**：验证黄金汇随黄金生成扩展
5. **物品价值**：检查是否有任何物品在任何阶段都不值得购买

**检查清单**：
- [ ] 资源源头和汇已映射
- [ ] 资源积累已预测
- [ ] 无限循环已检查
- [ ] 黄金汇已验证
- [ ] 物品价值已检查

### 步骤6：执行进度平衡分析

**目标**：分析进度系统平衡

**操作方法**：
1. **XP曲线**：绘制XP曲线和力量曲线
2. **死区**：检查死区（长时间无有意义进度）
3. **力量尖峰**：检查力量尖峰（能力突然跳跃）
4. **内容门控**：验证内容门控与预期玩家力量对齐
5. **跳过/grind策略**：检查跳过/grind策略是否破坏预期节奏

**检查清单**：
- [ ] XP曲线已绘制
- [ ] 死区已检查
- [ ] 力量尖峰已检查
- [ ] 内容门控已验证
- [ ] 跳过/grind策略已检查

### 步骤7：执行掉落平衡分析

**目标**：分析掉落系统平衡

**操作方法**：
1. **获取时间**：计算获取每个稀有度等级的预期时间
2. **保底计时器**：检查保底计时器数学
3. **物品有用性**：验证没有掉落在任何阶段完全无用
4. **库存压力**：检查库存压力vs获取率

**检查清单**：
- [ ] 获取时间已计算
- [ ] 保底计时器已检查
- [ ] 物品有用性已验证
- [ ] 库存压力已检查

### 步骤8：检测异常值

**目标**：识别数据中的异常值

**操作方法**：
1. 识别超出预期范围的值
2. 分析异常值的原因
3. 评估异常值的影响

**检查清单**：
- [ ] 异常值已识别
- [ ] 原因已分析
- [ ] 影响已评估

### 步骤9：识别退化策略

**目标**：识别破坏游戏体验的策略

**操作方法**：
1. 识别玩家可能采取的优化策略
2. 评估策略是否破坏设计意图
3. 描述策略和为什么有问题

**检查清单**：
- [ ] 退化策略已识别
- [ ] 影响已评估
- [ ] 描述已记录

### 步骤10：输出平衡检查报告

**目标**：生成结构化报告

**操作方法**：
1. 输出分析的数据源
2. 输出健康摘要（健康/关注/关键问题）
3. 输出检测到的异常值
4. 输出发现的退化策略
5. 输出进度分析
6. 输出推荐建议（优先级、问题、建议修复、影响）
7. 输出需要关注的值（具体值及建议调整）

**检查清单**：
- [ ] 数据源已输出
- [ ] 健康摘要已输出
- [ ] 异常值已输出
- [ ] 退化策略已输出
- [ ] 进度分析已输出
- [ ] 推荐建议已输出
- [ ] 需要关注的值已输出

## 操作注意事项

### 数据驱动的分析

1. **基于数据分析**：所有分析基于实际数据，避免直觉判断
2. **使用数学验证**：使用数学公式验证平衡性
3. **避免主观判断**：不基于感觉，基于可验证数据

### 系统性分析

1. **全面分析系统**：不只看单个值，要看整个系统
2. **识别连锁影响**：识别一个值变化对其他系统的影响
3. **考虑边缘情况**：考虑极端情况和边缘案例

### 可操作的建议

1. **提供具体建议**：不只说"不平衡"，要提供具体数值调整
2. **提供优先级**：按优先级排序建议
3. **说明影响**：说明每个调整的影响

### 持续监控

1. **定期检查**：定期运行平衡检查
2. **持续监控**：持续监控平衡状态
3. **及时调整**：及时调整发现的问题

## 常见错误

### 错误1：基于直觉判断

**问题**：不基于数据分析，凭感觉判断平衡

**修正**：所有分析必须基于实际数据和数学验证

### 错误2：忽略连锁影响

**问题**：只看单个值，不看对其他系统的影响

**修正**：必须识别连锁影响，考虑整个系统

### 错误3：建议不具体

**问题**：只说"需要调整"而不说明具体数值

**修正**：必须提供具体数值调整和理由

### 错误4：忽略退化策略

**问题**：不检查玩家可能采取的优化策略

**修正**：必须识别玩家可能采取的退化策略

### 错误5：不检查边缘情况

**问题**：只检查正常情况，不考虑极端情况

**修正**：必须考虑边缘情况和极端案例
