# Balance Check

分析游戏平衡数据文件、公式和配置，识别异常值、破坏的进度曲线、退化策略和经济失衡。在修改任何平衡相关数据或设计后使用。

## 平衡分析要素

### 1. 战斗平衡 (Combat Balance)
- DPS计算：每个武器/能力在每个力量等级的DPS
- 击杀时间：每个等级的击杀时间
- 支配选项：识别支配所有其他选项的选项
- 防御状态：检查防御选项是否能创造不可杀状态
- 伤害交互：验证伤害类型/抗性交互是否平衡

### 2. 经济平衡 (Economy Balance)
- 资源源头和汇：映射所有资源源头和汇及其流速
- 资源积累：随时间预测资源积累
- 无限循环：检查无限资源循环
- 黄金汇：验证黄金汇随黄金生成扩展
- 物品价值：检查是否有任何物品在任何阶段都不值得购买

### 3. 进度平衡 (Progression Balance)
- XP曲线：绘制XP曲线和力量曲线
- 死区：检查死区（长时间无有意义进度）
- 力量尖峰：检查力量尖峰（能力突然跳跃）
- 内容门控：验证内容门控与预期玩家力量对齐
- 跳过/ grind策略：检查跳过/grind策略是否破坏预期节奏

### 4. 掉落平衡 (Loot Balance)
- 获取时间：计算获取每个稀有度等级的预期时间
- 保底计时器：检查保底计时器数学
- 物品有用性：验证没有掉落在任何阶段完全无用
- 库存压力：检查库存压力vs获取率

## 平衡分析原则

### 1. 数据驱动
- 基于数据分析
- 避免直觉判断
- 使用数学验证

### 2. 系统性
- 全面分析系统
- 识别连锁影响
- 考虑边缘情况

### 3. 可操作性
- 提供具体建议
- 提供数值调整
- 提供优先级

### 4. 持续性
- 定期检查
- 持续监控
- 及时调整

## 平衡分析流程

### 1. 识别平衡领域
- 从参数识别平衡领域
- 确定分析范围
- 确定分析重点

### 2. 读取相关数据文件
- 读取 assets/data/ 中的数据文件
- 读取 design/balance/ 中的数据文件
- 读取配置文件

### 3. 读取设计文档
- 读取 design/gdd/ 中的系统设计文档
- 理解设计意图
- 理解平衡目标

### 4. 执行分析
- 战斗平衡分析
- 经济平衡分析
- 进度平衡分析
- 掉落平衡分析

### 5. 输出分析报告
- 健康摘要
- 异常值检测
- 退化策略
- 进度分析
- 推荐建议
