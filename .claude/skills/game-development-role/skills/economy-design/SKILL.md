# Economy Design

设计游戏经济系统，定义虚拟货币、奖励机制和进阶曲线，确保经济平衡和玩家激励。

## 适用场景

- 从 0 到 1 设计游戏经济系统（货币、奖励、消费、进阶）
- 需要把经济系统拆解为可配置的货币获取/消费规则、数值区间与平衡参数
- 经济系统出现通货膨胀/通货紧缩、玩家缺乏激励、进阶曲线不合理等问题，需要迭代优化
- 需要与系统设计/数值平衡/商业化协作，明确经济与系统/关卡的结合点

## 操作步骤

### 步骤1：把经济目标写成"可验收"的一句话

**目标**：明确经济系统服务什么体验目标、成功标准是什么

**操作方法**：
1. 写经济主题与一句话目标（面向玩家动机与商业模式）
2. 列出经济输入/输出（玩家行为、货币流入流出、资源变化）
3. 写 3-5 条成功标准（例如：货币流入流出平衡在 ±10% 以内；玩家愿意为进阶消费）
4. 写 3-5 条失败信号（例如：货币大量囤积导致通胀；玩家不知道货币用途）
5. **核心价值准则**：确立"时间是衡量虚拟物品价值的唯一标尺"原则
   - 游戏内所有有价物品的价值，最终都应能通过"获取所需平均时间"来进行锚定和换算
   - 付费内容的价值应基于节省的时间成本来定价

**检查清单**：
- [ ] 有清晰边界：经济系统"包含什么/不包含什么"
- [ ] 有成功标准：可观测、可测试
- [ ] 有失败信号：能快速触发调整

### 步骤2：用"货币-奖励-消费-进阶"四块拆解经济

**目标**：把经济拆成可实现的设计单元，避免只写概念

**操作方法**：
1. **货币系统**：列主货币/次级货币（每条写获取方式、消费用途、转换规则、上限/下限）
2. **奖励机制**：列即时奖励/延迟奖励/随机奖励/固定奖励（每条写触发时机、数值区间、概率分布）
3. **消费系统**：列消费类型（每条写消费价值、消费时机、消费限制、折扣/加成规则）
4. **进阶曲线**：列线性/指数/对数/混合进阶（每条写曲线公式、关键节点、解锁条件、时间成本）

**检查清单**：
- [ ] 每种货币都有获取方式与消费用途
- [ ] 每种奖励都有触发时机与数值区间
- [ ] 每种消费都有价值与限制
- [ ] 每条进阶曲线都有公式与关键节点

### 步骤3：设计货币流入流出平衡表

**目标**：确保经济系统可长期运行，避免通胀/通缩

**操作方法**：
1. 列所有货币流入渠道（任务/活动/战斗/社交/购买）
2. 列所有货币流出渠道（消费/交易/损耗/过期）
3. 为每个渠道写基准数值与波动范围（例如：每日任务 100-150 货币）
4. 计算流入流出比（目标 0.9-1.1，允许短期偏离但长期回归）
5. 设计调节机制（动态调整、上限控制、货币回收）

**检查清单**：
- [ ] 所有流入流出渠道已列出
- [ ] 每个渠道有基准数值与波动范围
- [ ] 流入流出比在目标区间
- [ ] 有调节机制应对偏离

### 步骤4：设计进阶曲线与关键节点

**目标**：让玩家有持续进阶动力，避免前期过快/后期过慢

**操作方法**：
1. 画进阶曲线图（时间 vs 进度，标注关键节点）
2. 为每个关键节点写解锁条件（等级/时间/成就/消费）
3. 为每个关键节点写奖励内容（货币/物品/能力/社交）
4. 设计进阶节奏（新手期快速、成长期平稳、成熟期长期）
5. 设计跳过/加速机制（付费/活动/奖励）

**检查清单**：
- [ ] 进阶曲线图完整
- [ ] 每个关键节点有解锁条件与奖励
- [ ] 进阶节奏合理
- [ ] 有跳过/加速机制

### 步骤5：输出经济规格并组织测试迭代

**目标**：让程序与设计能直接落地，并能高效迭代

**操作方法**：
1. 输出货币配置表（类型、获取、消费、转换、上限）
2. 输出奖励配置表（类型、触发、数值、概率）
3. 输出消费配置表（类型、价值、时机、限制）
4. 输出进阶配置表（曲线公式、关键节点、解锁条件）
5. 输出平衡参数表（流入流出比、调节阈值、波动范围）
6. **输出CSV配置数据**：生成可直接导入游戏引擎的CSV格式配置
   - CSV格式表头：ID, Name, Type, InitialAmount, MaxCap, Description
   - ID：唯一标识符（大写英文，如GOLD, DIAMOND, STAMINA）
   - Name：显示名称
   - Type：类型（Currency, Premium, Item, Replenishable）
   - InitialAmount：初始数量
   - MaxCap：上限（无上限填999999999）
   - Description：简短描述
7. 收集数据与反馈（货币存量、消费行为、进阶速度）并形成迭代记录

**检查清单**：
- [ ] 所有配置表完整且可配置
- [ ] 平衡参数有默认值与可调范围
- [ ] 测试用例可复现
- [ ] 有迭代记录（问题-假设-改动-结果）

## 操作注意事项

### 平衡性与可持续性

1. 货币流入流出必须长期平衡，避免通胀/通缩。
2. 必须有货币回收机制（消耗、过期、上限），防止无限囤积。

### 激励性与可玩性

1. 奖励必须与付出匹配，避免"付出很多、奖励很少"或"付出很少、奖励很多"。
2. 进阶曲线必须让玩家看到希望，避免后期进阶成本过高导致放弃。

### 可调整性与数据驱动

1. 所有数值必须可配置，支持动态调整与 A/B 测试。
2. 必须有数据监控（货币存量、消费行为、进阶速度），及时发现问题。

## 输出格式

### 经济设计文档

- 经济系统目标、约束、验收标准、失败信号
- 货币配置表（类型、获取、消费、转换、上限）
- 奖励配置表（类型、触发、数值、概率）
- 消费配置表（类型、价值、时机、限制）
- 进阶配置表（曲线公式、关键节点、解锁条件）
- 平衡参数表（流入流出比、调节阈值、波动范围）
- 调节机制（动态调整、上限控制、货币回收）
- 测试用例与迭代记录

## 协作协议

### 向谁汇报

- lead-designer：经济与整体体验目标对齐
- technical-director：经济可实现性与性能约束
- monetization-director：经济与商业化目标对齐

### 协调对象

- system-designer：经济与系统结合点（任务/活动/社交）
- level-designer：经济与关卡结合点（奖励/消费/进阶）
- balance-designer：数值平衡与难度曲线
- qa-tester：经济测试、数据收集、迭代验证

## 常见错误

### 错误1：只写"主题/氛围"，没有具体的货币配置与平衡参数

**问题**：实现阶段无法拆分任务，经济容易空洞或不可落地。

**修正**：按步骤2/3补齐货币配置表与平衡参数表。

### 错误2：货币流入流出不平衡，导致通胀/通缩

**问题**：玩家货币大量囤积或严重短缺，失去游戏动力。

**修正**：在步骤3重新评估流入流出渠道，增加调节机制（上限控制、货币回收）。

### 错误3：进阶曲线不合理，前期过快/后期过慢

**问题**：新手期快速失去目标，成熟期进阶成本过高导致放弃。

**修正**：在步骤4重新设计进阶曲线，确保节奏合理（新手期快速、成长期平稳、成熟期长期）。
