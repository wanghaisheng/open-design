# Scope Check

通过将当前范围与原始计划对比来分析功能或冲刺的范围蔓延。标记添加、量化膨胀，并建议削减。

## 适用场景

- 需要分析范围蔓延
- 需要对比原始vs当前范围
- 需要评估范围风险
- 需要建议范围削减
- 需要量化范围变化

## 操作步骤

### 步骤1：读取原始计划

**目标**：读取原始计划

**操作方法**：
1. 功能名称：读取 design/gdd/ 中的设计文档
2. 冲刺编号：读取 production/sprints/ 中的冲刺计划
3. 里程碑：读取 production/milestones/ 中的里程碑定义

**检查清单**：
- [ ] 设计文档已读取
- [ ] 冲刺计划已读取
- [ ] 里程碑定义已读取

### 步骤2：读取当前状态

**目标**：读取当前状态

**操作方法**：
1. 扫描代码库中与功能/冲刺相关的文件
2. 读取与此工作相关的git提交
3. 检查TODO注释以指示未完成范围添加

**检查清单**：
- [ ] 代码库已扫描
- [ ] git提交已读取
- [ ] TODO注释已检查

### 步骤3：对比原始vs当前范围

**目标**：对比范围

**操作方法**：
1. 列出原始范围（原始项目列表）
2. 列出当前范围（当前项目列表）
3. 识别范围添加（不在原始计划中的添加）
4. 识别范围移除（在原始中但已删除的项目）
5. 计算膨胀评分（原始项目数、当前项目数、净范围变化）

**检查清单**：
- [ ] 原始范围已列出
- [ ] 当前范围已列出
- [ ] 范围添加已识别
- [ ] 范围移除已识别
- [ ] 膨胀评分已计算

### 步骤4：评估风险

**目标**：评估范围风险

**操作方法**：
1. 进度风险（进度影响评估）
2. 质量风险（质量影响评估）
3. 集成风险（集成影响评估）

**检查清单**：
- [ ] 进度风险已评估
- [ ] 质量风险已评估
- [ ] 集成风险已评估

### 步骤5：输出建议

**目标**：输出范围建议

**操作方法**：
1. 削减建议（应删除以保持时间线的项目）
2. 延期建议（可移动到未来冲刺/版本的项目）
3. 保留建议（真正必要的添加）
4. 标记建议（需要制作人/创意总监决策的项目）

**检查清单**：
- [ ] 削减建议已输出
- [ ] 延期建议已输出
- [ ] 保留建议已输出
- [ ] 标记建议已输出

## 输出格式

### 范围检查报告
```markdown
## 范围检查：[功能/冲刺名称]
生成日期：[日期]

### 原始范围
[原始计划中的项目列表]

### 当前范围
[当前实现或进行中的项目列表]

### 范围添加（不在原始计划中）
| 添加 | 添加者 | 时间 | 合理？ | 工作量 |
|------|--------|------|--------|--------|
| [项目] | [提交/人] | [日期] | [是/否/不清楚] | [小/中/大] |

### 范围移除（在原始中但已删除）
| 删除项目 | 原因 | 影响 |
|----------|------|------|
| [项目] | [为什么删除] | [受影响内容] |

### 膨胀评分
- 原始项目：[N]
- 当前项目：[N]
- 添加项目：[N] (+[X]%)
- 删除项目：[N]
- 净范围变化：[+/-N] ([X]%)

### 风险评估
- **进度风险**：[低/中/高] — [解释]
- **质量风险**：[低/中/高] — [解释]
- **集成风险**：[低/中/高] — [解释]

### 建议
1. **削减**：[应删除以保持时间线的项目]
2. **延期**：[可移动到未来冲刺/版本的项目]
3. **保留**：[真正必要的添加]
4. **标记**：[需要制作人/创意总监决策的项目]
```

### 决策
- **按计划**：范围在原始10%以内
- **轻微蔓延**：10-25%范围增加——可通过调整管理
- **显著蔓延**：25-50%范围增加——需要削减或延长时间线
- **失控**：>50%范围增加——停止并重新规划

## 操作注意事项

### 量化分析

1. **始终量化范围变化**：确保始终量化范围变化
2. **使用具体数字**：确保使用具体数字
3. **避免模糊描述**：避免"感觉更大"等模糊描述

### 合理性评估

1. **不是所有添加都是坏的**：确保不是所有添加都是坏的
2. **有些是发现的需求**：确保有些是发现的需求
3. **必须确认并考虑**：确保必须确认并考虑

### 风险管理

1. **评估进度风险**：确保评估进度风险
2. **评估质量风险**：确保评估质量风险
3. **评估集成风险**：确保评估集成风险

### 优先级管理

1. **优先保护核心玩家体验**：确保优先保护核心玩家体验
2. **削减锦上添花**：确保削减锦上添花
3. **保留真正必要的添加**：确保保留真正必要的添加

## 常见错误

### 错误1：不读取原始计划

**问题**：不读取设计文档、不读取冲刺计划

**修正**：必须读取设计文档，冲刺计划，里程碑定义

### 错误2：不读取当前状态

**问题**：不扫描代码库、不读取git提交

**修正**：必须扫描代码库，读取git提交，检查TODO注释

### 错误3：不对比范围

**问题**：不列出原始范围、不列出当前范围

**修正**：必须列出原始范围，当前范围，识别范围添加和移除

### 错误4：不评估风险

**问题**：不评估进度风险、不评估质量风险

**修正**：必须评估进度风险，质量风险，集成风险

### 错误5：不量化范围变化

**问题**：不计算膨胀评分、不使用具体数字

**修正**：必须计算膨胀评分，使用具体数字，避免模糊描述
