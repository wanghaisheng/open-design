# Visual Exploration

探索不同视觉方向，通过多个概念变体探索游戏视觉风格，找到最佳视觉方案。

## 适用场景

- 需要探索视觉风格
- 需要创作概念变体
- 需要对比视觉方案
- 需要评估方案可行性
- 需要推荐最佳方案

## 操作步骤

### 步骤1：需求理解

**目标**：理解视觉需求

**操作方法**：
1. 理解游戏需求（游戏类型、目标受众、平台要求）
2. 理解视觉目标（视觉风格、视觉氛围、视觉定位）
3. 确定探索方向（风格方向、色彩方向、造型方向）

**检查清单**：
- [ ] 游戏需求已理解
- [ ] 视觉目标已理解
- [ ] 探索方向已确定

### 步骤2：方案设计

**目标**：设计探索方案

**操作方法**：
1. 设计探索方向（风格探索、色彩探索、造型探索、氛围探索）
2. 设计探索方案（每个方向的具体方案）
3. 设计探索数量（变体数量、覆盖范围）

**检查清单**：
- [ ] 探索方向已设计
- [ ] 探索方案已设计
- [ ] 探索数量已设计

### 步骤3：变体创作

**目标**：创作概念变体

**操作方法**：
1. 创作概念变体（多个概念版本）
2. 创作风格变体（不同风格版本）
3. 创作色彩变体（不同色彩方案）
4. 创作细节变体（不同细节处理）

**检查清单**：
- [ ] 概念变体已创作
- [ ] 风格变体已创作
- [ ] 色彩变体已创作
- [ ] 细节变体已创作

### 步骤4：方案对比

**目标**：对比和评估方案

**操作方法**：
1. 对比不同方案（视觉对比、风格对比）
2. 评估方案优劣（可行性对比、成本对比）
3. 推荐最佳方案（推荐理由、选择建议）

**检查清单**：
- [ ] 不同方案已对比
- [ ] 方案优劣已评估
- [ ] 最佳方案已推荐

### 步骤5：探索交付

**目标**：交付探索结果

**操作方法**：
1. 交付探索结果（概念变体图集、风格变体图集）
2. 编写对比报告（方案对比文档、评估结果）
3. 提供选择建议（选择建议文档、实施建议）

**检查清单**：
- [ ] 探索结果已交付
- [ ] 对比报告已编写
- [ ] 选择建议已提供

## 输出格式

### 探索图集
- 概念变体图集
- 风格变体图集
- 色彩变体图集

### 文档
- 探索报告
- 方案对比文档
- 选择建议文档

## 操作注意事项

### 多样性

1. **探索多种风格**：确保探索多种风格
2. **探索多种方案**：确保探索多种方案
3. **避免单一思路**：避免单一思路

### 深度

1. **每个方向深入探索**：确保每个方向深入探索
2. **每个方案充分发展**：确保每个方案充分发展
3. **避免表面探索**：避免表面探索

### 对比性

1. **方案之间有对比**：确保方案之间有对比
2. **风格之间有区别**：确保风格之间有区别
3. **便于选择**：确保便于选择

### 可行性

1. **考虑技术可行性**：确保考虑技术可行性
2. **考虑资源可行性**：确保考虑资源可行性
3. **考虑时间可行性**：确保考虑时间可行性

## 常见错误

### 错误1：不理解视觉需求

**问题**：不理解游戏需求、不确定视觉目标

**修正**：必须理解游戏需求，理解视觉目标，确定探索方向

### 错误2：不设计探索方案

**问题**：不设计探索方向、不设计探索方案

**修正**：必须设计探索方向，设计探索方案，设计探索数量

### 错误3：不创作概念变体

**问题**：不创作概念变体、不创作风格变体

**修正**：必须创作概念变体，风格变体，色彩变体，细节变体

### 错误4：不对比方案

**问题**：不对比不同方案、不评估方案优劣

**修正**：必须对比不同方案，评估方案优劣，推荐最佳方案

### 错误5：不考虑可行性

**问题**：不考虑技术可行性、不考虑资源可行性

**修正**：必须考虑技术可行性，资源可行性，时间可行性
