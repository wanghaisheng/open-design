# Narrative Writing

编写游戏故事和叙事内容，创造引人入胜的游戏故事和角色背景。

## 适用场景

- 需要编写故事结构
- 需要设计角色发展
- 需要编写对话
- 需要设计叙事分支
- 需要整合叙事到游戏

## 操作步骤

### 步骤1：故事概念

**目标**：定义故事概念

**操作方法**：
1. 定义故事主题（故事主题确定）
2. 确定故事目标（故事目标确定）
3. 设计故事结构（主线故事、支线故事、背景故事、角色故事）

**检查清单**：
- [ ] 故事主题已定义
- [ ] 故事目标已确定
- [ ] 故事结构已设计

### 步骤2：角色设计

**目标**：设计角色

**操作方法**：
1. 设计主要角色（角色背景和性格）
2. 设计次要角色（角色背景和性格）
3. 设计角色关系（角色之间的关系）

**检查清单**：
- [ ] 主要角色已设计
- [ ] 次要角色已设计
- [ ] 角色关系已设计

### 步骤3：对话编写

**目标**：编写对话

**操作方法**：
1. 编写对话内容（使用 Word/Google Docs/Final Draft/Celtx）
2. 设计对话分支（对话分支设计）
3. 设计对话触发（对话触发条件）

**检查清单**：
- [ ] 对话内容已编写
- [ ] 对话分支已设计
- [ ] 对话触发已设计

### 步骤4：叙事分支

**目标**：设计叙事分支

**操作方法**：
1. 设计选择系统（玩家选择系统）
2. 设计后果系统（选择后果系统）
3. 设计多结局（不同结局设计）

**检查清单**：
- [ ] 选择系统已设计
- [ ] 后果系统已设计
- [ ] 多结局已设计

### 步骤5：叙事整合

**目标**：整合叙事到游戏

**操作方法**：
1. 整合到游戏（游戏整合）
2. 整合到玩法（玩法整合）
3. 整合到世界（世界整合）

**检查清单**：
- [ ] 已整合到游戏
- [ ] 已整合到玩法
- [ ] 已整合到世界

### 步骤6：测试优化

**目标**：测试和优化

**操作方法**：
1. 玩家测试（玩家测试）
2. 叙事测试（叙事测试）
3. 迭代优化（迭代优化）

**检查清单**：
- [ ] 玩家测试已完成
- [ ] 叙事测试已完成
- [ ] 迭代优化已完成

## 输出格式

### 叙事文档
- 故事概述
- 故事结构
- 角色设定
- 对话系统
- 叙事分支
- 整合计划
- 测试计划

## 操作注意事项

### 沉浸感

1. **故事引人入胜**：确保故事引人入胜
2. **角色真实可信**：确保角色真实可信
3. **世界丰富完整**：确保世界丰富完整

### 互动性

1. **玩家选择有意义**：确保玩家选择有意义
2. **选择影响故事**：确保选择影响故事
3. **后果合理**：确保后果合理

### 一致性

1. **叙事内部一致**：确保叙事内部一致
2. **角色行为一致**：确保角色行为一致
3. **世界规则一致**：确保世界规则一致

### 可玩性

1. **叙事与玩法结合**：确保叙事与玩法结合
2. **不打断游戏体验**：确保不打断游戏体验
3. **增强游戏体验**：确保增强游戏体验

## 协作协议

### 向谁汇报

- **narrative-director**：叙事质量和方向
- **creative-director**：创意方向
- **writer**：叙事内容

### 协调对象

- **dialogue-writing**：对话编写
- **character-designer**：角色设计
- **level-designer**：关卡中的叙事实现

## 常见错误

### 错误1：不定义故事概念

**问题**：不定义故事主题、目标、结构，直接编写

**修正**：必须先定义故事主题、目标、结构

### 错误2：不设计角色

**问题**：不设计主要角色、次要角色、角色关系

**修正**：必须设计主要角色、次要角色、角色关系

### 错误3：不设计叙事分支

**问题**：不设计选择系统、后果系统、多结局

**修正**：必须设计选择系统、后果系统、多结局

### 错误4：不整合叙事

**问题**：不整合叙事到游戏、玩法、世界

**修正**：必须整合叙事到游戏、玩法、世界

### 错误5：不测试叙事

**问题**：不进行玩家测试、叙事测试

**修正**：必须进行玩家测试、叙事测试、迭代优化
