# Team Narrative

协调叙事团队：协调叙事总监、编剧、世界构建者和关卡设计师，创建连贯的故事内容、世界传说和叙事驱动的关卡设计。

## 适用场景

- 需要创建故事内容
- 需要创建世界传说
- 需要叙事驱动的关卡设计
- 需要编写对话
- 需要本地化叙事内容

## 操作步骤

### 步骤1：叙事方向

**目标**：定义叙事方向

**操作方法**：
1. 委托给叙事总监（定义内容的叙事目的、识别涉及的角色和动机、设置情感基调和节奏目标、指定传说依赖或新传说）
2. 输出带故事要求的叙事简报

**检查清单**：
- [ ] 叙事目的已定义
- [ ] 角色和动机已识别
- [ ] 情感基调已设置
- [ ] 传说依赖已指定
- [ ] 叙事简报已输出

### 步骤2：世界基础

**目标**：创建世界基础

**操作方法**：
1. 并行委托（世界构建者、编剧）
2. 世界构建者：创建或更新派系、位置和历史的传说条目、与现有传说交叉引用、为新条目设置正典级别
3. 编剧：使用声音配置文件起草角色对话、确保所有行在120字符以下、使用命名占位符表示变量、准备好本地化

**检查清单**：
- [ ] 传说条目已创建或更新
- [ ] 传说交叉引用已完成
- [ ] 正典级别已设置
- [ ] 角色对话已起草
- [ ] 字符限制已检查
- [ ] 本地化已准备

### 步骤3：关卡叙事集成

**目标**：集成关卡叙事

**操作方法**：
1. 委托给关卡设计师（审查叙事简报和传说基础、在关卡中设计环境叙事元素、放置叙事触发器和对话区域、确保节奏服务游戏玩法和故事）

**检查清单**：
- [ ] 叙事简报已审查
- [ ] 传说基础已审查
- [ ] 环境叙事元素已设计
- [ ] 叙事触发器已放置
- [ ] 节奏已对齐

### 步骤4：审查和一致性

**目标**：审查和一致性检查

**操作方法**：
1. 委托给叙事总监（根据角色声音配置文件审查所有对话、验证新旧条目之间的传说一致性、确认叙事节奏与关卡设计对齐、检查所有谜团都有记录的"真实答案"）

**检查清单**：
- [ ] 对话已审查
- [ ] 传说一致性已验证
- [ ] 叙事节奏已对齐
- [ ] 谜团解答已记录

### 步骤5：润色

**目标**：润色叙事内容

**操作方法**：
1. 编剧审查所有文本的本地化准备度
2. 验证没有行超过对话框约束
3. 确认所有文本使用字符串键（本地化管道就绪）
4. 世界构建者为所有新传说最终确定正典级别

**检查清单**：
- [ ] 本地化准备度已审查
- [ ] 对话框约束已验证
- [ ] 字符串键已确认
- [ ] 正典级别已最终确定

## 常见错误

### 错误1：不定义叙事方向

**问题**：不定义叙事目的、角色动机

**修正**：必须委托给叙事总监，定义叙事方向

### 错误2：不创建世界基础

**问题**：不创建传说条目、不交叉引用

**修正**：必须委托给世界构建者和编剧，创建世界基础

### 错误3：不集成关卡叙事

**问题**：不设计环境叙事元素、不放置叙事触发器

**修正**：必须委托给关卡设计师，集成关卡叙事

### 错误4：不审查一致性

**问题**：不审查对话、不验证传说一致性

**修正**：必须委托给叙事总监，审查和一致性检查

### 错误5：不润色内容

**问题**：不审查本地化准备度、不验证对话框约束

**修正**：必须编剧审查本地化准备度，验证对话框约束
