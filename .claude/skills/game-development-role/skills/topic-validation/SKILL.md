# Topic Validation

题材验证技能，使用两大滤网（广告素材潜力、系统深度潜力）验证游戏题材。

## 适用场景

- 验证游戏题材潜力
- 评估CPI（获客成本）潜力
- 评估LTV（用户终身价值）潜力
- 立项决策支持
- 避免无效开发

## 操作步骤

### 步骤1：广告素材潜力测试（CPI潜力）

**目标**：评估题材能否产生低CPI的广告素材

**操作方法**：
1. **核心问题测试**
   - 这个创意能否在15-30秒视频里展示满足感或挫败感场景？
   - 能否在5秒内让用户看懂核心玩法？

2. **具体执行要点**
   - **5秒原则**：视频前5秒内让用户看懂核心玩法
   - **情绪放大**：展示"挑战失败"场景激发好奇心
   - **场景代入**：使用生活场景增加代入感
   - **视觉优化**：强色彩对比，简化画面
   - **本土化**：制作多语言素材，加入文化元素

3. **评估标准**
   - 如果很难用短视频讲清楚 → CPI可能超标 → 淘汰
   - 如果核心爽点不直观 → CPI可能超标 → 淘汰

**检查清单**：
- [ ] 5秒原则已验证
- [ ] 情绪放大已设计
- [ ] 场景代入已实现
- [ ] 视觉优化已完成
- [ ] 本土化已考虑

### 步骤2：系统深度潜力测试（LTV潜力）

**目标**：评估题材能否支撑长期生态系统

**操作方法**：
1. **核心问题测试**
   - 这个题材能否自然支持收集、成长、社交？

2. **四大维度考量**
   - **收集（Collection）**
     - 有足够丰富的东西让玩家收集吗？
     - 英雄、武器、皮肤、卡牌等

   - **成长（Progression）**
     - 收集到的东西能成长吗？
     - 有清晰可视化的成长路径吗？
     - 英雄升级、装备锻造、技能树等

   - **策略与社交（Strategy & Social）**
     - **PVP**：能否支持实时或异步对抗？
     - **PVE/合作**：能否支持联盟、公会合作？
     - **策略深度**：能否构建克制系统？

3. **评估标准**
   - 无法支撑收集/成长/社交 → LTV潜力低 → 淘汰
   - 天然适合三大要素 → 具备混合休闲潜力 → 继续

**检查清单**：
- [ ] 收集维度已评估
- [ ] 成长维度已评估
- [ ] 社交维度已评估
- [ ] 策略深度已评估
- [ ] 长期潜力已确认

### 步骤3：综合评估与决策

**目标**：基于两大滤网结果做出决策

**操作方法**：
1. **通过两道滤网** → 继续开发
2. **只通过滤网一** → 可能是纯超休闲，考虑是否转型
3. **只通过滤网二** → CPI可能过高，重新设计
4. **两道都不通过** → 立即淘汰

**检查清单**：
- [ ] CPI潜力已评估
- [ ] LTV潜力已评估
- [ ] 综合评估已完成
- [ ] 决策已做出

## 操作注意事项

### 验证原则

1. **严格过滤**：不达标的题材果断淘汰
2. **数据驱动**：用数据和事实说话
3. **早期决策**：在立项阶段就做出决策

### CPI评估

1. **用户视角**：从用户角度理解广告
2. **情绪设计**：设计强烈的情绪体验
3. **文化适配**：考虑不同市场文化

### LTV评估

1. **长期视角**：考虑6-12个月的发展
2. **系统协同**：确保各系统相互促进
3. **玩家分层**：考虑不同类型玩家需求

## 协作协议

### 向谁汇报

- creative-director：验证结果和决策
- producer：立项决策支持

### 协调对象

- marketing-team：广告素材评估
- game-designer：系统设计评估

## 常见错误

### 错误1：只关注CPI

**问题**：只关注广告潜力，忽视长期系统

**修正**：必须同时通过两道滤网

### 错误2：过度乐观

**问题**：对题材潜力过度乐观，忽视风险

**修正**：客观评估，用数据说话

### 错误3：不舍得淘汰

**问题**：对喜欢的题材不舍得淘汰

**修正**：严格按标准决策，果断止损

### 错误4：忽视本土化

**问题**：不考虑不同市场的文化差异

**修正**：为目标市场设计本土化方案

### 错误5：系统设计不足

**问题**：系统深度设计不足，无法支撑长期

**修正**：在验证阶段就设计完整的系统框架
