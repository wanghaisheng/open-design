# Brainstorm

引导游戏概念创意生成 - 从零想法到结构化游戏概念文档。使用专业工作室创意技巧、玩家心理框架和结构化创意探索。

## 适用场景

- 需要从零开始生成游戏概念
- 需要探索多个游戏概念方向
- 需要定义游戏的核心循环和支柱
- 需要验证概念的可行性
- 需要生成结构化的游戏概念文档

## 操作步骤

### 步骤1：创意发现 - 理解人和约束

**目标**：建立创意简报，明确情感目标和实际约束

**操作方法**：
1. **情感锚点问题**：询问"游戏中什么时刻打动你？"，记录情感锚点
2. **品味档案分析**：询问玩家游戏偏好（喜欢/不喜欢的游戏），识别模式
3. **实际约束界定**：明确团队规模、时间预算、目标平台
4. **生成创意简报**：总结情感目标、品味档案、实际约束

**检查清单**：
- [ ] 情感锚点已记录
- [ ] 品味档案已分析
- [ ] 实际约束已界定
- [ ] 创意简报已生成

### 步骤2：概念生成 - 生成多个概念

**目标**：生成3个不同的游戏概念供选择

**操作方法**：
1. **动词优先设计**：从核心动词开始（如"探索"、"战斗"、"建造"）
2. **混搭方法**：结合两个意外元素（如"太空+农场"、"恐怖+解谜"）
3. **体验优先设计**：从情感目标倒推玩法
4. **概念呈现**：为每个概念生成标题、电梯演讲（1-2句话）、核心动词
5. 生成3个不同概念，确保多样性

**检查清单**：
- [ ] 3个概念已生成
- [ ] 每个概念有标题、电梯演讲、核心动词
- [ ] 概念多样性已确保

### 步骤3：用户选择概念

**目标**：让用户选择要深入发展的概念

**操作方法**：
1. 展示3个概念
2. 让用户选择一个概念
3. 记录选择理由

**检查清单**：
- [ ] 用户已选择概念
- [ ] 选择理由已记录

### 步骤4：核心循环设计

**目标**：定义游戏的核心循环结构

**操作方法**：
1. **30秒循环**：描述时刻到时刻的玩法（玩家在30秒内做什么）
2. **5分钟循环**：描述短期目标结构（玩家在5分钟内完成什么）
3. **会话循环**：描述完整游戏会话（玩家在1小时内体验什么）
4. **进度循环**：描述长期成长目标（玩家在数周内追求什么）
5. **玩家动机分析**：使用自我决定理论分析动机（自主性、胜任感、关联性）

**检查清单**：
- [ ] 30秒循环已定义
- [ ] 5分钟循环已定义
- [ ] 会话循环已定义
- [ ] 进度循环已定义
- [ ] 玩家动机已分析

### 步骤5：定义游戏支柱和边界

**目标**：建立设计决策框架

**操作方法**：
1. **定义3-5个游戏支柱**：描述游戏的核心设计原则
2. **定义反支柱**：明确游戏不做什么（避免功能蔓延）
3. **设计测试**：为每个支柱设计测试问题（"这个功能是否支持支柱X？"）
4. **平衡张力**：识别支柱间的张力，说明如何平衡

**检查清单**：
- [ ] 3-5个支柱已定义
- [ ] 反支柱已定义
- [ ] 设计测试已创建
- [ ] 张力已识别和平衡

### 步骤6：玩家类型验证

**目标**：验证概念对目标玩家的吸引力

**操作方法**：
1. **Bartle分类**：分析概念吸引哪些玩家类型（成就者、探索者、社交者、杀手）
2. **Quantic Foundry动机模型**：分析概念满足哪些动机（动作、社交、精通、成就、沉浸、创造）
3. **目标玩家定义**：描述目标玩家画像
4. **市场验证**：与市场趋势比较，评估差异化

**检查清单**：
- [ ] Bartle分类已分析
- [ ] Quantic Foundry动机已分析
- [ ] 目标玩家已定义
- [ ] 市场差异化已评估

### 步骤7：范围和可行性分析

**目标**：评估概念的可实现性

**操作方法**：
1. **引擎推荐**：推荐适合的引擎（Unity/Unreal/Godot等）
2. **美术管线**：描述美术制作流程
3. **内容范围**：估算内容需求（角色、关卡、资产）
4. **MVP定义**：定义最小可行产品范围
5. **最大风险**：识别最大的技术或设计风险

**检查清单**：
- [ ] 引擎已推荐
- [ ] 美术管线已描述
- [ ] 内容范围已估算
- [ ] MVP已定义
- [ ] 最大风险已识别

### 步骤8：输出结构化游戏概念文档

**目标**：生成完整的游戏概念文档

**操作方法**：
1. 输出概念概述（标题、电梯演讲、核心动词）
2. 输出核心循环（30秒、5分钟、会话、进度）
3. 输出游戏支柱和反支柱
4. 输出玩家类型分析
5. 输出范围定义和可行性分析

**检查清单**：
- [ ] 概念概述已输出
- [ ] 核心循环已输出
- [ ] 游戏支柱已输出
- [ ] 玩家类型已输出
- [ ] 可行性分析已输出

## 操作注意事项

### 创意探索期的无判断原则

1. **不判断创意**：在创意探索期不判断想法的好坏
2. **鼓励非常规想法**：鼓励意外和非常规的组合
3. **建立而非否定**：在他人想法上建立，而非否定

### 约束作为创意燃料

1. **约束激发创意**：约束（团队规模、时间、平台）激发创意
2. **限制产生最佳想法**：无限选择导致瘫痪，限制产生最佳想法
3. **时间框定阶段**：每个阶段时间框定，避免无限探索

### 用户驱动的协作

1. **用户做最终决策**：AI作为创意促进者，用户做最终选择
2. **逐步推进决策**：不一次性做所有决策，逐步推进
3. **尊重用户品味**：基于用户的品味档案生成概念

### 概念的多样性

1. **生成3个概念**：确保3个概念足够不同
2. **避免同质化**：避免3个概念本质相同
3. **覆盖不同方向**：确保概念覆盖不同玩法方向

- **creative-director**：创意方向和风格指导

## 常见错误

### 错误1：创意探索期过早判断

**问题**：在生成概念时立即判断想法好坏

**修正**：在创意探索期不判断，鼓励所有想法， judgment留到选择阶段

### 错误2：忽略实际约束

**问题**：生成不切实际的概念（超出团队规模或时间）

**修正**：始终基于实际约束（团队规模、时间、平台）生成概念

### 错误3：概念同质化

**问题**：3个概念本质相同（如3个都是射击游戏）

**修正**：确保3个概念足够不同，使用混搭方法结合意外元素

### 错误4：支柱定义模糊

**问题**：支柱如"好玩"或"有趣"，无法指导设计决策

**修正**：支柱应具体可测试（如"玩家每次会话都有明确进展"）

### 错误5：忽略反支柱

**问题**：只定义游戏做什么，不定义游戏不做什么

**修正**：必须定义反支柱，明确游戏不做什么，避免功能蔓延
