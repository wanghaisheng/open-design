# Functional Testing

执行游戏功能测试，验证游戏功能正确性、功能完整性和功能稳定性，确保游戏按预期运行。

## 适用场景

- 需要验证功能正确性
- 需要验证功能完整性
- 需要验证功能稳定性
- 需要执行回归测试
- 需要执行集成测试

## 操作步骤

### 步骤1：测试准备

**目标**：准备测试

**操作方法**：
1. 理解功能需求（读取GDD、需求文档）
2. 编写测试用例（正常场景、异常场景、边界场景、压力场景）
3. 准备测试环境（测试环境配置）
4. 准备测试数据（测试数据准备）

**检查清单**：
- [ ] 功能需求已理解
- [ ] 测试用例已编写
- [ ] 测试环境已准备
- [ ] 测试数据已准备

### 步骤2：测试执行

**目标**：执行测试

**操作方法**：
1. 执行测试用例（手动测试、自动化测试）
2. 记录测试结果（通过/失败）
3. 记录Bug（使用bug-report技能）
4. 收集测试数据（测试覆盖率、测试效果）

**检查清单**：
- [ ] 测试用例已执行
- [ ] 测试结果已记录
- [ ] Bug已记录
- [ ] 测试数据已收集

### 步骤3：测试分析

**目标**：分析测试结果

**操作方法**：
1. 分析测试结果（通过率、失败率）
2. 分析Bug分布（按模块、按严重性）
3. 分析测试覆盖率（功能覆盖率、场景覆盖率）
4. 分析测试效果（测试有效性）

**检查清单**：
- [ ] 测试结果已分析
- [ ] Bug分布已分析
- [ ] 测试覆盖率已分析
- [ ] 测试效果已分析

### 步骤4：测试报告

**目标**：编写测试报告

**操作方法**：
1. 编写测试报告（测试范围、测试用例、测试结果、Bug列表）
2. 提供测试建议（改进建议）
3. 跟踪Bug修复（跟踪修复进度）
4. 验证修复结果（回归测试）

**检查清单**：
- [ ] 测试报告已编写
- [ ] 测试建议已提供
- [ ] Bug修复已跟踪
- [ ] 修复结果已验证

## 输出格式

### 测试报告
- 测试范围

## 操作注意事项

### 准确性

1. **测试结果准确**：确保测试结果准确
2. **Bug描述准确**：确保Bug描述准确
3. **测试数据准确**：确保测试数据准确

### 完整性

1. **测试覆盖完整**：确保测试覆盖完整
2. **测试场景完整**：确保测试场景完整
3. **测试数据完整**：确保测试数据完整

### 可重复性

1. **测试可重复**：确保测试可重复
2. **测试结果可重复**：确保测试结果可重复
3. **Bug可重现**：确保Bug可重现

### 及时性

1. **及时发现Bug**：及时发现Bug
2. **及时报告Bug**：及时报告Bug
3. **及时验证修复**：及时验证修复

## 常见错误

### 错误1：不理解功能需求

**问题**：不读取GDD和需求文档，直接编写测试用例

**修正**：必须先理解功能需求，再编写测试用例

### 错误2：不准备测试环境

**问题**：不准备测试环境，直接执行测试

**修正**：必须先准备测试环境和测试数据

### 错误3：不记录Bug

**问题**：发现Bug不记录，只口头告知

**修正**：必须使用bug-report技能记录Bug

### 错误4：不分析测试覆盖率

**问题**：不分析测试覆盖率，遗漏测试场景

**修正**：必须分析测试覆盖率，确保测试覆盖完整

### 错误5：不验证修复结果

**问题**：不验证Bug修复结果

**修正**：必须验证修复结果，执行回归测试
