# Release Checklist

生成全面发布前验证检查清单，涵盖构建验证、认证要求、商店元数据和启动准备度。

## 适用场景

- 需要生成发布检查清单
- 需要验证代码库健康
- 需要检查构建验证
- 需要验证质量门控
- 需要检查启动准备度

## 操作步骤

### 步骤1：读取参数

**目标**：读取发布参数

**操作方法**：
1. 读取目标平台（pc、console、mobile或all）
2. 如果未指定，默认为all
3. 确定发布版本

**检查清单**：
- [ ] 目标平台已读取
- [ ] 发布版本已确定

### 步骤2：读取项目上下文

**目标**：读取项目上下文

**操作方法**：
1. 读取 CLAUDE.md
2. 理解项目上下文
3. 理解版本信息
4. 理解平台目标

**检查清单**：
- [ ] CLAUDE.md 已读取
- [ ] 项目上下文已理解
- [ ] 版本信息已理解
- [ ] 平台目标已理解

### 步骤3：读取当前里程碑

**目标**：读取当前里程碑

**操作方法**：
1. 读取 production/milestones/
2. 理解应包含的功能和内容
3. 确定发布范围

**检查清单**：
- [ ] 里程碑已读取
- [ ] 功能和内容已理解
- [ ] 发布范围已确定

### 步骤4：扫描代码库

**目标**：扫描代码库健康

**操作方法**：
1. 计数TODO注释
2. 计数FIXME注释
3. 计数HACK注释
4. 记录位置和严重性

**检查清单**：
- [ ] TODO计数已完成
- [ ] FIXME计数已完成
- [ ] HACK计数已完成
- [ ] 位置和严重性已记录

### 步骤5：检查测试结果

**目标**：检查测试结果

**操作方法**：
1. 检查测试输出目录
2. 检查CI日志
3. 验证测试通过

**检查清单**：
- [ ] 测试输出已检查
- [ ] CI日志已检查
- [ ] 测试通过已验证

### 步骤6：生成发布检查清单

**目标**：生成发布检查清单

**操作方法**：
1. 代码库健康（TODO、FIXME、HACK计数）
2. 构建验证（干净构建、无警告、资产包含）
3. 质量门控（零S1/S2错误、性能预算）
4. 内容完成（占位符替换、文本校对、本地化）
5. 平台特定部分（PC/Console/Mobile要求）
6. 商店和分发（元数据、截图、年龄评级）
7. 启动准备度（分析、崩溃报告、待命团队）

**检查清单**：
- [ ] 代码库健康已检查
- [ ] 构建验证已检查
- [ ] 质量门控已检查
- [ ] 内容完成已检查
- [ ] 平台要求已检查
- [ ] 商店分发已检查
- [ ] 启动准备度已检查

### 步骤7：保存检查清单

**目标**：保存检查清单

**操作方法**：
1. 保存到 production/releases/
2. 创建目录（如不存在）
3. 使用标准命名格式

**检查清单**：
- [ ] 检查清单已保存
- [ ] 目录已创建（如需要）
- [ ] 命名格式正确

### 步骤8：输出摘要

**目标**：输出检查清单摘要

**操作方法**：
1. 输出总检查清单项目
2. 输出已知阻塞数
3. 输出文件路径
4. 输出通过/不通过决策

**检查清单**：
- [ ] 总项目已输出
- [ ] 阻塞数已输出
- [ ] 文件路径已输出
- [ ] 决策已输出

## 输出格式

### 发布检查清单
```markdown
## 发布检查清单：[版本] -- [平台]
生成日期：[日期]

### 代码库健康
- TODO计数：[N]（如果多，列出前5个）
- FIXME计数：[N]（列出全部——这些是潜在阻塞）
- HACK计数：[N]（列出全部——这些需要审查）

### 构建验证
- [ ] 所有目标平台干净构建成功
- [ ] 无编译器警告（零警告策略）
- [ ] 所有资产包含且正确加载
- [ ] 构建大小在预算内（[目标大小]）
- [ ] 构建版本号正确设置（[版本]）
- [ ] 构建可从标记提交重现

### 质量门控
- [ ] 零S1（关键）错误
- [ ] 零S2（主要）错误——或制作人批准的文档化例外
- [ ] 所有关键路径功能测试并由QA签署
- [ ] 性能在预算内：
  - [ ] 最低规格硬件上达到目标FPS
  - [ ] 内存使用在预算内
  - [ ] 加载时间在预算内
  - [ ] 长时间游戏会话无内存泄漏
- [ ] 无前一构建回归
- [ ] 浸泡测试通过（4+小时连续游玩）

### 内容完成
- [ ] 所有占位符资产替换为最终版本
- [ ] 所有内容文件中的TODO/FIXME解决或记录
- [ ] 所有面向玩家的文本校对
- [ ] 所有文本本地化就绪（无硬编码字符串）
- [ ] 音频混音最终确定并批准
- [ ] 演职员表完整准确

### 平台要求：PC
- [ ] 最低和推荐规格验证并记录
- [ ] 键盘+鼠标控件完全功能
- [ ] 控制器支持测试（Xbox、PlayStation、通用）
- [ ] 分辨率缩放测试（1080p、1440p、4K、超宽）
- [ ] 窗口化、无边框和全屏模式工作
- [ ] 图形设置正确保存和加载
- [ ] Steam/Epic/GOG SDK集成并测试
- [ ] 成就功能
- [ ] 云保存功能
- [ ] Steam Deck兼容性验证（如目标）

### 商店/分发
- [ ] 商店页面元数据完成并校对
  - [ ] 简短描述
  - [ ] 长描述
  - [ ] 功能列表
  - [ ] 系统要求（PC）
- [ ] 截图更新并满足每平台分辨率要求
- [ ] 预告片更新
- [ ] 关键艺术和胶囊图像当前
- [ ] 年龄评级获得并配置：
  - [ ] ESRB
  - [ ] PEGI
  - [ ] 其他区域评级（如需要）
- [ ] 法律通知、EULA和隐私政策到位
- [ ] 第三方许可归属完成
- [ ] 所有区域定价配置

### 启动准备度
- [ ] 分析/遥证验证并接收数据
- [ ] 崩溃报告配置且仪表板可访问
- [ ] 第一天补丁准备并测试（如需要）
- [ ] 前72小时待命团队计划设置
- [ ] 社区启动公告起草
- [ ] 媒体/影响者密钥准备分发
- [ ] 支持团队简报已知问题和FAQ
- [ ] 回滚计划文档化（如发布后发现关键问题）

### 通过/不通过：[准备就绪 / 未准备就绪]

**理由：**
[准备度评估摘要。列出启动前必须解决的阻塞项目。如果未准备就绪，列出需要解决的具体项目和解决估计时间。]

**所需签署：**
- [ ] QA负责人
- [ ] 技术总监
- [ ] 制作人
- [ ] 创意总监
```

## 操作注意事项

### 全面性

1. **覆盖所有发布方面**：确保覆盖所有发布方面
2. **不遗漏关键检查项**：确保不遗漏关键检查项
3. **考虑所有平台要求**：确保考虑所有平台要求

### 严格性

1. **零容忍关键错误**：确保零容忍关键错误
2. **严格质量标准**：确保严格质量标准
3. **明确通过/不通过决策**：确保明确通过/不通过决策

### 可操作性

1. **每个检查项可验证**：确保每个检查项可验证
2. **明确验收标准**：确保明确验收标准
3. **明确负责人**：确保明确负责人

### 及时性

1. **发布前充分时间运行**：确保发布前充分时间运行
2. **及早发现问题**：确保及早发现问题
3. **留出修复时间**：确保留出修复时间

## 协作协议

### 向谁汇报

- **qa-lead**：质量门控结果
- **technical-lead**：构建验证结果
- **producer**：发布准备度

### 协调对象

- **platform-team**：平台要求
- **marketing-team**：商店元数据
- **support-team**：启动准备

## 常见错误

### 错误1：不扫描代码库

**问题**：不计数TODO、不计数FIXME

**修正**：必须计数TODO，FIXME，HACK，记录位置和严重性

### 错误2：不检查测试结果

**问题**：不检查测试输出、不验证测试通过

**修正**：必须检查测试输出，CI日志，验证测试通过

### 错误3：不生成完整检查清单

**问题**：不检查代码库健康、不检查构建验证

**修正**：必须检查代码库健康，构建验证，质量门控，内容完成

### 错误4：不检查平台要求

**问题**：不检查PC要求、不检查Console要求

**修正**：必须检查PC要求，Console要求，Mobile要求

### 错误5：不检查启动准备度

**问题**：不检查分析配置、不检查崩溃报告

**修正**：必须检查分析配置，崩溃报告，待命团队计划
