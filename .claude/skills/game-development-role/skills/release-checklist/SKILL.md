# Release Checklist

生成全面发布前验证检查清单，涵盖构建验证、认证要求、商店元数据和启动准备度。

## 发布检查清单要素

### 1. 代码库健康 (Codebase Health)
- TODO计数
- FIXME计数
- HACK计数
- 位置和严重性

### 2. 构建验证 (Build Verification)
- 干净构建成功
- 无编译器警告
- 所有资产包含且正确加载
- 构建大小在预算内
- 构建版本号正确设置
- 构建可从标记提交重现

### 3. 质量门控 (Quality Gates)
- 零S1（关键）错误
- 零S2（主要）错误
- 所有关键路径功能测试并由QA签署
- 性能在预算内
- 无回归
- 浸泡测试通过

### 4. 内容完成 (Content Complete)
- 所有占位符资产替换为最终版本
- 所有内容文件中的TODO/FIXME解决或记录
- 所有面向玩家的文本校对
- 所有文本本地化就绪
- 音频混音最终确定并批准
- 演职员表完整准确

### 5. 平台要求 (Platform Requirements)
- PC：键盘鼠标、控制器支持、分辨率缩放、Steam/Epic/GOG SDK
- Console：TRC/TCR/Lotcheck要求、控制器提示、挂起/恢复
- Mobile：应用商店指南、设备权限、隐私政策、触摸控制

### 6. 商店/分发 (Store/Distribution)
- 商店页面元数据完成
- 截图更新
- 预告片更新
- 关键艺术和胶囊图像
- 年龄评级获得并配置
- 法律通知、EULA和隐私政策到位
- 第三方许可归属完成
- 所有区域定价配置

### 7. 启动准备度 (Launch Readiness)
- 分析/遥证验证并接收数据
- 崩溃报告配置
- 第一天补丁准备并测试
- 待命团队计划设置
- 社区启动公告起草
- 媒体/影响者密钥准备
- 支持团队简报
- 回滚计划文档化

## 发布检查清单原则

### 1. 全面性
- 覆盖所有发布方面
- 不遗漏关键检查项
- 考虑所有平台要求

### 2. 严格性
- 零容忍关键错误
- 严格质量标准
- 明确通过/不通过决策

### 3. 可操作性
- 每个检查项可验证
- 明确验收标准
- 明确负责人

### 4. 及时性
- 发布前充分时间运行
- 及早发现问题
- 留出修复时间

## 发布检查清单流程

### 1. 读取参数
- 目标平台（pc、console、mobile或all）
- 如果未指定，默认为all

### 2. 读取项目上下文
- 读取 CLAUDE.md
- 理解项目上下文
- 理解版本信息
- 理解平台目标

### 3. 读取当前里程碑
- 读取 production/milestones/
- 理解应包含的功能和内容

### 4. 扫描代码库
- 计数TODO注释
- 计数FIXME注释
- 计数HACK注释
- 记录位置和严重性

### 5. 检查测试结果
- 检查测试输出目录
- 检查CI日志
- 验证测试通过

### 6. 生成发布检查清单
- 代码库健康
- 构建验证
- 质量门控
- 内容完成
- 平台特定部分
- 商店和分发
- 启动准备度

### 7. 保存检查清单
- 保存到 production/releases/
- 创建目录（如不存在）

### 8. 输出摘要
- 总检查清单项目
- 已知阻塞数
- 文件路径

## 输出格式

### 发布检查清单
```markdown
## 发布检查清单：[版本] -- [平台]
生成日期：[日期]

### 代码库健康
- TODO计数：[N]（如果多，列出前5个）
- FIXME计数：[N]（列出全部——这些是潜在阻塞）
- HACK计数：[N]（列出全部——这些需要审查）

### 构建验证
- [ ] 所有目标平台干净构建成功
- [ ] 无编译器警告（零警告策略）
- [ ] 所有资产包含且正确加载
- [ ] 构建大小在预算内（[目标大小]）
- [ ] 构建版本号正确设置（[版本]）
- [ ] 构建可从标记提交重现

### 质量门控
- [ ] 零S1（关键）错误
- [ ] 零S2（主要）错误——或制作人批准的文档化例外
- [ ] 所有关键路径功能测试并由QA签署
- [ ] 性能在预算内：
  - [ ] 最低规格硬件上达到目标FPS
  - [ ] 内存使用在预算内
  - [ ] 加载时间在预算内
  - [ ] 长时间游戏会话无内存泄漏
- [ ] 无前一构建回归
- [ ] 浸泡测试通过（4+小时连续游玩）

### 内容完成
- [ ] 所有占位符资产替换为最终版本
- [ ] 所有内容文件中的TODO/FIXME解决或记录
- [ ] 所有面向玩家的文本校对
- [ ] 所有文本本地化就绪（无硬编码字符串）
- [ ] 音频混音最终确定并批准
- [ ] 演职员表完整准确

### 平台要求：PC
- [ ] 最低和推荐规格验证并记录
- [ ] 键盘+鼠标控件完全功能
- [ ] 控制器支持测试（Xbox、PlayStation、通用）
- [ ] 分辨率缩放测试（1080p、1440p、4K、超宽）
- [ ] 窗口化、无边框和全屏模式工作
- [ ] 图形设置正确保存和加载
- [ ] Steam/Epic/GOG SDK集成并测试
- [ ] 成就功能
- [ ] 云保存功能
- [ ] Steam Deck兼容性验证（如目标）

### 商店/分发
- [ ] 商店页面元数据完成并校对
  - [ ] 简短描述
  - [ ] 长描述
  - [ ] 功能列表
  - [ ] 系统要求（PC）
- [ ] 截图更新并满足每平台分辨率要求
- [ ] 预告片更新
- [ ] 关键艺术和胶囊图像当前
- [ ] 年龄评级获得并配置：
  - [ ] ESRB
  - [ ] PEGI
  - [ ] 其他区域评级（如需要）
- [ ] 法律通知、EULA和隐私政策到位
- [ ] 第三方许可归属完成
- [ ] 所有区域定价配置

### 启动准备度
- [ ] 分析/遥证验证并接收数据
- [ ] 崩溃报告配置且仪表板可访问
- [ ] 第一天补丁准备并测试（如需要）
- [ ] 前72小时待命团队计划设置
- [ ] 社区启动公告起草
- [ ] 媒体/影响者密钥准备分发
- [ ] 支持团队简报已知问题和FAQ
- [ ] 回滚计划文档化（如发布后发现关键问题）

### 通过/不通过：[准备就绪 / 未准备就绪]

**理由：**
[准备度评估摘要。列出启动前必须解决的阻塞项目。如果未准备就绪，列出需要解决的具体项目和解决估计时间。]

**所需签署：**
- [ ] QA负责人
- [ ] 技术总监
- [ ] 制作人
- [ ] 创意总监
```
