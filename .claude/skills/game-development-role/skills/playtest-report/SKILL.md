# Playtest Report

生成结构化玩测报告模板或分析现有玩测笔记为结构化格式。用于标准化玩测反馈收集和分析。

## 报告要素

### 1. 会话信息 (Session Info)
- 日期：玩测日期
- 版本：版本/提交
- 时长：游戏时间
- 测试者：姓名/ID
- 平台：PC/主机/移动
- 输入方式：键鼠/手柄/触摸
- 会话类型：首次/返回/目标测试

### 2. 测试重点 (Test Focus)
- 测试功能：测试的具体功能或流程
- 测试目标：测试目标
- 测试范围：测试范围

### 3. 第一印象 (First Impressions)
- 理解目标：是否理解目标
- 理解控制：是否理解控制
- 情感反应：投入/困惑/无聊/沮丧/兴奋
- 观察记录：观察记录

### 4. 游戏流程 (Gameplay Flow)
- 效果好的地方：观察到的有效设计
- 痛点：问题和困难点
- 困惑点：玩家困惑的地方
- 惊喜时刻：让玩家惊喜或愉悦的时刻

### 5. 遇到的Bug (Bugs Encountered)
- 描述：Bug描述
- 严重性：高/中/低
- 可重现性：是否可重现

### 6. 特定功能反馈 (Feature-Specific Feedback)
- 功能理解：是否理解功能目的
- 参与度：是否发现功能有趣
- 建议：测试者建议

### 7. 定量数据 (Quantitative Data)
- 死亡次数：死亡次数和位置
- 区域时间：每个区域的时间
- 物品使用：使用什么和何时使用
- 发现vs遗漏：发现vs遗漏的功能

### 8. 整体评估 (Overall Assessment)
- 愿意再次玩：是/否/可能
- 难度：太简单/正好/太难
- 节奏：太慢/好/太快
- 会话长度偏好：更短/好/更长

- **输入方式**: [键鼠/手柄/触摸]

## 测试重点

## 第一印象（前5分钟）

- **记录**: [观察]

## 游戏流程
### 效果好的地方
- [观察1]
- [观察2]

### 痛点
- [问题1 -- 严重性: 高/中/低]
- [问题2 -- 严重性: 高/中/低]

### 困惑点
- [玩家困惑的地方和原因]

### 惊喜时刻
- [让玩家惊喜或愉悦的地方]

## 遇到的Bug
| # | 描述 | 严重性 | 可重现 |
|---|-------------|----------|-------------|

## 特定功能反馈
### [功能1]
- **理解目的？** [是/否]
- **发现有趣？** [是/否]
- **建议**: [测试者建议]

## 定量数据（如果可用）
- **死亡**: [次数和位置]
- **区域时间**: [细分]
- **物品使用**: [什么和何时]
- **发现vs遗漏**: [列表]

## 整体评估
- **愿意再次玩？** [是/否/可能]
- **难度**: [太简单/正好/太难]
- **节奏**: [太慢/好/太快]

## 本次会话前3优先级

2. [第二优先级]
3. [第三优先级]

## 分析流程

### 1. 新建模式
- 生成报告模板
- 提供填写指南
- 保存模板文件

### 2. 分析模式
- 读取原始笔记
- 交叉参考设计文档
- 填充结构化发现
- 标记冲突观察

- 笔记软件：Notion, Obsidian

## 分析原则

### 1. 结构化
- 标准化格式
- 结构化数据
- 易于分析

### 2. 可追溯
- 记录来源
- 记录上下文
- 记录时间

### 3. 可操作
- 提供可操作反馈
- 提供具体建议
- 提供优先级

### 4. 一致性
- 一致标准
- 一致格式
- 一致评估
