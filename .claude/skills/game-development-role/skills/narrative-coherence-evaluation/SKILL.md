# Narrative Coherence Evaluation

评估游戏叙事连贯性，包括时间逻辑、角色发展、世界观一致性和对话质量，使用量化评分体系和专业评审流程，确保叙事体验的完整性和沉浸感。

## 适用场景

- 叙事设计完成后需要质量评估
- 剧情出现逻辑漏洞或角色行为不一致
- 需要量化评估叙事质量，为迭代提供依据
- 玩家反馈剧情理解困难或情感共鸣不足
- 需要建立持续改进的叙事质量监控机制

## 操作步骤

### 步骤1：定义连贯性维度

**目标**：建立评估维度和权重体系

**操作方法**：
1. 定义五大评估维度：
   - 时间逻辑（30%）：时间线和因果关系的合理性
   - 角色发展（25%）：角色弧线和动机的一致性
   - 世界观（20%）：世界设定的统一性和逻辑性
   - 对话质量（15%）：对话的自然性和信息传递
   - 情感共鸣（10%）：情感曲线的流畅性和感染力
2. 为每个维度定义子维度
3. 确定各维度权重（可根据项目调整）

**检查清单**：
- [ ] 评估维度已定义
- [ ] 子维度已明确
- [ ] 权重分配合理

### 步骤2：时间连贯性检查

**目标**：验证时间线和因果关系的合理性

**操作方法**：
1. 时间线一致性检查：
   - 事件发生的时间逻辑
   - 角色年龄和时代背景的一致性
   - 时间间隔的合理性
2. 因果关系检查：
   - 前后事件的因果链条
   - 每个结果都有明确的原因
   - 避免机械降神式的解决方案
3. 时代背景检查：
   - 历史设定与剧情的统一
   - 科技/魔法水平的时代一致性

**检查清单**：
- [ ] 时间线逻辑自洽
- [ ] 因果关系清晰
- [ ] 时代背景一致

### 步骤3：角色连贯性检查

**目标**：验证角色行为与性格的一致性

**操作方法**：
1. 性格一致性检查：
   - 角色行为与性格的匹配
   - 角色对话风格的统一性
   - 角色知识水平的一致性
2. 成长轨迹检查：
   - 角色发展的合理渐进
   - 角色动机变化的自然性
   - 角色缺陷到蜕变的逻辑性
3. 关系发展检查：
   - 角色关系变化的自然性
   - 角色互动的合理性
   - 角色情感状态与对话的匹配

**检查清单**：
- [ ] 角色性格一致
- [ ] 成长轨迹合理
- [ ] 关系发展自然

### 步骤4：世界观连贯性检查

**目标**：验证世界设定的统一性和逻辑性

**操作方法**：
1. 设定一致性检查：
   - 魔法/科技规则的一致应用
   - 魔法/科技限制的合理设定
   - 魔法/科技代价的平衡设计
   - 社会结构的合理性
   - 文化传统的统一性
   - 价值观念的逻辑性
2. 逻辑自洽检查：
   - 世界内部逻辑的无矛盾
   - 设定与剧情的统一
3. 细节呼应检查：
   - 前期伏笔的合理回收
   - 前后剧情细节的呼应

**检查清单**：
- [ ] 世界设定一致
- [ ] 内部逻辑自洽
- [ ] 细节呼应到位

### 步骤5：对话连贯性检查

**目标**：验证对话的自然性和信息传递效果

**操作方法**：
1. 对话逻辑检查：
   - 对话触发条件的合理性
   - 信息传递的逐步展开逻辑
   - 分支选择的逻辑合理性
2. 对话一致性检查：
   - 语言风格的统一性
   - 情感状态与对话的匹配
3. 分支对话检查：
   - 所有分支都符合角色性格
   - 分支结果有合理的逻辑解释
   - 重要信息在所有分支中都能获得

**检查清单**：
- [ ] 对话逻辑合理
- [ ] 对话风格统一
- [ ] 分支设计合理

### 步骤6：情感共鸣评估

**目标**：评估情感曲线的流畅性和感染力

**操作方法**：
1. 情感曲线检查：
   - 剧情情感起伏的自然流畅
   - 情感高潮的时机合理性
   - 情感低谷的处理恰当性
2. 情感共鸣测试：
   - 玩家情感投入程度
   - 关键场景的情感冲击力
   - 角色命运的情感共鸣

**检查清单**：
- [ ] 情感曲线流畅
- [ ] 情感冲击到位
- [ ] 共鸣感强

### 步骤7：量化评分

**目标**：使用标准化评分体系输出评估结果

**操作方法**：
1. 按维度评分（0-100分）
2. 按权重计算总分
3. 确定评分等级：
   - 90-100分（优秀）：逻辑完全自洽，无明显漏洞
   - 80-89分（良好）：逻辑基本合理，存在细微优化空间
   - 70-79分（合格）：逻辑存在小问题，需要调整
   - 60-69分（需改进）：逻辑存在明显问题
   - <60分（不合格）：逻辑严重混乱
4. 生成评分报告

**检查清单**：
- [ ] 各维度已评分
- [ ] 总分已计算
- [ ] 评分等级已确定
- [ ] 评分报告已生成

### 步骤8：设计改进方案

**目标**：根据评估结果提出具体改进建议

**操作方法**：
1. 识别主要问题
2. 分析问题根源
3. 提出改进方案
4. 设定改进优先级
5. 制定改进时间表

**检查清单**：
- [ ] 问题已识别
- [ ] 根源已分析
- [ ] 方案已提出
- [ ] 优先级已设定

## 操作注意事项

### 评估客观性

1. 评估应基于明确的评分标准，避免主观臆断
2. 使用量化指标，确保评估可重复
3. 多轮评审，减少个人偏见

### 改进持续性

1. 建立迭代优化记录表
2. 定期进行叙事连贯性评估
3. 监控玩家剧情完成率和选择分布
4. 收集社区讨论和客服反馈

### 专业评审

1. 第一轮：策划内部评审（剧情逻辑、角色一致性、世界观统一性）
2. 第二轮：跨部门评审（美术风格一致性、程序实现可行性、音效配合合理性）
3. 第三轮：外部评审（资深策划专家评审、目标用户测试、文学顾问评估）

### 协调对象

- narrative-designer：评估结果与叙事设计对接
- writer：评估结果与内容写作对接
- qa-tester：评估结果与测试验证对接

## 常见错误

### 错误1：评估标准不明确

**问题**：评分标准模糊，导致评估结果主观性强

**修正**：在步骤1明确定义每个维度的评分标准和等级定义

### 错误2：忽略玩家反馈

**问题**：只从专业角度评估，忽略玩家实际体验

**修正**：在步骤7加入玩家测试，收集理解度、沉浸感、合理性、满意度数据

### 错误3：改进方案不具体

**问题**：只指出问题，不提供具体改进方案

**修正**：在步骤8为每个问题提供具体的改进方案和优先级
