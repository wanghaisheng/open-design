# Tech Debt

跟踪、分类和优先级化代码库中的技术债务。扫描债务指示器，维护债务登记册，并建议偿还计划。

## 适用场景

- 需要扫描技术债务
- 需要添加技术债务条目
- 需要优先级化技术债务
- 需要生成技术债务报告
- 需要建议偿还计划

## 操作步骤

### 步骤1：扫描

**目标**：扫描技术债务

**操作方法**：
1. 扫描代码库的债务指示器（TODO注释、FIXME注释、HACK注释、@deprecated标记、重复代码块、大文件、大函数、高圈复杂度）
2. 分类每个发现（架构债务、代码质量债务、测试债务、文档债务、依赖债务、性能债务）
3. 更新债务登记册

**检查清单**：
- [ ] 债务指示器已扫描
- [ ] 发现已分类
- [ ] 债务登记册已更新

### 步骤2：添加

**目标**：添加技术债务条目

**操作方法**：
1. 提示（描述、类别、受影响文件、估计修复工作量、不修复的影响）
2. 追加到债务登记册

**检查清单**：
- [ ] 描述已提供
- [ ] 类别已指定
- [ ] 受影响文件已列出
- [ ] 修复工作量已估计
- [ ] 影响已说明
- [ ] 已追加到债务登记册

### 步骤3：优先级

**目标**：优先级化技术债务

**操作方法**：
1. 读取债务登记册
2. 按评分排序每个项目（评分公式：(不修复的影响 * 遇到频率) / 修复工作量）
3. 推荐下一个冲刺包含的项目

**检查清单**：
- [ ] 债务登记册已读取
- [ ] 项目已按评分排序
- [ ] 下一个冲刺项目已推荐

### 步骤4：报告

**目标**：生成技术债务报告

**操作方法**：
1. 读取债务登记册
2. 生成摘要统计（总项目、估计总工作量、自上次报告添加的项目、自上次报告解决的项目、趋势方向）
3. 标记超过3个冲刺的项目
4. 输出报告

**检查清单**：
- [ ] 债务登记册已读取
- [ ] 摘要统计已生成
- [ ] 超期项目已标记
- [ ] 报告已输出

## 操作注意事项

### 意识决策

1. **技术债务是工具**：技术债务不是固有的坏事——它是工具
2. **登记册跟踪意识决策**：登记册跟踪意识决策
3. **解释为什么被接受**：每个债务条目必须解释为什么被接受

### 优先级管理

1. **基于影响和频率评分**：基于影响和频率评分
2. **考虑修复工作量**：考虑修复工作量
3. **平衡新功能与债务偿还**：平衡新功能与债务偿还

### 持续监控

1. **每个冲刺至少扫描一次**：每个冲刺至少扫描一次以捕获新债务
2. **定期审查债务登记册**：定期审查债务登记册
3. **跟踪债务趋势**：跟踪债务趋势

### 及时处理

1. **超过3个冲刺无行动的项目应修复**：超过3个冲刺无行动的项目应修复或有意识地接受
2. **避免债务积累**：避免债务积累到无法管理的程度

## 常见错误

### 错误1：不扫描技术债务

**问题**：不扫描代码库的债务指示器

**修正**：必须扫描代码库的债务指示器，分类每个发现，更新债务登记册

### 错误2：不添加技术债务条目

**问题**：不提供描述、类别、受影响文件、修复工作量、影响

**修正**：必须提供完整提示，追加到债务登记册

### 错误3：不优先级化技术债务

**问题**：不按评分排序项目，不推荐下一个冲刺项目

**修正**：必须按评分排序项目，推荐下一个冲刺包含的项目

### 错误4：不生成技术债务报告

**问题**：不生成摘要统计，不标记超期项目

**修正**：必须生成摘要统计，标记超过3个冲刺的项目，输出报告

### 错误5：不定期扫描

**问题**：不每个冲刺至少扫描一次

**修正**：必须每个冲刺至少扫描一次以捕获新债务
