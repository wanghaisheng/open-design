name: core-experience-purification
description: 基于"核心体验提纯法"优化慢热型游戏的核心体验，移除操作摩擦、认知负荷、节奏赘余，提升智识和情感体验纯度。
tags: [游戏, 设计, 体验, 提纯]

---

## 适用场景

- 优化慢热型游戏的核心体验
- 移除游戏中的"杂质"（操作摩擦、认知负荷、节奏赘余）
- 提升游戏的智识和情感体验纯度
- 适用于策略游戏、解谜游戏、叙事游戏
- 设计核心心流和体验升华

---

## 操作步骤

### 步骤1：读取PRD和游戏类型

**目标**：读取游戏PRD，理解游戏类型和核心体验目标

**操作方法**：
1. 读取游戏PRD文档
2. 提取游戏类型（策略、解谜、叙事等）
3. 提取游戏愿景和目标
4. 提取核心玩法和体验目标
5. 分析游戏是否属于慢热型游戏
6. 提取当前设计的优势和问题

**检查清单**：
- [ ] PRD文档已读取
- [ ] 游戏类型已提取
- [ ] 游戏愿景已提取
- [ ] 核心玩法已提取
- [ ] 慢热型特征已识别
- [ ] 优势问题已分析

---

### 步骤2：识别核心心流

**目标**：精准提炼玩家在游戏中追求的最终"心流状态"

**操作方法**：
1. 分析原始体验目标
2. 识别核心心流类型：
   - **战略心流**：运筹帷幄、史诗成就感（如《文明》）
   - **顿悟心流**：灵光一闪、智识高峰体验（如《见证者》）
   - **共情心流**：情感共鸣、深入思考（如《艾迪芬奇的记忆》）
3. 提炼核心心流状态
4. 验证心流状态的准确性

**检查清单**：
- [ ] 原始体验目标已分析
- [ ] 心流类型已识别
- [ ] 核心心流已提炼
- [ ] 心流准确性已验证

---

### 步骤3：设计提纯策略 - 移除操作摩擦

**目标**：移除繁琐、重复的微操或糟糕的UI，让玩家专注于核心思考

**操作方法**：
1. 分析当前操作摩擦
   - 无意义的微操（反复调整、重复指令）
   - 糟糕的UI/UX设计
   - 繁琐的操作流程
2. 提出移除操作摩擦的具体方案
   - 自动化系统（智能总督、一键应用模板）
   - UI/UX优化（清晰、优雅的界面）
   - 操作简化（减少点击、优化流程）
3. 评估方案对核心心流的影响

**检查清单**：
- [ ] 操作摩擦已分析
- [ ] 自动化方案已提出
- [ ] UI/UX优化已提出
- [ ] 操作简化已提出
- [ ] 方案影响已评估

---

### 步骤4：设计提纯策略 - 移除认知负荷

**目标**：让规则更清晰、信息更直观，消除不必要的困惑和挫败感

**操作方法**：
1. 分析当前认知负荷
   - 模糊的信息（UI/UX设计糟糕）
   - 规则的歧义（规则不清晰）
   - 永久卡关的挫败感（单线性设计）
2. 提出移除认知负荷的具体方案
   - 信息反馈优化（清晰、直观的信息展示）
   - 规则澄清（明确的规则说明）
   - 非线性设计（提供多条路径）
3. 评估方案对核心心流的影响

**检查清单**：
- [ ] 认知负荷已分析
- [ ] 信息反馈优化已提出
- [ ] 规则澄清已提出
- [ ] 非线性设计已提出
- [ ] 方案影响已评估

---

### 步骤5：设计提纯策略 - 移除节奏赘余

**目标**：砍掉或转化"垃圾时间"（跑腿、等待、空洞内容）

**操作方法**：
1. 分析当前节奏赘余
   - 无效的奔波（漫无目的寻找）
   - 垃圾时间（无事可做的等待）
   - 冗长空洞的对话（无信息价值）
   - 出戏的游戏性（与主题无关的玩法）
   - 无意义的跑腿（拿A给B式任务）
2. 提出移除节奏赘余的具体方案
   - 探索效率优化（逻辑分布、引导清晰）
   - 等待时间优化（减少AI思考时间）
   - 对话优化（信息密度提升）
   - 玩法与叙事融合（玩法即叙事）
3. 评估方案对核心心流的影响

**检查清单**：
- [ ] 节奏赘余已分析
- [ ] 探索效率优化已提出
- [ ] 等待时间优化已提出
- [ ] 对话优化已提出
- [ ] 玩法叙事融合已提出
- [ ] 方案影响已评估

---

### 步骤6：构想体验升华

**目标**：描述提纯策略执行后，游戏体验将如何升华

**操作方法**：
1. 描述提纯后的体验
   - 玩家将获得怎样深刻、纯粹的感受
   - 体验的持久性和回味性
2. 定义体验标签
   - 例如："互动思想实验"、"可玩的编年史"、"情感建筑学"
3. 分析体验的独特性和创新性

**检查清单**：
- [ ] 提纯体验已描述
- [ ] 体验标签已定义
- [ ] 体验独特性已分析

---

### 步骤7：设计节奏与留存

**目标**：设计张弛有度的节奏，通过长期目标或内在驱动力驱动玩家持续投入

**操作方法**：
1. 设计核心节奏循环
   - 描述游戏的核心节奏
   - 例如："学习-应用-变体-综合"的解谜循环
   - 例如："挑战-喘息-紧张-放松"的叙事循环
2. 设计长期内在驱动力
   - 提出驱动玩家长期游玩的内在动机
   - 例如："揭开世界终极秘密的好奇心"
   - 例如："亲手塑造一个文明的成就感"
3. 评估节奏和留存设计

**检查清单**：
- [ ] 核心节奏循环已设计
- [ ] 长期内在驱动力已设计
- [ ] 节奏留存设计已评估

## 操作注意事项

### 核心原则

1. **重新定义"浓缩"的对象**：对于慢热型游戏，浓缩"核心体验的纯度与密度"，而非"动作的频率"或"感官的刺激"
2. **移除一切阻碍核心心流的"杂质"**：让每一次决策、每一次发现、每一次情感共鸣都更加深刻和有意义
3. **追求智力与情感体验的"纯"与"深"**：不是追求感官刺激的"爽"，而是追求深刻、持久的满足感

### 不同游戏类型的提纯重点

**策略游戏（如《文明》）：**
- 核心体验：运筹帷幄、史诗成就感
- 提纯重点：浓缩决策质量、浓缩信息反馈、浓缩"史诗感"

**解谜游戏（如《见证者》）：**
- 核心体验：灵光一闪、智识高峰体验
- 提纯重点：浓缩"顿悟密度"、浓缩探索效率、浓缩学习过程

**叙事游戏（如《艾迪芬奇的记忆》）：**
- 核心体验：情感共鸣、深入思考
- 提纯重点：浓缩"叙事密度"、浓缩"玩法即叙事"、浓缩"有效信息"

### 提纯策略实施原则

1. **自动化优于手动**：通过自动化系统消除重复性微操
2. **清晰优于复杂**：设计清晰、优雅的UI，让信息一目了然
3. **有意义优于无意义**：将"垃圾时间"转化为有意义的体验
4. **融合优于分离**：将玩法与叙事深度融合，避免出戏

### 节奏设计原则

1. **张弛有度**：设计挑战与喘息、紧张与放松的交替
2. **内在驱动**：通过内在驱动力（好奇心、成就感）而非简单奖励驱动玩家
3. **长期目标**：设计能驱动玩家长期游玩的长期目标
4. **渐进式揭示**：逐步揭示游戏深度，保持玩家好奇心

## 常见错误

### 错误1：错误识别核心心流

**问题**：未能准确识别游戏的核心心流类型

**修正**：深入分析游戏的核心体验，确定是战略心流、顿悟心流还是共情心流

### 错误2：过度简化操作

**问题**：过度简化操作，导致游戏失去深度和挑战性

**修正**：平衡操作简化和游戏深度，确保简化后仍有足够的挑战性

### 错误3：忽视游戏类型差异

**问题**：用同一套提纯策略处理不同类型的游戏

**修正**：根据游戏类型（策略、解谜、叙事）调整提纯策略的重点

### 错奏4：移除必要内容

**问题**：在移除"杂质"时，误删了必要的游戏内容

**修正**：仔细区分"杂质"和必要内容，确保提纯后游戏仍完整

### 错误5：节奏设计不合理

**问题**：节奏设计过于紧凑或过于松散，影响玩家体验

**修正**：设计张弛有度的节奏，确保玩家有适当的喘息和挑战

### 错误6：忽视标杆案例

**问题**：不参考成功的慢热型游戏，导致提纯方案缺乏依据

**修正**：参考成功的慢热型游戏（如《陷阵之志》、《艾迪芬奇的记忆》、《极乐迪斯科》），学习其提纯策略
