# Design Iteration Tracker

跟踪变化内容、变化原因和结果。防止循环设计和构建机构记忆。

## 适用场景

- 需要记录设计决策及理由
- 需要检测提议变更是否移向先前拒绝的状态
- 需要跟踪跨系统的变更影响链
- 需要维护关键设计参数的版本历史
- 需要构建可搜索的设计演变历史

## 操作步骤

### 步骤1：记录设计变更

**目标**：记录设计变更

**操作方法**：
1. 记录当前状态（它现在是什么）
2. 记录提议变更（它将变成什么）
3. 陈述理由（为什么此变更）
4. 检查还原警告（我们之前尝试过吗）
5. 映射影响链（还有什么受影响）
6. 实现变更
7. 更新参数历史（如适用）
8. 标记受影响技能到一致性引擎

**检查清单**：
- [ ] 当前状态已记录
- [ ] 提议变更已记录
- [ ] 理由已陈述
- [ ] 还原警告已检查
- [ ] 影响链已映射
- [ ] 变更已实现
- [ ] 参数历史已更新
- [ ] 受影响技能已标记

### 步骤2：运行迭代审查

**目标**：运行迭代审查

**操作方法**：
1. 收集自上次审查以来的所有变更
2. 按类型和驱动分类
3. 识别模式（我们是否不断调整相同内容）
4. 检查循环变更（A→B→A模式）
5. 评估整体设计方向（游戏收敛还是发散）
6. 设置下次迭代优先级

**检查清单**：
- [ ] 所有变更已收集
- [ ] 已按类型和驱动分类
- [ ] 模式已识别
- [ ] 循环变更已检查
- [ ] 设计方向已评估
- [ ] 下次迭代优先级已设置

### 步骤3：生成迭代摘要报告

**目标**：生成迭代摘要报告

**操作方法**：
1. 记录迭代期间（日期范围或玩测范围）
2. 记录总变更（计数）
3. 列出主要变更
4. 记录还原（任何已撤销的变更）
5. 记录开放问题（什么仍未解决）
6. 评估设计方向（游戏方向如何转移）
7. 设置下一步优先级（下次迭代应关注什么）

**检查清单**：
- [ ] 迭代期间已记录
- [ ] 总变更已记录
- [ ] 主要变更已列出
- [ ] 还原已记录
- [ ] 开放问题已记录
- [ ] 设计方向已评估
- [ ] 下一步优先级已设置

## 输出

此技能产生：
1. 决策日志——每个变更带理由、数据和影响
2. 还原警告——循环设计模式检测
3. 影响链跟踪——变更的级联效应
4. 参数版本历史——关键设计值的演变
5. 迭代摘要报告——定期设计演变概述

这些输出馈入：
- 所有技能——任何设计元素都可以改变
- 平衡技能——平衡是最常迭代的元素
- GDD技能——GDD必须反映当前状态
- 一致性技能——馈送引擎的变更感知

## 操作注意事项

### 跟踪什么

**始终跟踪：**
- 核心机制变更（规则、值、系统）
- 支柱或愿景变更（最高影响变更）
- 玩测驱动变更（链接玩测数据）
- 还原（为什么撤销某些内容）

**重要时跟踪：**
- 内容变更（新关卡、遭遇、角色）
- 平衡调整（带理由的值调整）
- 美学变更（艺术方向转移、音频变更）

**不跟踪：**
- 拼写修复和格式
- 进行中状态（仅决策时跟踪）
- 未达到共享设计的个别开发者实验

### 如何分类

**变更类型：**
- 添加——设计中的新元素
- 移除——从设计中删除元素
- 修改——现有元素改变
- 还原——先前变更撤销
- 重新平衡——现有值的定量调整
- 重组——组织变更（系统重组，非修改）

**变更驱动：**
- 玩测数据——来自测试的可观察证据
- 玩家反馈——测试者报告的体验
- 设计师洞察——专业判断
- 技术约束——实现强制变更
- 范围压力——时间/预算强制削减
- 市场转移——外部因素改变优先级

## 协作协议

### 向谁汇报

- **lead-designer**：设计变更和迭代结果
- **creative-director**：设计方向和模式
- **producer**：迭代进度和优先级

### 协调对象

- **game-designer**：设计变更实施
- **qa-lead**：玩测数据链接
- **technical-lead**：技术约束变更

## 常见错误

### 错误1：未记录变更

**问题**：变更后不记录，无人知道为什么

**修正**：必须记录所有有意义的设计变更

### 错误2：不检查还原警告

**问题**：不检查提议变更是否移向先前拒绝的状态

**修正**：必须检查还原警告，评估"此次不同"条件

### 错误3：不映射影响链

**问题**：不跟踪跨系统的变更影响链

**修正**：必须映射影响链，识别直接和间接影响

### 错误4：不更新参数历史

**问题**：不维护关键设计参数的版本历史

**修正**：必须更新参数历史，避免"调整跑步机"

### 错误5：不运行迭代审查

**问题**：不定期运行迭代审查，无法识别模式

**修正**：必须定期运行迭代审查，识别循环变更
