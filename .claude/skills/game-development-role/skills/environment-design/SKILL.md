# Environment Design

设计游戏环境和景观，创造沉浸式的游戏场景和视觉效果。

## 适用场景

- 从 0 到 1 设计游戏环境（场景布局、视觉风格、氛围营造、玩法支持）
- 需要把环境拆解为可实现的空间布局、光照材质、互动元素与性能优化方案
- 环境出现玩家迷失、氛围不足、性能问题、玩法冲突等问题，需要迭代优化
- 需要与关卡设计/美术/程序协作，明确环境与玩法/美术/性能的结合点

## 操作步骤

### 步骤1：把环境目标写成"可验收"的一句话

**目标**：明确环境服务什么体验目标、成功标准是什么

**操作方法**：
1. 写环境主题与一句话目标（面向玩家情感体验与玩法需求）
2. 列出环境输入/输出（玩家行为、视觉反馈、探索路径）
3. 写 3-5 条成功标准（例如：玩家能在 30 秒内理解环境方向；环境氛围与玩法匹配）
4. 写 3-5 条失败信号（例如：玩家在环境中迷失；环境与玩法冲突）

**检查清单**：
- [ ] 有清晰边界：环境"包含什么/不包含什么"
- [ ] 有成功标准：可观测、可测试
- [ ] 有失败信号：能快速触发调整

### 步骤2：用"场景布局-视觉风格-氛围营造-玩法支持"四块拆解环境

**目标**：把环境拆成可实现的设计单元，避免只写概念

**操作方法**：
1. **场景布局**：列空间布局/视觉层次/玩家路径/隐藏区域（每条写尺寸、连通性、引导方式）
2. **视觉风格**：列色彩方案/光照设计/材质表现/细节丰富（每条写色调、光照类型、材质参数、细节密度）
3. **氛围营造**：列氛围基调/情绪表达/故事融入/环境叙事（每条写基调、情绪触发点、故事线索、叙事方式）
4. **玩法支持**：列遭遇设计/掩体设计/互动元素/隐藏物品（每条写触发条件、交互方式、奖励内容）

**检查清单**：
- [ ] 每个空间布局都有尺寸与连通性
- [ ] 每个视觉风格都有具体参数
- [ ] 每个氛围营造都有触发点
- [ ] 每个玩法支持都有交互方式

### 步骤3：画环境布局图并标注玩家路径

**目标**：把环境可视化，确保玩家不会迷失

**操作方法**：
1. 画环境布局图（俯视图/侧视图，标注区域划分）
2. 标注玩家主路径（从起点到终点的主要路线）
3. 标注次级路径（探索分支、隐藏区域）
4. 标注关键地标（视觉锚点、引导标识）
5. 标注视线遮挡（避免玩家看到不该看的内容）

**检查清单**：
- [ ] 环境布局图完整
- [ ] 玩家主路径清晰
- [ ] 关键地标明确
- [ ] 视线遮挡合理

### 步骤4：定义环境与玩法/美术/性能的结合点

**目标**：让环境与玩法/美术/性能结合，不割裂体验

**操作方法**：
1. 列环境与玩法的结合点（遭遇、掩体、互动、隐藏）
2. 列环境与美术的结合点（色彩、光照、材质、风格一致性）
3. 列环境与性能的结合点（LOD、遮挡剔除、烘焙、资源优化）
4. 写环境在不同阶段的呈现方式（白天/夜晚、天气变化、动态元素）

**检查清单**：
- [ ] 环境与玩法结合点明确
- [ ] 环境与美术结合点明确
- [ ] 环境与性能结合点明确
- [ ] 动态呈现方式可维护

### 步骤5：输出环境规格并组织测试迭代

**目标**：让美术与程序能直接落地，并能高效迭代

**操作方法**：
1. 输出环境布局图（区域划分、玩家路径、关键地标）
2. 输出视觉配置表（色彩、光照、材质、细节密度）
3. 输出氛围配置表（基调、情绪触发点、故事线索）
4. 输出玩法配置表（遭遇、掩体、互动、隐藏）
5. 输出性能配置表（LOD、遮挡剔除、烘焙、资源预算）
6. 收集玩家反馈（迷失度、氛围满意度、探索乐趣）并形成迭代记录

**检查清单**：
- [ ] 环境布局图完整
- [ ] 所有配置表可配置且有默认值
- [ ] 性能配置有预算与监控
- [ ] 有迭代记录（问题-假设-改动-结果）

## 操作注意事项

### 沉浸感与一致性

1. 环境必须与世界观一致，不要出现风格冲突。
2. 氛围必须与玩法匹配，不要在紧张场景使用轻松氛围。

### 可玩性与引导

1. 玩家路径必须有清晰引导，不要让玩家迷失。
2. 关键地标必须有视觉突出，让玩家能快速定位。

### 性能与优化

1. 必须考虑性能预算（多边形数量、纹理大小、光照计算）。
2. 必须使用 LOD、遮挡剔除、烘焙等技术优化性能。

## 输出格式

### 环境设计文档

- 环境目标、约束、验收标准、失败信号
- 环境布局图（区域划分、玩家路径、关键地标）
- 视觉配置表（色彩、光照、材质、细节密度）
- 氛围配置表（基调、情绪触发点、故事线索）
- 玩法配置表（遭遇、掩体、互动、隐藏）
- 性能配置表（LOD、遮挡剔除、烘焙、资源预算）
- 动态呈现方式（白天/夜晚、天气变化、动态元素）
- 测试用例与迭代记录

## 协作协议

### 向谁汇报

- lead-artist：环境与整体美术风格对齐
- lead-level-designer：环境与关卡玩法对齐
- technical-director：环境可实现性与性能约束

### 协调对象

- level-designer：环境与玩法结合点（遭遇、掩体、互动）
- environment-artist：环境美术实现（模型、材质、光照）
- gameplay-programmer：环境互动实现（触发、碰撞、状态）
- qa-tester：环境测试、性能测试、迭代验证

## 常见错误

### 错误1：只写"主题/氛围"，没有具体的环境布局与参数

**问题**：实现阶段无法拆分任务，环境容易空洞或不可落地。

**修正**：按步骤2/3补齐环境布局图与配置表。

### 错误2：玩家路径不清晰，导致玩家迷失

**问题**：玩家在环境中不知道去哪里，失去探索乐趣。

**修正**：在步骤3重新设计玩家路径，增加关键地标与引导标识。

### 错误3：环境与玩法冲突，导致体验割裂

**问题**：环境不支持玩法，或玩法破坏环境氛围。

**修正**：在步骤4重新评估环境与玩法的结合点，确保一致性。
