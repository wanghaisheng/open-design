## 操作步骤

### 步骤1：读取PRD和成长需求

**目标**：读取游戏PRD，理解玩家成长需求和目标

**操作方法**：
1. 读取游戏PRD文档
2. 提取游戏类型和核心玩法
3. 提取目标用户和游戏时长预期
4. 提取现有成长系统设计
5. 分析玩家成长的核心驱动力
6. 提取商业化目标和付费成长需求

**检查清单**：
- [ ] PRD文档已读取

- [ ] 目标用户已提取

## 操作注意事项

### 核心设计原则

1. **激励驱动**：成长系统应提供清晰的激励，驱动玩家持续游玩
2. **成就感**：成长系统应让玩家感受到成就感和进步
3. **长期留存**：成长系统应支持长期留存，提供持续目标
4. **平衡性**：成长系统应平衡免费和付费玩家体验

### 成长曲线设计原则

1. **早期快速成长**：新手期快速成长，让玩家快速体验核心玩法
2. **中期稳定成长**：中期稳定成长，保持玩家兴趣
3. **后期慢速成长**：后期慢速成长，延长游戏寿命
4. **里程碑设计**：在关键节点设置里程碑，提供重大奖励

### 技能树设计原则

1. **多样性**：提供多样的技能选择，支持不同玩法风格
2. **深度**：技能树应有足够深度，支持长期成长
3. **平衡性**：技能应保持平衡，避免最优解
4. **重置机制**：提供技能重置机制，允许玩家尝试不同配置

### 关卡解锁设计原则

1. **循序渐进**：关卡解锁应循序渐进，难度逐步提升
2. **内容丰富**：解锁的内容应丰富多样，保持新鲜感
3. **重复价值**：提供重复游玩价值，延长游戏寿命
4. **挑战性**：提供挑战性内容，满足硬核玩家需求

### 付费平衡原则

1. **公平性**：付费不应破坏游戏平衡性
2. **加速而非必需**：付费应加速成长而非必需
3. **价值感**：付费内容应有明确价值感
4. **免费路径**：提供完整的免费成长路径

## 常见错误

### 错误1：成长曲线设计不合理

**问题**：成长曲线过快或过慢，影响游戏体验

**修正**：设计合理的成长曲线，早期快速、中期稳定、后期慢速

### 错误2：技能树缺乏多样性

**问题**：技能树选择单一，缺乏策略深度

**修正**：设计多样的技能选择，支持不同玩法风格

### 错误3：关卡解锁节奏不当

**问题**：关卡解锁过快或过慢，影响玩家体验

**修正**：设计循序渐进的关卡解锁，难度逐步提升

### 错误4：长期目标缺乏吸引力

**问题**：长期目标奖励不足，缺乏吸引力

**修正**：设计有吸引力的长期目标，提供稀有和独特奖励

### 错误5：付费影响平衡性

**问题**：付费严重破坏游戏平衡性

**修正**：确保付费加速而非必需，保持游戏公平性

### 错误6：成长反馈不足

**问题**：成长反馈不清晰，玩家感受不到进步

**修正**：设计清晰的成长反馈，包括视觉、数值和成就反馈
