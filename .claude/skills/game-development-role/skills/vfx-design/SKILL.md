# VFX Design

设计游戏特效概念和视觉风格，创造震撼的视觉特效概念。

## 适用场景

- 从 0 到 1 设计游戏特效（特效概念、视觉风格、技术考虑、玩法整合）
- 需要把特效拆解为可实现的类型、触发时机、视觉参数与性能预算
- 特效出现视觉冲击不足、性能问题、玩法冲突、风格不一致等问题，需要迭代优化
- 需要与美术/程序/关卡设计协作，明确特效与玩法/美术/性能的结合点

## 操作步骤

### 步骤1：把特效目标写成"可验收"的一句话

**目标**：明确特效服务什么体验目标、成功标准是什么

**操作方法**：
1. 写特效主题与一句话目标（面向玩家视觉体验与玩法反馈）
2. 列出特效输入/输出（玩家行为、触发条件、视觉反馈）
3. 写 3-5 条成功标准（例如：特效在 0.5 秒内触发；视觉冲击力强且帧率稳定）
4. 写 3-5 条失败信号（例如：特效延迟导致反馈模糊；特效导致帧率下降）

**检查清单**：
- [ ] 有清晰边界：特效"包含什么/不包含什么"
- [ ] 有成功标准：可观测、可测试
- [ ] 有失败信号：能快速触发调整

### 步骤2：用"特效概念-视觉风格-技术考虑-玩法整合"四块拆解特效

**目标**：把特效拆成可实现的设计单元，避免只写概念

**操作方法**：
1. **特效概念**：列特效类型/功能/场景/时机（每条写类型、触发条件、应用场景、持续时间）
2. **视觉风格**：列色彩风格/粒子风格/光照风格/动态风格（每条写色调、粒子参数、光照类型、动态曲线）
3. **技术考虑**：列性能影响/实现难度/资源需求/兼容性（每条写性能预算、实现方式、资源大小、平台适配）
4. **玩法整合**：列玩法支持/反馈机制/平衡调整/玩家体验（每条写玩法关联、反馈时机、平衡参数、体验优化）

**检查清单**：
- [ ] 每个特效都有触发条件与持续时间
- [ ] 每个视觉风格都有具体参数
- [ ] 每个技术考虑都有性能预算
- [ ] 每个玩法整合都有反馈时机

### 步骤3：设计特效触发时机与视觉反馈

**目标**：确保特效与玩法紧密结合，提供清晰反馈

**操作方法**：
1. 列所有触发条件（按键、碰撞、状态变化、时间到达）
2. 为每个触发条件写延迟要求（例如：按键后 0.1 秒内触发）
3. 设计视觉反馈层次（主反馈/次反馈/背景反馈）
4. 设计特效优先级（重要特效优先渲染，避免性能冲突）

**检查清单**：
- [ ] 所有触发条件已列出
- [ ] 每个触发条件有延迟要求
- [ ] 视觉反馈层次清晰
- [ ] 特效优先级合理

### 步骤4：定义特效与美术/性能的结合点

**目标**：让特效与美术/性能结合，不割裂体验

**操作方法**：
1. 列特效与美术的结合点（色彩、风格、场景一致性）
2. 列特效与性能的结合点（粒子数量、纹理大小、着色器复杂度、LOD）
3. 列特效与玩法的结合点（反馈机制、平衡调整、玩家体验）
4. 写特效在不同平台的呈现方式（PC/移动/主机，不同性能预算）

**检查清单**：
- [ ] 特效与美术结合点明确
- [ ] 特效与性能结合点明确
- [ ] 特效与玩法结合点明确
- [ ] 平台呈现方式可维护

### 步骤5：输出特效规格并组织测试迭代

**目标**：让美术与程序能直接落地，并能高效迭代

**操作方法**：
1. 输出特效配置表（类型、触发条件、持续时间、视觉参数）
2. 输出视觉配置表（色彩、粒子、光照、动态参数）
3. 输出技术配置表（性能预算、实现方式、资源大小、平台适配）
4. 输出玩法配置表（玩法关联、反馈时机、平衡参数）
5. 收集玩家反馈（视觉冲击力、反馈清晰度、性能体验）并形成迭代记录

**检查清单**：
- [ ] 所有配置表完整且可配置
- [ ] 性能配置有预算与监控
- [ ] 测试用例可复现
- [ ] 有迭代记录（问题-假设-改动-结果）

## 操作注意事项

### 视觉冲击与风格一致

1. 特效必须有强烈视觉冲击，避免平淡无奇。
2. 特效必须与游戏/美术/场景风格一致，避免风格冲突。

### 性能优化与资源管理

1. 必须考虑性能预算（粒子数量、纹理大小、着色器复杂度）。
2. 必须使用 LOD、对象池、烘焙等技术优化性能。

### 玩法支持与反馈清晰

1. 特效必须提供清晰反馈，不要让玩家困惑。
2. 特效必须支持玩法，不要为了视觉而影响游戏体验。

## 输出格式

### 特效设计文档

- 特效目标、约束、验收标准、失败信号
- 特效配置表（类型、触发条件、持续时间、视觉参数）
- 视觉配置表（色彩、粒子、光照、动态参数）
- 技术配置表（性能预算、实现方式、资源大小、平台适配）
- 玩法配置表（玩法关联、反馈时机、平衡参数）
- 平台呈现方式（PC/移动/主机，不同性能预算）
- 测试用例与迭代记录

## 协作协议

### 向谁汇报

- lead-artist：特效与整体美术风格对齐
- lead-technical-artist：特效可实现性与性能约束
- technical-director：特效技术方案与性能预算

### 协调对象

- vfx-artist：特效美术实现（粒子、着色器、动画）
- gameplay-programmer：特效触发实现（触发条件、状态管理）
- level-designer：特效与关卡结合点（场景、遭遇、反馈）
- qa-tester：特效测试、性能测试、迭代验证

## 常见错误

### 错误1：只写"主题/氛围"，没有具体的特效参数与性能预算

**问题**：实现阶段无法拆分任务，特效容易空洞或不可落地。

**修正**：按步骤2/3补齐特效配置表与性能预算。

### 错误2：特效延迟导致反馈模糊

**问题**：玩家操作后没有及时反馈，失去游戏沉浸感。

**修正**：在步骤3重新设计触发时机，确保延迟在可接受范围内（例如 0.1 秒内）。

### 错误3：特效导致性能问题

**问题**：特效粒子过多或资源过大，导致帧率下降。

**修正**：在步骤4重新评估性能预算，使用 LOD、对象池、烘焙等技术优化。
