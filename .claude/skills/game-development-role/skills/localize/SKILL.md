# Localize

运行本地化工作流程：提取字符串、验证本地化准备度、检查硬编码文本、生成翻译就绪的字符串表。

## 适用场景

- 需要提取字符串
- 需要验证本地化准备度
- 需要检查硬编码文本
- 需要生成翻译就绪的字符串表
- 需要生成本地化状态报告

## 操作步骤

### 步骤1：扫描

**目标**：扫描本地化问题

**操作方法**：
1. 搜索硬编码面向玩家的字符串（使用Grep）
2. 搜索本地化反模式（日期/时间格式化、数字格式化、图像/纹理中的文本、假设从左到右文本方向）
3. 检查缺失键
4. 检查本地化问题

**检查清单**：
- [ ] 硬编码字符串已搜索
- [ ] 本地化反模式已搜索
- [ ] 缺失键已检查
- [ ] 本地化问题已检查

### 步骤2：提取

**目标**：提取本地化字符串

**操作方法**：
1. 扫描所有源文件的本地化字符串引用
2. 对比现有字符串表（CSV, JSON, XML）
3. 为无键字符串生成新条目
4. 建议键名：[category].[subcategory].[description]
5. 输出要添加到字符串表的新字符串差异

**检查清单**：
- [ ] 本地化字符串引用已扫描
- [ ] 现有字符串表已对比
- [ ] 新条目已生成
- [ ] 键名已建议
- [ ] 新字符串差异已输出

### 步骤3：验证

**目标**：验证本地化

**操作方法**：
1. 读取所有字符串表文件
2. 检查每个条目（缺失翻译、占位符不匹配、字符串长度违规、孤立键）
3. 按语言和严重性分组报告验证结果

**检查清单**：
- [ ] 字符串表文件已读取
- [ ] 缺失翻译已检查
- [ ] 占位符不匹配已检查
- [ ] 字符串长度违规已检查
- [ ] 孤立键已检查
- [ ] 验证结果已报告

### 步骤4：状态

**目标**：生成本地化状态

**操作方法**：
1. 计数总本地化字符串
2. 每语言：已翻译、未翻译、过时
3. 生成覆盖率矩阵

**检查清单**：
- [ ] 总本地化字符串已计数
- [ ] 每语言翻译状态已统计
- [ ] 覆盖率矩阵已生成

## 规则

- 英语（en）始终是源语言
- 每个字符串表条目必须包含解释上下文的翻译注释
- 从不直接修改翻译文件——生成差异供审查
- 必须定义每个UI元素的字符限制并自动执行
- 从一开始考虑从右到左（RTL）语言支持，而不是后来附加

## 常见错误

### 错误1：不扫描硬编码字符串

**问题**：不搜索硬编码面向玩家的字符串

**修正**：必须搜索硬编码面向玩家的字符串

### 错误2：不验证占位符

**问题**：不检查占位符不匹配

**修正**：必须检查占位符不匹配

### 错误3：不检查字符串长度

**问题**：不检查字符串长度违规

**修正**：必须检查字符串长度违规

### 错误4：不生成覆盖率矩阵

**问题**：不生成覆盖率矩阵

**修正**：必须生成覆盖率矩阵

### 错误5：直接修改翻译文件

**问题**：直接修改翻译文件

**修正**：从不直接修改翻译文件——生成差异供审查
