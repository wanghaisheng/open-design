# Skill Logic Evaluation

评估游戏技能系统的逻辑性，包括技能链完整性、条件一致性、数值合理性和交互逻辑，使用量化评分体系和测试验证方法，确保技能系统的逻辑自洽和平衡性。

## 适用场景

- 技能系统设计完成后需要逻辑评估
- 技能出现逻辑冲突或数值失衡
- 需要量化评估技能逻辑性，为迭代提供依据
- 玩家反馈技能体验不流畅或平衡性差
- 需要建立持续改进的技能质量监控机制

## 操作步骤

### 步骤1：定义技能分类体系

**目标**：建立清晰的技能分类，便于系统化评估

**操作方法**：
1. 定义技能类型分类：
   - 主动技能：瞬发技能、持续技能、蓄力技能
   - 被动技能：常驻效果、条件触发
   - 连携技能：连锁逻辑、协同机制
2. 为每个子类型定义评估要点
3. 建立技能分类与评估维度的映射关系

**检查清单**：
- [ ] 技能分类完整
- [ ] 子类型评估要点明确
- [ ] 映射关系清晰

### 步骤2：技能链完整性检查

**目标**：验证每个技能从输入到输出的完整逻辑链

**操作方法**：
1. 绘制技能逻辑流程图：
   - 输入检测 → 条件验证 → 资源检查 → 动画播放 → 效果计算 → 结果应用 → 冷却开始
2. 检查每个环节的完整性：
   - 输入检测：按键/触发是否明确
   - 条件验证：状态/距离/目标是否合理
   - 资源检查：MP/CD/能量是否充足
   - 动画播放：动作/特效是否匹配
   - 效果计算：伤害/Buff是否正确
   - 结果应用：HP变更/状态改变是否生效
   - 冷却开始：计时器是否启动
3. 识别断链环节

**检查清单**：
- [ ] 逻辑流程图完整
- [ ] 每个环节已检查
- [ ] 断链环节已识别

### 步骤3：条件一致性检查

**目标**：验证触发条件与效果表现的逻辑自洽

**操作方法**：
1. 检查触发条件：
   - 条件是否明确（距离、状态、目标类型）
   - 条件是否可达成
   - 条件是否与效果匹配
2. 检查效果表现：
   - 效果是否与条件对应
   - 效果范围是否合理
   - 效果持续时间是否恰当
3. 识别条件与效果不匹配的情况

**检查清单**：
- [ ] 触发条件已检查
- [ ] 效果表现已检查
- [ ] 不匹配情况已识别

### 步骤4：数值合理性检查

**目标**：验证伤害倍率、冷却时间、资源消耗的平衡

**操作方法**：
1. 检查伤害公式：
   - BaseDamage × (1 + StatBonus) × (1 + ComboBonus) × (1 + ElementBonus) - Defense
   - 验证公式逻辑自洽
   - 检查数值范围合理性
2. 检查冷却时间：
   - 冷却时间与技能强度匹配
   - 冷却时间与战斗节奏匹配
3. 检查资源消耗：
   - 消耗与效果匹配
   - 消耗与资源恢复匹配
4. 识别数值异常

**检查清单**：
- [ ] 伤害公式已验证
- [ ] 冷却时间已检查
- [ ] 资源消耗已检查
- [ ] 数值异常已识别

### 步骤5：交互逻辑检查

**目标**：验证技能间的相互作用和冲突解决机制

**操作方法**：
1. 检查技能组合：
   - 连携技能的连锁逻辑
   - 协同机制的站位要求
   - 时机窗口的合理性
2. 检查技能冲突：
   - 同时释放的冲突解决
   - 效果叠加的处理
   - 优先级的定义
3. 识别交互问题

**检查清单**：
- [ ] 技能组合已检查
- [ ] 技能冲突已检查
- [ ] 交互问题已识别

### 步骤6：边界情况检查

**目标**：验证极端场景下的技能行为定义

**操作方法**：
1. 测试极端数值：
   - 最大/最小属性值
   - 最大/最小距离
   - 最大/最小持续时间
2. 测试极端场景：
   - 无目标释放
   - 目标死亡释放
   - 资源不足释放
3. 识别边界处理缺失

**检查清单**：
- [ ] 极端数值已测试
- [ ] 极端场景已测试
- [ ] 边界处理缺失已识别

### 步骤7：平衡性验证

**目标**：使用量化指标验证技能平衡性

**操作方法**：
1. DPS评估：
   - 理论DPS计算
   - 实际DPS测试
   - 偏差控制在15%以内
2. 效用价值评估：
   - 非伤害技能的等效伤害换算
   - 辅助技能的价值评估
3. 风险评估：
   - 高收益技能的风险成本匹配度
   - 低风险技能的收益合理性
4. 多样性验证：
   - 同类型技能的差异化程度 > 30%
   - 避免技能同质化

**检查清单**：
- [ ] DPS已评估
- [ ] 效用价值已评估
- [ ] 风险已评估
- [ ] 多样性已验证

### 步骤8：量化评分

**目标**：使用标准化评分体系输出评估结果

**操作方法**：
1. 按评估维度评分（0-100分）：
   - 技能链完整性：20%
   - 条件一致性：20%
   - 数值合理性：25%
   - 交互逻辑：20%
   - 边界处理：15%
2. 计算总分
3. 确定评分等级：
   - 90-100分：逻辑完全自洽，无冲突，边界情况全覆盖
   - 80-89分：逻辑基本自洽，存在细微优化空间
   - 70-79分：逻辑存在小问题，需要调整
   - 60-69分：逻辑存在明显冲突，需要重新设计
   - <60分：逻辑严重缺陷，不建议采用
4. 生成评分报告

**检查清单**：
- [ ] 各维度已评分
- [ ] 总分已计算
- [ ] 评分等级已确定
- [ ] 评分报告已生成

### 步骤9：设计改进方案

**目标**：根据评估结果提出具体改进建议

**操作方法**：
1. 识别主要问题
2. 分析问题根源
3. 提出改进方案
4. 设定改进优先级
5. 制定改进时间表

**检查清单**：
- [ ] 问题已识别
- [ ] 根源已分析
- [ ] 方案已提出
- [ ] 优先级已设定

## 操作注意事项

### 评估客观性

1. 评估应基于明确的评分标准，避免主观臆断
2. 使用量化指标，确保评估可重复
3. 多轮测试，减少偶然性

### 测试全面性

1. 基础功能测试：单技能在理想条件下的表现
2. 组合测试：多技能配合的协同效应
3. 对抗测试：技能在PvP环境中的平衡性
4. 边界测试：极端数值下的系统稳定性

### 改进持续性

1. 建立版本记录表
2. 定期进行技能逻辑评估
3. 监控玩家技能使用数据
4. 收集玩家反馈

## 常见错误

### 错误1：评估标准不明确

**问题**：评分标准模糊，导致评估结果主观性强

**修正**：在步骤1明确定义每个维度的评分标准和等级定义

### 错误2：忽略边界情况

**问题**：只测试正常场景，忽略极端情况

**修正**：在步骤6增加边界情况检查，测试极端数值和极端场景

### 错误3：改进方案不具体

**问题**：只指出问题，不提供具体改进方案

**修正**：在步骤9为每个问题提供具体的改进方案和优先级
