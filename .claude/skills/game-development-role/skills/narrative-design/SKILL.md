# Narrative Design

设计游戏叙事系统，包括故事结构、角色发展、对话系统和叙事分支，创造引人入胜的游戏故事。

## 适用场景

- 从 0 到 1 设计游戏叙事系统（主线/支线/背景/角色故事）
- 需要把叙事拆解为故事结构、角色弧线、对话树与分支选择，并明确实现方式
- 叙事出现角色行为不一致、选择无意义、剧情割裂等问题，需要迭代优化
- 需要与关卡/程序/美术协作，明确对话触发、分支条件与呈现方式

## 操作步骤

### 步骤1：把叙事目标写成"可验收"的一句话

**目标**：明确叙事服务什么体验目标、成功标准是什么

**操作方法**：
1. 写叙事主题与一句话目标（面向玩家情感体验）
2. 列出叙事输入/输出（玩家行为、选择、状态变化）
3. 写 3-5 条成功标准（例如：玩家理解角色动机；选择有可见后果）
4. 写 3-5 条失败信号（例如：玩家不知道为什么选；剧情前后矛盾）

**检查清单**：
- [ ] 有清晰边界：叙事"包含什么/不包含什么"
- [ ] 有成功标准：可观测、可测试
- [ ] 有失败信号：能快速触发调整

### 步骤2：用"故事结构 + 角色弧线"拆解叙事

**目标**：把叙事拆成可实现的设计单元，避免只写概念

**操作方法**：
1. **故事结构**：列主线/支线/背景故事（每条写起止节点、关键事件、与玩法的关联）
2. **角色弧线**：列主要角色/次要角色（每条写起点-转折-终点、动机变化、成长/堕落）
3. **角色关系**：列角色之间关系（支持/对立/中立，写关系变化点）
4. **世界规则**：列世界观约束（魔法/科技/社会规则，写一致性要求）

**检查清单**：
- [ ] 每条故事线都有起止节点与关键事件
- [ ] 每个角色都有清晰动机与弧线
- [ ] 角色关系与世界观规则一致

### 步骤3：设计对话树与触发条件

**目标**：把对话从"点子"写成可实现的触发与分支

**操作方法**：
1. 为每个对话写目的（信息/教学/情感/决策）
2. 写触发条件（进入区域/拾取物/击败某敌/时间到/好感度阈值）
3. 写对话分支（选项、条件、后果、后续对话节点）
4. 写对话呈现方式（文本/语音/过场动画/镜头控制）

**检查清单**：
- [ ] 每个对话都有目的与触发条件
- [ ] 分支有明确条件与后果
- [ ] 对话呈现方式可实现

### 步骤4：设计选择系统与多结局

**目标**：让玩家选择有意义，并且后果可见

**操作方法**：
1. 列关键选择点（影响剧情/角色/世界）
2. 为每个选择写后果（短期/长期、可见/隐藏）
3. 设计多结局（不同选择路径导向不同结局，写达成条件）
4. 写叙事一致性检查（避免矛盾分支）

**检查清单**：
- [ ] 关键选择有可见后果
- [ ] 多结局有明确达成条件
- [ ] 分支之间无矛盾

### 步骤5：将叙事整合到玩法与世界

**目标**：让叙事与玩法结合，不割裂体验

**操作方法**：
1. 列叙事与玩法的结合点（任务/关卡/道具/技能）
2. 列叙事与世界观的结合点（环境/NPC/事件）
3. 写叙事呈现时机（不打断核心玩法，或在停顿点呈现）
4. 写叙事回放/跳过机制（尊重玩家选择）

**检查清单**：
- [ ] 叙事与玩法结合点明确
- [ ] 叙事呈现时机合理
- [ ] 有回放/跳过机制

### 步骤6：输出叙事规格并组织测试迭代

**目标**：让程序与美术能直接落地，并能高效迭代

**操作方法**：
1. 输出对话树（节点、条件、分支、后果）
2. 输出角色表（性格、动机、弧线、关系）
3. 输出触发条件表（事件、参数、阈值）
4. 输出测试用例（所有分支、结局、一致性检查）
5. 收集玩家反馈（理解度/情感共鸣/选择满意度）并形成迭代记录

**检查清单**：
- [ ] 对话树完整且可实现
- [ ] 角色表与触发条件表完整
- [ ] 测试用例覆盖所有分支
- [ ] 有迭代记录（问题-假设-改动-结果）

## 操作注意事项

### 沉浸感与一致性

1. 角色行为必须符合动机与世界观，不要为了剧情需要强行改变。
2. 世界规则必须前后一致，避免临时引入新规则解决剧情冲突。

### 互动性与有意义选择

1. 选择必须有可见后果，不要让玩家感觉"选了也没用"。
2. 分支数量要可控，避免维护成本爆炸（优先关键选择点）。

### 可玩性与体验节奏

1. 叙事呈现不要打断核心玩法节奏，优先在停顿点呈现。
2. 提供回放/跳过机制，尊重玩家的节奏偏好。

## 输出格式

### 叙事设计文档

- 叙事目标、约束、验收标准、失败信号
- 故事结构（主线/支线/背景/角色故事）
- 角色表（性格、动机、弧线、关系）
- 对话树（节点、条件、分支、后果、呈现方式）
- 选择系统（关键选择点、后果、多结局达成条件）
- 触发条件表（事件、参数、阈值）
- 玩法/世界整合点
- 测试用例与迭代记录

## 协作协议

### 向谁汇报

- lead-writer：叙事风格、世界观一致性
- creative-director：叙事与整体体验目标对齐

### 协调对象

- level-designer：叙事在关卡中的触发点与呈现时机
- gameplay-programmer：对话树实现、触发条件、分支逻辑
- character-artist：角色视觉与叙事一致性
- qa-tester：分支测试、结局测试、一致性检查

## 常见错误

### 错误1：只写"主题/氛围"，没有具体的故事结构与角色弧线

**问题**：实现阶段无法拆分任务，叙事容易空洞或割裂。

**修正**：按步骤2补齐故事结构、角色弧线与关系。

### 错误2：选择没有可见后果，玩家感觉"选了也没用"

**问题**：玩家对叙事失去兴趣，不再认真选择。

**修正**：在步骤4为每个关键选择写短期/长期后果，并确保可见。

### 错误3：叙事打断核心玩法节奏

**问题**：玩家体验割裂，容易跳过或反感。

**修正**：在步骤5重新规划叙事呈现时机，优先在停顿点呈现，并提供跳过机制。
