# World Consistency Check

检查世界一致性，确保游戏世界的地理、历史、文化和规则在不同场景和内容中保持一致。

## 适用场景

- 需要检查地理一致性
- 需要检查历史一致性
- 需要检查文化一致性
- 需要检查规则一致性
- 需要跟踪修正进度

## 操作步骤

### 步骤1：建立世界档案

**目标**：建立世界档案

**操作方法**：
1. 建立地理档案（地理位置、地形地貌、气候环境、空间关系）
2. 建立历史档案（历史事件、时间线、历史影响、历史人物）
3. 建立文化档案（社会结构、文化传统、宗教信仰、价值观念）
4. 建立规则档案（物理规则、魔法规则、社会规则、经济规则）

**检查清单**：
- [ ] 地理档案已建立
- [ ] 历史档案已建立
- [ ] 文化档案已建立
- [ ] 规则档案已建立

### 步骤2：执行检查

**目标**：执行一致性检查

**操作方法**：
1. 检查地理一致性（地理位置、地形地貌、气候环境、空间关系）
2. 检查历史一致性（历史事件、时间线、历史影响、历史人物）
3. 检查文化一致性（社会结构、文化传统、宗教信仰、价值观念）
4. 检查规则一致性（物理规则、魔法规则、社会规则、经济规则）

**检查清单**：
- [ ] 地理一致性已检查
- [ ] 历史一致性已检查
- [ ] 文化一致性已检查
- [ ] 规则一致性已检查

### 步骤3：问题记录

**目标**：记录不一致问题

**操作方法**：
1. 记录不一致问题（问题描述、问题位置）
2. 记录问题严重性（高/中/低严重性）
3. 记录修正建议（修正方法、修正优先级）

**检查清单**：
- [ ] 不一致问题已记录
- [ ] 问题严重性已记录
- [ ] 修正建议已记录

### 步骤4：反馈修正

**目标**：提供反馈并跟踪修正

**操作方法**：
1. 提供检查报告（检查结果、问题列表）
2. 提供修正建议（修正方案、修正时间）
3. 跟踪修正进度（修正状态、修正验证）

**检查清单**：
- [ ] 检查报告已提供
- [ ] 修正建议已提供
- [ ] 修正进度已跟踪

## 输出格式

### 检查报告
- 一致性检查结果
- 问题列表
- 修正建议

## 常见错误

### 错误1：不建立世界档案

**问题**：不建立地理档案、不建立历史档案

**修正**：必须建立地理档案，历史档案，文化档案，规则档案

### 错误2：不执行一致性检查

**问题**：不检查地理一致性、不检查历史一致性

**修正**：必须检查地理一致性，历史一致性，文化一致性，规则一致性

### 错误3：不记录不一致问题

**问题**：不记录不一致问题、不记录问题严重性

**修正**：必须记录不一致问题，记录问题严重性，记录修正建议

### 错误4：不跟踪修正进度

**问题**：不提供检查报告、不跟踪修正进度

**修正**：必须提供检查报告，提供修正建议，跟踪修正进度

### 错误5：检查不全面

**问题**：不全面检查世界、不全面检查场景

**修正**：必须全面检查世界，全面检查场景，全面检查内容
