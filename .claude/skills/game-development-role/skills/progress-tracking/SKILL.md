# Progress Tracking

跟踪项目进度，包括进度监控、里程碑跟踪、任务状态跟踪和进度报告，确保项目按计划进行。

## 适用场景

- 需要监控任务进度
- 需要跟踪里程碑
- 需要跟踪任务状态
- 需要生成进度报告
- 需要持续改进跟踪

## 操作步骤

### 步骤1：建立基线

**目标**：建立进度基线

**操作方法**：
1. 建立进度基线（进度基线）
2. 确定跟踪指标（跟踪指标）
3. 设置跟踪工具（Jira/Trello/Asana/Monday）
4. 培训团队（团队培训）

**检查清单**：
- [ ] 进度基线已建立
- [ ] 跟踪指标已确定
- [ ] 跟踪工具已设置
- [ ] 团队已培训

### 步骤2：日常跟踪

**目标**：日常跟踪

**操作方法**：
1. 每日更新进度（任务进度、里程碑进度、资源进度、预算进度）
2. 每日检查阻塞（阻塞任务）
3. 每日识别风险（风险识别）
4. 每日调整计划（计划调整）

**检查清单**：
- [ ] 进度已更新
- [ ] 阻塞已检查
- [ ] 风险已识别
- [ ] 计划已调整

### 步骤3：定期审查

**目标**：定期审查

**操作方法**：
1. 每周审查进度（进度审查）
2. 每周审查里程碑（里程碑审查）
3. 每周审查资源（资源审查）
4. 每周调整计划（计划调整）

**检查清单**：
- [ ] 进度已审查
- [ ] 里程碑已审查
- [ ] 资源已审查
- [ ] 计划已调整

### 步骤4：进度报告

**目标**：生成进度报告

**操作方法**：
1. 生成进度报告（日报、周报、月报、里程碑报告）
2. 分析进度数据（数据分析）
3. 识别进度问题（问题识别）
4. 提出改进建议（改进建议）

**检查清单**：
- [ ] 进度报告已生成
- [ ] 进度数据已分析
- [ ] 进度问题已识别
- [ ] 改进建议已提出

### 步骤5：持续改进

**目标**：持续改进

**操作方法**：
1. 优化跟踪方法（跟踪方法优化）
2. 优化跟踪工具（跟踪工具优化）
3. 优化跟踪流程（跟踪流程优化）
4. 提高跟踪效率（跟踪效率提升）

**检查清单**：
- [ ] 跟踪方法已优化
- [ ] 跟踪工具已优化
- [ ] 跟踪流程已优化
- [ ] 跟踪效率已提升

## 常见错误

### 错误1：不建立基线

**问题**：不建立进度基线，无法对比

**修正**：必须建立进度基线，确定跟踪指标

### 错误2：不日常跟踪

**问题**：不每日更新进度，不检查阻塞

**修正**：必须每日更新进度，检查阻塞，识别风险

### 错误3：不定期审查

**问题**：不每周审查进度、里程碑、资源

**修正**：必须每周审查进度、里程碑、资源

### 错误4：不生成报告

**问题**：不生成进度报告

**修正**：必须生成进度报告，分析进度数据

### 错误5：不持续改进

**问题**：不优化跟踪方法、工具、流程

**修正**：必须优化跟踪方法、工具、流程，提高跟踪效率
