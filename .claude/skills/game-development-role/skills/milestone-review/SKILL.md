# Milestone Review

生成全面的里程碑进度审查，包括功能完整性、质量指标、风险评估和通过/不通过建议。在里程碑检查点或评估里程碑截止日期准备度时使用。

## 审查要素

### 1. 功能完整性 (Feature Completeness)
- 完全完成：功能完全实现并通过验收
- 部分完成：功能部分完成，剩余工作明确
- 未开始：功能未开始，可考虑削减

### 2. 质量指标 (Quality Metrics)
- S1错误：严重错误数量和列表
- S2错误：重要错误数量
- S3错误：轻微错误数量
- 测试覆盖：测试覆盖率
- 性能：性能是否在预算内

### 3. 代码健康 (Code Health)
- TODO计数：代码库中TODO标记数量
- FIXME计数：需要修复的标记数量
- HACK计数：临时hack标记数量
- 技术债务：关键技术债务项目

### 4. 风险评估 (Risk Assessment)
- 风险状态：当前风险状态
- 实现影响：如果风险实现的影响
- 缓解状态：缓解措施状态

### 5. 速度分析 (Velocity Analysis)
- 计划vs完成：所有冲刺的计划vs完成
- 趋势：改善/稳定/下降
- 调整估算：当前速度下剩余工作所需天数

## 审查原则

### 1. 全面性
- 全面审查功能
- 全面审查质量
- 全面审查风险

### 2. 客观性
- 基于数据
- 避免主观判断
- 可验证指标

### 3. 可操作性
- 提供可操作建议
- 提供明确行动项
- 提供明确决策

### 4. 及时性
- 及时审查
- 及时反馈
- 及时调整

## 审查流程

### 1. 读取里程碑定义
- 读取 production/milestones/
- 理解里程碑目标
- 理解里程碑范围

### 2. 读取所有冲刺报告
- 读取 production/sprints/
- 理解冲刺进度
- 理解速度趋势

### 3. 扫描代码库
- 扫描TODO标记
- 扫描FIXME标记
- 扫描HACK标记
- 识别未完成工作

### 4. 检查风险登记册
- 读取 production/risk-register/
- 评估风险状态
- 评估缓解措施

### 5. 生成里程碑审查
- 评估功能完整性
- 评估质量指标
- 评估代码健康
- 评估风险
- 分析速度
- 提供建议
- 评估通过/不通过

|------|------|-----------------|----------|

- **计划vs完成**（所有冲刺）：[X/Y任务 = Z%]
- **趋势**：[改善/稳定/下降]
- **剩余工作的调整估算**：[当前速度下所需天数]

## 通过/不通过评估

**建议**：[通过/条件通过/不通过]
