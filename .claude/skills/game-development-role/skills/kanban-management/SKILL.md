# Kanban Management

看板管理技能，使用看板方法进行工作流管理和可视化。

## 适用场景

- 管理开发工作流
- 可视化任务状态
- 限制在制品（WIP）
- 识别瓶颈
- 持续改进流程

## 操作步骤

### 步骤1：设计看板

**目标**：创建看板工作流

**操作方法**：
1. 定义工作流阶段（待办、进行中、测试、完成）
2. 设置WIP限制
3. 定义列和泳道
4. 配置看板工具
5. 建立看板规则

**检查清单**：
- [ ] 工作流阶段已定义
- [ ] WIP限制已设置
- [ ] 列和泳道已配置
- [ ] 看板工具已设置
- [ ] 规则已建立

### 步骤2：管理任务

**目标**：在看板上管理任务

**操作方法**：
1. 创建任务卡片
2. 添加任务详情和标签
3. 分配负责人
4. 设置优先级
5. 跟踪任务进度

**检查清单**：
- [ ] 任务卡片已创建
- [ ] 详情和标签已添加
- [ ] 负责人已分配
- [ ] 优先级已设置
- [ ] 进度已跟踪

### 步骤3：监控流程

**目标**：监控工作流效率

**操作方法**：
1. 跟踪周期时间
2. 监控WIP限制
3. 识别瓶颈
4. 分析流程效率
5. 生成流程报告

**检查清单**：
- [ ] 周期时间已跟踪
- [ ] WIP限制已监控
- [ ] 瓶颈已识别
- [ ] 效率已分析
- [ ] 报告已生成

### 步骤4：持续改进

**目标**：持续改进看板流程

**操作方法**：
1. 定期审查看板
2. 收集团队反馈
3. 调整WIP限制
4. 优化工作流
5. 实施改进措施

**检查清单**：
- [ ] 看板已审查
- [ ] 反馈已收集
- [ ] WIP限制已调整
- [ ] 工作流已优化
- [ ] 改进已实施

## 输出格式

### 看板配置
```markdown
## 看板配置

### 列
- 待办 (WIP: 无限制)
- 进行中 (WIP: 3)
- 测试 (WIP: 2)
- 完成 (WIP: 无限制)

### 泳道
- 高优先级
- 中优先级
- 低优先级

### 标签
- bug
- feature
- improvement
```

## 操作注意事项

### WIP管理

1. **设置合理限制**：基于团队能力设置WIP限制
2. **遵守限制**：严格遵守WIP限制
3. **调整限制**：根据实际情况调整WIP限制
4. **可视化WIP**：清晰显示WIP状态

### 流程优化

1. **识别瓶颈**：定期识别流程瓶颈
2. **减少浪费**：消除不增值的活动
3. **平滑流程**：平衡工作负载
4. **持续改进**：持续优化流程

### 团队协作

1. **透明可见**：保持看板对团队可见
2. **定期同步**：定期同步看板状态
3. **鼓励反馈**：鼓励团队提供反馈
4. **协作改进**：团队协作改进流程

## 协作协议

### 向谁汇报

- producer：工作流状态
- team-lead：团队效率

### 协调对象

- project-manager：项目管理
- team-members：任务分配

## 常见错误

### 错误1：WIP限制过高

**问题**：WIP限制设置过高，失去限制意义

**修正**：基于团队能力设置合理的WIP限制

### 错误2：不更新看板

**问题**：看板不及时更新，失去可视化价值

**修正**：建立定期更新看板的机制

### 错误3：忽视瓶颈

**问题**：识别瓶颈但不采取行动

**修正**：识别瓶颈后立即采取改进措施

### 错误4：过度复杂

**问题**：看板过于复杂，难以维护

**修正**：保持看板简单有效

### 错误5：缺乏规则

**问题**：看板没有明确规则，导致混乱

**修正**：建立并遵守看板规则
