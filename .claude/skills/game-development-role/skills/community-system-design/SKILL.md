## 适用场景

- 设计游戏社区和社交系统
- 设计玩家互动和内容分享机制
- 设计社区与游戏内系统的联动
- 设计社区运营和用户引导策略
- 提升玩家付费意愿和社区活跃度

---

## 操作步骤

### 步骤1：读取PRD和社区需求

**目标**：读取游戏PRD，理解社区需求和目标

**操作方法**：
1. 读取游戏PRD文档
2. 提取游戏类型和目标用户
3. 提取社区需求和目标
4. 提取现有社交系统设计
5. 分析社区的核心价值主张
6. 提取商业化目标和付费转化需求

**检查清单**：
- [ ] PRD文档已读取
- [ ] 游戏类型已提取
- [ ] 社区需求已提取
- [ ] 社交系统已提取
- [ ] 核心价值主张已提取
- [ ] 商业化目标已提取

---

### 步骤2：确定核心设计理念

**目标**：确定社区系统的核心设计理念

**操作方法**：
1. 确定价值驱动原则
   - 社区功能应围绕提升玩家对游戏、社群以及付费内容的"感知价值"
2. 确定认同强化原则
   - 鼓励并强化玩家对其"化身"的认同感
3. 确定互动赋能原则
   - 设计"工具型"和"人际型"互动机制
4. 确定数据洞察原则
   - 持续追踪用户行为和反馈，优化社区功能

**检查清单**：
- [ ] 价值驱动原则已确定
- [ ] 认同强化原则已确定
- [ ] 互动赋能原则已确定
- [ ] 数据洞察原则已确定

---

### 步骤3：设计信息与支持中心

**目标**：设计强化工具型互动和感知价值的信息与支持中心

**操作方法**：
1. 设计官方公告区
   - 发布游戏更新、活动、付费内容详情
2. 设计知识库/Wiki
   - 官方维护+玩家共建的游戏百科
3. 设计Q&A论坛/专区
   - 玩家提问、解答，官方客服/资深玩家巡回答疑
4. 设计玩家攻略区
   - 玩家分享图文、视频攻略，设立精华推荐和奖励机制
5. 设计付费内容解析/评测区
   - 官方详细介绍付费道具/服务的功能、性价比
6. 设计用户引导策略
   - 新手指引、一键跳转、活动激励、高亮显示

**检查清单**：
- [ ] 官方公告区已设计
- [ ] 知识库已设计
- [ ] Q&A论坛已设计
- [ ] 攻略区已设计
- [ ] 付费内容解析区已设计
- [ ] 用户引导策略已设计

---

### 步骤4：设计社交与连接平台

**目标**：设计强化人际型互动、化身认同和感知价值的社交平台

**操作方法**：
1. 设计动态/广场
   - 玩家分享游戏截图、成就、心情
2. 设计话题讨论区
   - 围绕游戏剧情、角色、策略、活动等开设不同版块
3. 设计公会/战队专属空间
   - 独立的聊天、活动组织、成员管理、荣誉展示区域
4. 设计好友/私信系统
   - 支持添加好友、私聊、群聊
5. 设计活动中心
   - 官方/玩家组织线上线下活动
6. 设计LFG系统
   - 方便玩家寻找组队伙伴
7. 设计用户引导策略
   - 分享引导、社区任务、热门话题、游戏内绑定

**检查清单**：
- [ ] 动态广场已设计
- [ ] 话题讨论区已设计
- [ ] 公会专属空间已设计
- [ ] 好友系统已设计
- [ ] 活动中心已设计
- [ ] LFG系统已设计
- [ ] 用户引导策略已设计

### 步骤5：设计化身展示与认同空间

**目标**：设计强化化身认同和感知价值的化身展示空间

**操作方法**：
1. 设计个人主页/角色卡
   - 展示玩家的游戏角色外观、装备、成就、自定义签名/故事
2. 设计化身装扮/搭配分享区
   - 玩家分享自己的角色装扮、时装搭配心得
3. 设计角色故事/同人创作区
   - 鼓励玩家创作并分享围绕其化身或游戏角色的故事、图画、视频
4. 设计"我的时刻"/成就墙
   - 自动或手动记录并展示玩家化身在游戏中的高光时刻和重要成就
5. 设计化身相关投票/评选
   - 如"最受欢迎时装"、"最佳角色扮演"等
6. 设计用户引导策略
   - 完善主页引导、活动激励、分享提示、优质内容展示、游戏内关联

**检查清单**：
- [ ] 个人主页已设计
- [ ] 装扮分享区已设计
- [ ] 同人创作区已设计
- [ ] 成就墙已设计
- [ ] 投票评选已设计
- [ ] 用户引导策略已设计

### 步骤6：设计价值感知与转化枢纽

**目标**：设计直接引导付费意愿的价值感知与转化枢纽

**操作方法**：
1. 设计"种草"/推荐区
   - 结合玩家数据和社群热点，智能推荐或由KOL推荐付费道具/服务
2. 设计限时优惠/活动聚合页
   - 汇总游戏内外的所有付费优惠信息
3. 设计付费道具/服务体验反馈区
   - 收集玩家对付费内容的正面和负面反馈
4. 设计社区专属福利/积分商城
   - 参与社区互动可获得积分，积分可兑换游戏内小道具或社区专属奖励
5. 设计交易/市场信息区（谨慎使用）
   - 如果游戏允许或有二级市场，提供官方监管或引导的讨论区域
6. 设计用户引导策略
   - 个性化推送、适时展示、口碑效应、任务链、积分价值

**检查清单**：
- [ ] 种草推荐区已设计
- [ ] 优惠聚合页已设计
- [ ] 体验反馈区已设计
- [ ] 积分商城已设计
- [ ] 交易信息区已设计
- [ ] 用户引导策略已设计

### 步骤7：设计核心用户引导策略

**目标**：设计新手指引、激励反馈、活动驱动等用户引导策略

**操作方法**：
1. 设计新手指引与融入
   - 清晰导览、新手任务、推荐内容
2. 设计激励与反馈机制
   - 积分与等级系统、成就与徽章、内容推荐与曝光、官方互动与认可
3. 设计活动驱动与氛围营造
   - 定期活动、节日主题活动、玩家自发活动
4. 设计KOL/核心用户培育与赋能
   - 识别扶持、专属权益、带动氛围
5. 设计游戏内社区联动
   - 一键分享、链接回游戏、游戏内提示
6. 设计个性化与精准推送
   - 基于游戏行为、社区偏好、付费记录的推送

### 步骤8：设计玩家反馈闭环

**目标**：建立完整的玩家反馈收集、处理和公示机制

**操作方法**：
1. **反馈收集渠道设计**
   - 游戏内反馈按钮
   - 专门的Discord频道
   - 定期问卷调查
   - 社区论坛反馈区
2. **反馈处理流程设计**
   - 设计一套反馈的收集、分类、分析和响应流程
   - 确保玩家的声音能被有效传达给开发团队
3. **反馈公示机制**
   - 建议定期（如通过开发者日志）向社区公示已收到的热门反馈
   - 公示团队的相应计划，建立信任

### 步骤9：设计沟通与公关策略

**目标**：设计日常沟通和危机公关预案

**操作方法**：
1. **日常沟通模板设计**
   - 版本更新公告模板
   - 新功能介绍模板
   - Bug修复说明模板
   - 基调建议（友好、透明、专业）
2. **危机公关预案设计**
   - 针对重大Bug的沟通策略和响应流程
   - 针对服务器问题的沟通策略和响应流程
   - 针对负面舆情的沟通策略和响应流程

- [ ] KOL培育已设计

## 操作注意事项

### 核心设计原则

1. **价值驱动**：社区的核心功能应围绕提升玩家对游戏、社群以及付费内容的"感知价值"展开
2. **认同强化**：鼓励并强化玩家对其"化身"的认同感，使其成为付费意愿的重要情感基础
3. **互动赋能**：精心设计"工具型"和"人际型"互动机制，作为提升价值和认同的催化剂
4. **数据洞察**：持续追踪用户行为和反馈，优化社区功能和引导策略

### 系统模块设计原则

1. **信息与支持中心**：强化工具型互动和感知价值（功能、认知）
2. **社交与连接平台**：强化人际型互动、化身认同和感知价值（社会）
3. **化身展示与认同空间**：强化化身认同和感知价值（情感、社会）
4. **价值感知与转化枢纽**：直接引导付费意愿

### 用户引导策略原则

1. **新手指引与融入**：提供清晰导览、新手任务、推荐内容
2. **激励与反馈机制**：积分等级、成就徽章、内容推荐、官方认可
3. **活动驱动与氛围营造**：定期活动、节日主题、玩家自发
4. **KOL/核心用户培育**：识别扶持、专属权益、带动氛围
5. **游戏内社区联动**：一键分享、链接回游戏、游戏内提示
6. **个性化与精准推送**：基于行为、偏好、记录的推送

### 关键成功因素

1. **内容质量**：高质量、有价值的内容是吸引和留存用户的根本
2. **积极氛围**：营造友好、互助、尊重的社区氛围至关重要
3. **官方参与度**：官方不能只是搭建平台，还需要积极参与、引导和管理
4. **技术支持**：稳定、易用、功能完善的社区平台是基础
5. **持续迭代**：根据用户反馈和数据分析，不断优化社区功能和运营策略

## 常见错误

### 错误1：忽视核心设计理念

**问题**：社区设计缺乏明确的核心设计理念，功能零散

**修正**：明确价值驱动、认同强化、互动赋能、数据洞察四大核心设计理念

### 错误2：系统模块设计不完整

**问题**：只设计了部分系统模块，导致社区功能不完整

**修正**：设计完整的四大系统模块：信息与支持中心、社交与连接平台、化身展示空间、价值感知枢纽

### 错误3：用户引导策略缺失

**问题**：设计了社区功能，但没有配套的用户引导策略

**修正**：为每个系统模块设计配套的用户引导策略，包括新手指引、激励反馈、活动驱动等

### 错误4：游戏内社区联动不足

**问题**：社区与游戏内系统脱节，缺乏联动

**修正**：设计游戏内社区联动机制，包括一键分享、链接回游戏、游戏内提示等

### 错误5：忽视付费转化目标

**问题**：社区设计没有考虑付费转化目标

**修正**：设计价值感知与转化枢纽，通过种草推荐、优惠聚合、积分商城等引导付费意愿

### 错误6：缺乏持续迭代机制

**问题**：社区设计完成后缺乏持续迭代和优化

**修正**：建立数据洞察机制，持续追踪用户行为和反馈，不断优化社区功能和运营策略
