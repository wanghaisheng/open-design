# Team Combat

协调战斗团队：协调游戏设计师、游戏玩法程序员、AI程序员、技术艺术家、声音设计师和QA测试员，端到端设计、实施和验证战斗功能。

## 适用场景

- 需要设计战斗功能
- 需要实施战斗功能
- 需要集成战斗系统
- 需要验证战斗功能
- 需要生成战斗团队报告

## 操作步骤

### 步骤1：设计

**目标**：设计战斗功能

**操作方法**：
1. 委托给游戏设计师（创建或更新 design/gdd/ 中的设计文档）
2. 涵盖内容（机制概述、玩家幻想、详细规则、带变量定义的公式、边缘情况、依赖关系、带安全范围的调音旋钮、验收标准）
3. 输出完成的设计文档

**检查清单**：
- [ ] 设计文档已创建或更新
- [ ] 机制概述已完成
- [ ] 详细规则已完成
- [ ] 验收标准已定义

### 步骤2：架构

**目标**：设计代码架构

**操作方法**：
1. 委托给游戏玩法程序员（如涉及AI，与AI程序员一起）
2. 审查设计文档
3. 设计代码架构（类结构、接口、数据流）
4. 识别与现有系统的集成点
5. 输出架构草图（带文件列表和接口定义）

**检查清单**：
- [ ] 设计文档已审查
- [ ] 代码架构已设计
- [ ] 集成点已识别
- [ ] 架构草图已输出

### 步骤3：实施

**目标**：实施战斗功能

**操作方法**：
1. 并行委托（游戏玩法程序员、AI程序员、技术艺术家、声音设计师）
2. 游戏玩法程序员：实施核心战斗机制代码
3. AI程序员：实施AI行为（如功能涉及NPC反应）
4. 技术艺术家：创建VFX和着色器效果
5. 声音设计师：定义音频事件列表和混音说明

**检查清单**：
- [ ] 核心战斗机制已实施
- [ ] AI行为已实施
- [ ] VFX效果已创建
- [ ] 音频事件已定义

### 步骤4：集成

**目标**：集成战斗系统

**操作方法**：
1. 连接游戏玩法代码、AI、VFX和音频
2. 确保所有调音旋钮暴露且数据驱动
3. 验证功能与现有战斗系统配合工作

**检查清单**：
- [ ] 游戏玩法代码已连接
- [ ] AI已连接
- [ ] VFX已连接
- [ ] 音频已连接
- [ ] 调音旋钮已暴露
- [ ] 功能已验证

### 步骤5：验证

**目标**：验证战斗功能

**操作方法**：
1. 委托给QA测试员
2. 从验收标准编写测试用例
3. 测试设计中记录的所有边缘情况
4. 验证性能影响在预算内
5. 为发现的任何问题文件错误报告

**检查清单**：
- [ ] 测试用例已编写
- [ ] 边缘情况已测试
- [ ] 性能已验证
- [ ] 错误报告已文件

### 步骤6：签署

**目标**：签署战斗功能

**操作方法**：
1. 收集所有团队成员的结果
2. 报告功能状态（完成 / 需要工作 / 阻塞）
3. 列出任何未决问题及其指定负责人

**检查清单**：
- [ ] 团队结果已收集
- [ ] 功能状态已报告
- [ ] 未决问题已列出

## 常见错误

### 错误1：不设计战斗功能

**问题**：不设计战斗功能，直接实施

**修正**：必须先设计战斗功能，创建设计文档

### 错误2：不设计架构

**问题**：不设计代码架构，直接实施

**修正**：必须先设计代码架构，识别集成点

### 错误3：不集成系统

**问题**：不集成游戏玩法代码、AI、VFX和音频

**修正**：必须连接游戏玩法代码、AI、VFX和音频

### 错误4：不验证功能

**问题**：不验证战斗功能

**修正**：必须编写测试用例，测试边缘情况，验证性能

### 错误5：不签署功能

**问题**：不收集团队结果，不报告功能状态

**修正**：必须收集团队结果，报告功能状态，列出未决问题
