# Team Level

协调关卡设计团队：关卡设计师+叙事总监+世界构建者+艺术总监+系统设计师+QA测试员，完成区域/关卡创建。

## 适用场景

- 需要创建区域/关卡
- 需要协调关卡设计团队
- 需要编译关卡设计文档
- 需要生成关卡团队报告
- 需要定义关卡验收标准

## 操作步骤

### 步骤1：读取参数和收集上下文

**目标**：读取参数和收集上下文

**操作方法**：
1. 读取参数（目标关卡或区域，例如：教程、森林地牢、枢纽城镇、最终Boss竞技场）
2. 收集上下文（读取游戏概念、游戏支柱、现有关卡文档、相关叙事文档、区域/派系的世界构建文档）

**检查清单**：
- [ ] 参数已读取
- [ ] 游戏概念已读取
- [ ] 游戏支柱已读取
- [ ] 现有关卡文档已读取
- [ ] 叙事文档已读取
- [ ] 世界构建文档已读取

### 步骤2：叙事上下文

**目标**：定义叙事上下文

**操作方法**：
1. 委托给叙事总监（定义区域的叙事目的、识别关键角色和对话触发器、指定情感弧线）
2. 委托给世界构建者（提供区域的传说背景、定义环境叙事机会、指定影响游戏玩法的世界规则）

**检查清单**：
- [ ] 叙事目的已定义
- [ ] 关键角色已识别
- [ ] 情感弧线已指定
- [ ] 传说背景已提供
- [ ] 环境叙事已定义
- [ ] 世界规则已指定

### 步骤3：布局和遭遇设计

**目标**：设计空间布局和遭遇

**操作方法**：
1. 委托给关卡设计师（设计空间布局、定义节奏曲线、放置具有难度进度的遭遇、设计环境谜题或导航挑战、定义路标兴趣点和地标、指定入口/出口点和到相邻区域的连接）

**检查清单**：
- [ ] 空间布局已设计
- [ ] 节奏曲线已定义
- [ ] 遭遇已放置
- [ ] 导航挑战已设计
- [ ] 路标已定义
- [ ] 入口/出口已指定

### 步骤4：系统集成

**目标**：集成系统

**操作方法**：
1. 委托给系统设计师（指定敌人构成和遭遇公式、定义战利品表和奖励放置、平衡相对于预期玩家等级/装备的难度、设计任何区域特定机制或环境危害、指定资源分布）

**检查清单**：
- [ ] 敌人构成已指定
- [ ] 战利品表已定义
- [ ] 难度已平衡
- [ ] 区域机制已设计
- [ ] 资源分布已指定

### 步骤5：视觉方向

**目标**：定义视觉方向

**操作方法**：
1. 委托给艺术总监（定义区域的视觉主题和色彩调色板、指定灯光情绪和时间-of-day设置、列出所需艺术资产、定义视觉地标和视线、指定任何特殊VFX需求）

**检查清单**：
- [ ] 视觉主题已定义
- [ ] 色彩调色板已指定
- [ ] 灯光情绪已指定
- [ ] 资产需求已列出
- [ ] 视觉地标已定义
- [ ] VFX需求已指定

### 步骤6：QA规划

**目标**：规划QA测试

**操作方法**：
1. 委托给QA测试员（为关键路径编写测试用例、识别边界和边缘情况、为区域创建玩测检查清单、定义关卡完成的验收标准）

**检查清单**：
- [ ] 测试用例已编写
- [ ] 边界已识别
- [ ] 玩测检查清单已创建
- [ ] 验收标准已定义

### 步骤7：编译关卡设计文档

**目标**：编译关卡设计文档

**操作方法**：
1. 结合所有团队输出
2. 转换为关卡设计模板格式
3. 保存到 design/levels/[level-name].md

**检查清单**：
- [ ] 团队输出已结合
- [ ] 已转换为模板格式
- [ ] 已保存到设计目录

### 步骤8：输出摘要

**目标**：输出摘要

**操作方法**：
1. 区域概述
2. 遭遇计数
3. 估计资产列表
4. 叙事节拍
5. 任何跨团队依赖或开放问题

**检查清单**：
- [ ] 区域概述已输出
- [ ] 遭遇计数已输出
- [ ] 资产列表已估计
- [ ] 叙事节拍已输出
- [ ] 依赖和问题已列出

## 常见错误

### 错误1：不收集上下文

**问题**：不读取游戏概念、支柱、现有关卡文档

**修正**：必须收集完整上下文（游戏概念、支柱、现有关卡文档、叙事文档、世界构建文档）

### 错误2：不定义叙事上下文

**问题**：不定义叙事目的、情感弧线

**修正**：必须委托给叙事总监和世界构建者，定义叙事上下文

### 错误3：不设计空间布局

**问题**：不设计空间布局、节奏曲线、遭遇

**修正**：必须委托给关卡设计师，设计空间布局和遭遇

### 错误4：不集成系统

**问题**：不指定敌人构成、战利品表、难度平衡

**修正**：必须委托给系统设计师，集成系统

### 错误5：不规划QA测试

**问题**：不编写测试用例、不定义验收标准

**修正**：必须委托给QA测试员，规划QA测试
