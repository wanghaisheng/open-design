# World Building Concept

设计游戏世界观概念，包括体验目标、世界设定、叙事框架和视觉呈现，运用工业化世界观设计方法，从架构企划到内容写作，建立完整的世界观体系。

## 适用场景

- 新项目启动时建立世界观框架
- 需要明确游戏的情感体验和核心特色
- 从零开始构建游戏世界的历史、地理和人文
- 需要设计叙事结构和故事梗概
- 团队协作需要统一的世界观指导

## 操作步骤

### 步骤1：定义体验目标

**目标**：明确游戏的情感体验、设计目标和游戏特色

**操作方法**：
1. 设计目标（Design Goal）：
   - 用一句话描述游戏的核心定位
   - 示例："An uncompromising wilderness survival game full of science and magic"
2. 体验目标（Experience Goal）：
   - 描述玩家进入游戏后的感受
   - 示例："Enter a strange and unexplored world full of strange creatures, dangers, and surprises"
3. 游戏特色（Game Feature）：
   - 列出3-5个核心玩法特色
   - 示例："Gather resources to craft items and structures that match your survival style"
4. 建立灵感库：
   - 收集情绪关键词
   - 准备气氛图和参考素材
   - 组织快速设计Workshop

**检查清单**：
- [ ] 设计目标清晰简洁
- [ ] 体验目标具有感染力
- [ ] 游戏特色突出
- [ ] 灵感库已建立

### 步骤2：构建世界设定

**目标**：规划历史人文和空间版图，推动设计落地

**操作方法**：
1. 历史三问：
   - 有多个文明存在吗？发展到了哪个阶段？
   - 文明的原动力和终极目标是什么？
   - 玩家在文明中的位置？有哪些元素可以辅助感知？
2. 地图三问：
   - 地图是无缝的吗？各地图尺寸是多大？
   - 地图有几层？通行方式和障碍是什么？
   - 地图体验目标是什么？有哪些基本元素支撑这一目标？
3. 绘制时间轴：
   - 串起大事件
   - 写真正对叙事空间发生影响的事
   - 不要事无巨细，留足空间给故事写作
4. 视觉化呈现：
   - 绘制草图让大家对世界样貌有整体认知
   - 划分版块，不陷入细节纠结
   - 使用地图绘制工具（Inkarnate、OldMaps Online）

**检查清单**：
- [ ] 历史三问已回答
- [ ] 地图三问已回答
- [ ] 时间轴已绘制
- [ ] 视觉草图已完成

### 步骤3：设计叙事框架

**目标**：确定故事主体、故事梗概和叙事节奏

**操作方法**：
1. 确定故事主体：
   - 单机游戏：玩家直接扮演主角
   - 多人游戏：玩家有限自由度，避免叙事失控
   - 设计玩家选择的影响范围（任务结局、场景元素、收集进度）
2. 建置角色身份：
   - 用1-2个关键词定义身份和现状
   - 挖掘角色特质和能力
   - 确定核心体验
   - 示例：刺客信条 - 游走于历史阴影下的刺客
3. 编写故事梗概（Logline）：
   - 主角特质
   - 世界现状
   - 故事契机
   - 主要矛盾
   - 失败后果
   - 经典句式：当xxx之时，一个xxx的人，为阻止xxx发生，需要达成xxx
4. 设计叙事节奏：
   - 叙事切片：一共切几块，总时长
   - 叙事编排：每块多长，如何穿插
   - 叙事呈现：轻重分级，确定节奏

**检查清单**：
- [ ] 故事主体已确定
- [ ] 角色身份已建置
- [ ] 故事梗概已编写
- [ ] 叙事节奏已设计

### 步骤4：选择叙事载体

**目标**：根据打断程度和定制化程度选择合适的叙事载体

**操作方法**：
1. 重度叙事（打断操作体验，隐藏UI）：
   - CGI动画：片头、大章节衔接、外宣
   - Cutscene过场动画：引入新角色、新场景、预告战斗
   - Dialog对话镜头：单人/双人/多人对话
2. 中度叙事（暂时中断操作，隐藏部分UI）：
   - 脚本事件：伪装变身、潜入、护送、跟随
   - 面板对话：一般对话、分支对话、功能对话、交付对话
   - 观察镜头：展现线索、场景细节和指引方向
3. 轻度叙事（不干扰操作，不隐藏UI）：
   - 场景交互：可探索物配备观察镜头
   - 旁白与补充对白：旅行对白、NPC旁白
   - 界面滤镜：回忆杀、透视观察、艺术风格化
4. 周边系统支持：
   - 叙事碎片
   - 独立叙事位面
   - 蕴含情感的道具或象征物

**检查清单**：
- [ ] 叙事载体已分类
- [ ] 打断程度已评估
- [ ] 定制化需求已明确
- [ ] 周边系统已规划

### 步骤5：设计呈现工具

**目标**：规划调度、摄影、表演指导和脚本设计工具

**操作方法**：
1. 调度工具：
   - 场景编辑器
   - 事件编辑器
   - NPC编辑器
   - META AI
2. 摄影工具：
   - 剧情编辑器（如UE4 Sequencer）
   - 对话编辑器
3. 表演指导：
   - 动捕
   - 配音
4. 脚本设计：
   - 台词
   - 补充性文本

**检查清单**：
- [ ] 调度工具已规划
- [ ] 摄影工具已规划
- [ ] 表演指导已规划
- [ ] 脚本设计已规划

### 步骤6：建立协作体系

**目标**：建立团队知识库和在线协作文档

**操作方法**：
1. 团队知识库：
   - 使用Notion、TiddlyWiki等工具
   - 权限分明
   - 沉淀、归档和分享
2. 在线协作文档：
   - 创意涌现讨论
   - 素材分享和处理
   - 集中团队智慧
3. 协作内容：
   - 文本向内容
   - 视觉素材
   - 设计讨论

**检查清单**：
- [ ] 知识库已建立
- [ ] 协作文档已启用
- [ ] 权限已设置
- [ ] 协作流程已明确

## 操作注意事项

### 工业化思维

1. 专业化分工：
   - 架构企划：历史、神话、幻想故事
   - 内容写作：电影学院经验模版
   - 互动体验与视觉呈现：电影学院资源
   - 协调统筹：项目管理
2. 从小作坊到工业流水线：
   - 制作流程拆环
   - 各司其职
   - 精细化运作

### 工具链整合

1. 概念工具：
   - 地图查询及绘制（Inkarnate、OldMaps Online）
   - 历史查询及绘制（全历史、Excel时间轴）
2. 写作工具：
   - 文本写作（Typora）
   - 大纲编排（幕布、Dynalist）
   - 思维导图（XMind、TheBrain）
   - 灵感记录（Bear、Ulysses）
   - 剧本格式（剧云、Final Draft）
3. 呈现工具：
   - 场景编辑器
   - 事件编辑器
   - 剧情编辑器

### 叙事风格选择

1. 写实向叙事：
   - 人物与环境模拟真实
   - 在环境中讲故事
   - 常见于影视、历史、战争题材
2. 夸张化叙事：
   - 人物与环境自成一格
   - 跳出环境塑造人物
   - 常见于动漫、MOBA、卡牌对战

## 输出格式

### 世界观概念文档

- 体验目标（设计目标、体验目标、游戏特色）
- 世界设定（历史三问、地图三问、时间轴、视觉草图）
- 叙事框架（故事主体、角色身份、故事梗概、叙事节奏）
- 叙事载体（重度/中度/轻度叙事选择）
- 呈现工具规划
- 协作体系设计

### 视觉素材

- 气氛图
- 地图草图
- 角色概念图
- 时间轴图表

## 协作协议

### 向谁汇报

- lead-designer：世界观与整体体验目标对齐
- creative-director：世界观与创意方向对齐
- narrative-director：叙事框架与叙事设计对接

### 协调对象

- concept-artist：概念设计与视觉呈现
- level-designer：世界设定与关卡设计结合
- writer：叙事框架与内容写作
- ui-designer：叙事载体与UI设计

## 常见错误

### 错误1：过度考据

**问题**：陷入繁重的历史考据工作，拖慢设计节奏

**修正**：前期只回答关键问题，细节留待故事写作阶段；历史是养料，游戏是基于历史的再创作

### 错误2：细节过早

**问题**：在前期纠结地貌植被建筑风格等细节

**修正**：只做整体认知和版块划分，为故事写作留足空间

### 错误3：忽略工具

**问题**：停留在文字，不使用视觉化工具

**修正**：学会用视觉回答问题，使用地图绘制工具、时间轴工具等

### 错误4：叙事失控

**问题**：多人游戏中玩家自由度过大，叙事线变得复杂难以控制

**修正**：限制玩家选择的影响范围，避免做无法挽回的选项，提前埋下暗示
