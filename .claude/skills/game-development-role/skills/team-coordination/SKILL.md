# Team Coordination

协调游戏开发团队，包括团队沟通、任务分配、冲突解决和团队建设，确保团队高效协作。

## 适用场景

- 需要组建团队
- 需要规划任务
- 需要日常协调
- 需要定期回顾
- 需要团队发展

## 操作步骤

### 步骤1：团队组建

**目标**：组建团队

**操作方法**：
1. 组建团队（团队成员选择）
2. 定义角色（角色定义）
3. 建立规范（团队规范）
4. 培训团队（团队培训）

**检查清单**：
- [ ] 团队已组建
- [ ] 角色已定义
- [ ] 规范已建立
- [ ] 团队已培训

### 步骤2：任务规划

**目标**：规划任务

**操作方法**：
1. 分解任务（任务分解）
2. 分配任务（能力匹配、负载平衡、优先级管理）
3. 设定目标（目标设定）
4. 制定计划（计划制定）

**检查清单**：
- [ ] 任务已分解
- [ ] 任务已分配
- [ ] 目标已设定
- [ ] 计划已制定

### 步骤3：日常协调

**目标**：日常协调

**操作方法**：
1. 每日站会（每日进度同步）
2. 进度跟踪（进度跟踪）
3. 问题解决（问题解决）
4. 风险管理（风险管理）

**检查清单**：
- [ ] 每日站会已完成
- [ ] 进度已跟踪
- [ ] 问题已解决
- [ ] 风险已管理

### 步骤4：定期回顾

**目标**：定期回顾

**操作方法**：
1. 迭代回顾（迭代回顾会议）
2. 团队评估（团队评估）
3. 经验总结（经验总结）
4. 持续改进（持续改进）

**检查清单**：
- [ ] 迭代回顾已完成
- [ ] 团队评估已完成
- [ ] 经验已总结
- [ ] 持续改进已进行

### 步骤5：团队发展

**目标**：团队发展

**操作方法**：
1. 技能培训（技能培训）
2. 团队建设（团队建设）
3. 绩效评估（绩效评估）
4. 团队激励（团队激励）

**检查清单**：
- [ ] 技能培训已完成
- [ ] 团队建设已完成

- [ ] 团队激励已完成

## 协作协议

### 向谁汇报

- **producer**：团队状态和进度
- **project-manager**：任务分配和计划
- **team-lead**：团队协调

### 协调对象

- **所有团队成员**：团队沟通
- **hr**：团队建设
- **stakeholders**：团队状态

## 常见错误

### 错误1：不组建团队

**问题**：不组建团队，直接分配任务

**修正**：必须先组建团队，定义角色，建立规范

### 错误2：不规划任务

**问题**：不分解任务，不分配任务

**修正**：必须分解任务，分配任务，设定目标，制定计划

### 错误3：不日常协调

**问题**：不每日站会，不跟踪进度

**修正**：必须每日站会，跟踪进度，解决问题，管理风险

### 错误4：不定期回顾

**问题**：不迭代回顾，不团队评估

**修正**：必须迭代回顾，团队评估，经验总结，持续改进

### 错误5：不团队发展

**问题**：不技能培训，不团队建设

**修正**：必须技能培训，团队建设，绩效评估，团队激励
