# Dialogue Writing

编写角色对话和台词，创造生动自然的角色对话，增强角色个性和故事表达。

## 适用场景

- 需要编写角色对话
- 需要设计对话结构
- 需要设计对话分支
- 需要优化对话质量
- 需要测试对话效果

## 操作步骤

### 步骤1：角色分析

**目标**：分析角色特征

**操作方法**：
1. 分析角色性格（角色性格表达）
2. 分析角色背景（背景故事、角色关系）
3. 分析角色关系（与其他角色的关系）

**检查清单**：
- [ ] 角色性格已分析
- [ ] 角色背景已分析
- [ ] 角色关系已分析

### 步骤2：对话设计

**目标**：设计对话结构

**操作方法**：
1. 设计对话内容（主要对话、次要对话、背景对话、互动对话）
2. 设计对话结构（对话流程、对话分支、对话触发、对话结束）
3. 设计对话分支（分支条件、分支结果）

**检查清单**：
- [ ] 对话内容已设计
- [ ] 对话结构已设计
- [ ] 对话分支已设计

### 步骤3：对话编写

**目标**：编写对话内容

**操作方法**：
1. 编写对话内容（使用 Word/Google Docs/Final Draft/Celtx）
2. 编写对话分支（使用 Articy/Unity Dialog System）
3. 编写对话触发（触发条件、触发事件）

**检查清单**：
- [ ] 对话内容已编写
- [ ] 对话分支已编写
- [ ] 对话触发已编写

### 步骤4：对话优化

**目标**：优化对话质量

**操作方法**：
1. 优化对话自然性（避免生硬、符合日常习惯）
2. 优化对话一致性（与角色一致、语言风格一致、性格表达一致）
3. 优化对话表达力（情感表达准确、信息传达清晰）

**检查清单**：
- [ ] 对话自然性已优化
- [ ] 对话一致性已优化
- [ ] 对话表达力已优化

### 步骤5：对话测试

**目标**：测试对话效果

**操作方法**：
1. 测试对话流程（对话流程是否顺畅）
2. 测试对话分支（分支是否正确）
3. 测试对话效果（沉浸感、游戏体验）

**检查清单**：
- [ ] 对话流程已测试
- [ ] 对话分支已测试
- [ ] 对话效果已测试

## 输出格式

### 对话文档
- 对话内容
- 对话结构
- 对话分支
- 触发条件
- 对话测试报告

## 操作注意事项

### 角色一致性

1. **对话与角色一致**：确保对话与角色一致
2. **语言风格一致**：确保语言风格一致
3. **性格表达一致**：确保性格表达一致

### 自然流畅

1. **对话自然流畅**：确保对话自然流畅
2. **避免生硬**：避免生硬
3. **符合日常习惯**：符合日常习惯

### 表达力强

1. **对话表达力强**：确保对话表达力强
2. **情感表达准确**：确保情感表达准确
3. **信息传达清晰**：确保信息传达清晰

### 沉浸感

1. **对话增强沉浸感**：确保对话增强沉浸感
2. **不打断游戏体验**：不打断游戏体验
3. **符合游戏氛围**：符合游戏氛围

## 协作协议

### 向谁汇报

- **narrative-director**：对话质量和故事一致性
- **writer**：对话内容和风格
- **localization-lead**：多语言适配

### 协调对象

- **character-designer**：角色背景和性格
- **qa-lead**：对话测试和反馈
- **voice-actor**：语音录制和表达

## 常见错误

### 错误1：不分析角色

**问题**：不分析角色性格、背景、关系，直接编写对话

**修正**：必须先分析角色性格、背景、关系

### 错误2：对话不自然

**问题**：对话生硬，不符合日常习惯

**修正**：必须优化对话自然性，避免生硬

### 错误3：对话不一致

**问题**：对话与角色不一致，语言风格不一致

**修正**：必须优化对话一致性，确保与角色一致

### 错误4：不测试对话

**问题**：不测试对话流程、分支、效果

**修正**：必须测试对话流程、分支、效果

### 错误5：打断游戏体验

**问题**：对话过长或过于频繁，打断游戏体验

**修正**：必须确保对话不打断游戏体验，符合游戏氛围
