# Game Rule Analysis

游戏规则分析技能，分析游戏规则的核心结构、一级结构和二级结构。

## 适用场景

- 分析游戏规则结构
- 解构游戏设计
- 理解设计意图
- 优化游戏规则
- 进行设计迭代

## 操作步骤

### 步骤1：识别核心结构

**目标**：识别游戏规则的核心结构（主谓宾）

**操作方法**：
1. **核心结构定义**
   - 精简到无法去掉任何一个字
   - 决定设计最本质用途
   - 改动任何一个字都会导致方向性改变

2. **核心结构识别方法**
   - 确定最本质的设计目的
   - 识别符合设计目的的规则
   - 去掉后仍符合目的的规则
   - 去掉后目的产生偏移的规则
   - 重复直到无法再精简

3. **核心结构示例**
   - "杀死小兵会获得金币和经验奖励"
   - "野怪会刷新"
   - "毒圈会收缩"

**检查清单**：
- [ ] 核心结构已识别
- [ ] 本质目的已确定
- [ ] 精简已完成
- [ ] 核心结构已验证

### 步骤2：识别一级结构

**目标**：识别游戏规则的一级结构（形容词、定语）

**操作方法**：
1. **一级结构定义**
   - 核心结构的修饰和补充
   - 将规则补充得更完整
   - 同类产品的重要区分点
   - 游戏性的重要来源

2. **一级结构识别方法**
   - 什么样的主体对什么样的客体产生作用
   - 如何产生作用、触发机制是什么
   - 生效的过程是怎样的
   - 是否让游戏更好玩了

3. **一级结构示例**
   - "对小兵完成最后一击的玩家，获得经验和金钱"
   - "野怪在死后经过固定时间刷新"
   - "毒圈会每隔一段时间缓慢收缩"

**检查清单**：
- [ ] 一级结构已识别
- [ ] 修饰已补充
- [ ] 差异化已识别
- [ ] 游戏性已评估

### 步骤3：识别二级结构

**目标**：识别游戏规则的二级结构（分支处理、"如果"）

**操作方法**：
1. **二级结构定义**
   - 特定情况下的分支处理
   - 针对某种情况的说明
   - 逻辑完整性的补全
   - 设计的延伸

2. **二级结构识别方法**
   - 针对的情况是什么？在什么条件下？
   - 去掉是不是不影响正常体验？
   - 是否是补丁式设计？
   - 是否是细节优化？

3. **二级结构示例**
   - "如果击杀的是己方的小兵，则对方不会获得任何金钱和经验"
   - "当毒圈范围小于一定值时，将不再继续缩小"
   - "如果怪物刷新时，范围内有英雄或眼或召唤物，则无法刷新"

**检查清单**：
- [ ] 二级结构已识别
- [ ] 条件已明确
- [ ] 逻辑完整性已验证
- [ ] 细节已优化

### 步骤4：结构层次分析

**目标**：分析三个结构层次的关系和作用

**操作方法**：
1. **层次关系分析**
   - 核心结构：定海神针，设计原型
   - 一级结构：游戏性来源，体验品质
   - 二级结构：体验改良，细节调优

2. **层次作用分析**
   - 核心结构：品类创新
   - 一级结构：品类差异化
   - 二级结构：优化层面

3. **层次应用分析**
   - 改变核心结构：品类创新
   - 改变一级结构：品类差异化
   - 改变二级结构：特定问题解法

**检查清单**：
- [ ] 层次关系已分析
- [ ] 层次作用已分析
- [ ] 层次应用已分析
- [ ] 结构层次已理解

### 步骤5：应用结构分析

**目标**：将结构分析应用到设计或反推

**操作方法**：
1. **设计时应用**
   - 按核心结构、一级结构、二级结构顺序
   - 逐层补充规则
   - 避免被信息吞噬
   - 保持条理清晰

2. **反推时应用**
   - 从二级结构、一级结构逐层剥离
   - 剩下核心结构
   - 得出根本性意义
   - 明确改动层级

3. **迭代时应用**
   - 明确改动的是哪个层级
   - 核心结构改动：重新定义
   - 一级结构改动：差异化
   - 二级结构改动：优化

**检查清单**：
- [ ] 设计应用已完成
- [ ] 反推应用已完成
- [ ] 迭代应用已完成
- [ ] 结构分析已掌握

## 操作注意事项

### 核心结构

1. **精简到底**：精简到无法再精简
2. **本质目的**：抓住最本质的设计目的
3. **方向性**：改动会改变方向
4. **原型作用**：作为设计原型

### 一级结构

1. **补充完整**：将规则补充完整
2. **差异化**：体现产品差异化
3. **游戏性**：决定游戏性品质
4. **合理性**：确保设计合理

### 二级结构

1. **分支处理**：处理特定情况
2. **逻辑完整**：补全逻辑完整性
3. **细节优化**：优化体验细节
4. **补丁作用**：弥补设计不足

## 常见错误

### 错误1：混淆层次

**问题**：混淆三个结构层次，分析混乱

**修正**：明确三个层次定义，正确识别

### 错误2：过度精简

**问题**：过度精简，失去设计本质

**修正**：精简到无法再精简为止

### 错误3：忽视二级结构

**问题**：忽视二级结构，细节不完善

**修正**：重视二级结构，完善细节

### 错误4：不应用分析

**问题**：只分析不应用到设计

**修正**：将分析应用到设计和迭代

### 错误5：层次混乱

**问题**：层次混乱，改动层级不明确

**修正**：明确改动的是哪个层级
