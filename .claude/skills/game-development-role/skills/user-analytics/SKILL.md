# User Analytics

用户分析技能，分析用户行为和数据以优化产品体验。

## 适用场景

- 分析用户行为
- 追踪用户旅程
- 识别用户模式
- 优化用户体验
- 提高用户留存

## 操作步骤

### 步骤1：定义分析目标

**目标**：明确用户分析的目标

**操作方法**：
1. 确定分析问题
2. 定义关键指标
3. 设置分析范围
4. 确定数据需求
5. 制定分析计划

**检查清单**：
- [ ] 分析问题已确定
- [ ] 关键指标已定义
- [ ] 分析范围已设置
- [ ] 数据需求已确定
- [ ] 分析计划已制定

### 步骤2：收集用户数据

**目标**：收集用户行为数据

**操作方法**：
1. 配置数据追踪
2. 定义事件追踪
3. 设置用户属性
4. 收集会话数据
5. 验证数据质量

**检查清单**：
- [ ] 数据追踪已配置
- [ ] 事件追踪已定义
- [ ] 用户属性已设置
- [ ] 会话数据已收集
- [ ] 数据质量已验证

### 步骤3：分析用户行为

**目标**：分析用户行为模式

**操作方法**：
1. 分析用户旅程
2. 识别行为模式
3. 分段用户群体
4. 分析用户路径
5. 发现行为异常

**检查清单**：
- [ ] 用户旅程已分析
- [ ] 行为模式已识别
- [ ] 用户群体已分段
- [ ] 用户路径已分析
- [ ] 行为异常已发现

### 步骤4：生成洞察

**目标**：从数据中生成洞察

**操作方法**：
1. 分析数据趋势
2. 识别关键发现
3. 生成假设
4. 验证假设
5. 提取可行动洞察

**检查清单**：
- [ ] 数据趋势已分析

**目标**：基于分析提出改进建议

**操作方法**：
1. 识别改进机会
2. 制定改进方案
3. 评估改进影响
4. 优先级排序
5. 制定实施计划

**检查清单**：

## 输出格式

### 用户分析报告
```markdown
## 用户分析报告

### 关键指标
- 日活跃用户: 10,000
- 平均会话时长: 15分钟
- 用户留存率: 40%

### 关键发现
1. 新用户在首次3分钟内流失率最高
2. 完成教程的用户留存率提高30%
3. 社交功能使用率低

### 改进建议
1. 优化新手教程
2. 增加社交功能引导
3. 提高早期用户参与度
```

## 操作注意事项

### 数据隐私

1. **用户同意**：确保用户同意数据收集
2. **匿名化**：对敏感数据进行匿名化
3. **合规性**：遵守数据保护法规
4. **最小化**：只收集必要的数据

### 分析方法

1. **定量分析**：使用定量方法分析数据
2. **定性分析**：结合定性分析理解原因
3. **对比分析**：使用对比分析发现差异
4. **趋势分析**：分析数据趋势和变化

### 洞察验证

1. **假设驱动**：基于假设进行分析
2. **数据验证**：用数据验证假设
3. **A/B测试**：通过A/B测试验证洞察
4. **持续迭代**：持续迭代分析结果

- data-engineer：数据收集和存储
- ux-designer：用户体验优化

## 常见错误

### 错误1：数据质量问题

**问题**：数据不准确或不完整

**修正**：验证数据质量，确保数据准确完整

### 错误2：分析目标不明确

**问题**：分析目标模糊，难以得出结论

**修正**：明确分析目标和关键指标

### 错误3：忽视隐私

**问题**：忽视用户隐私，违反法规

**修正**：遵守数据保护法规，保护用户隐私

### 错误4：过度解读数据

**问题**：过度解读数据，得出错误结论

**修正**：谨慎解读数据，验证结论

### 错误5：不行动

**问题**：分析结果不转化为行动

**修正**：将分析结果转化为可行动的改进
