# Project Planning

规划游戏项目，包括项目里程碑、时间表、资源分配和风险管理，确保项目按时交付。

## 适用场景

- 需要确定项目范围
- 需要制定里程碑
- 需要制定时间表
- 需要分配资源
- 需要管理风险

## 操作步骤

### 步骤1：项目启动

**目标**：启动项目

**操作方法**：
1. 确定项目目标（项目目标）
2. 定义项目范围（项目范围、项目边界、项目约束）
3. 识别项目干系人（干系人识别）
4. 建立项目团队（团队建立）

**检查清单**：
- [ ] 项目目标已确定
- [ ] 项目范围已定义
- [ ] 干系人已识别
- [ ] 项目团队已建立

### 步骤2：项目规划

**目标**：规划项目

**操作方法**：
1. 分解项目任务（工作分解结构WBS）
2. 估算任务时间（任务估算）
3. 分配项目资源（人员、设备、预算、时间）
4. 制定时间表（甘特图、关键路径）

**检查清单**：
- [ ] 项目任务已分解
- [ ] 任务时间已估算
- [ ] 项目资源已分配
- [ ] 时间表已制定

### 步骤3：项目执行

**目标**：执行项目

**操作方法**：
1. 执行项目计划（计划执行）
2. 监控项目进度（进度监控）
3. 管理项目风险（风险管理）
4. 协调项目团队（团队协调）

**检查清单**：
- [ ] 项目计划已执行
- [ ] 项目进度已监控
- [ ] 项目风险已管理
- [ ] 项目团队已协调

### 步骤4：项目控制

**目标**：控制项目

**操作方法**：
1. 跟踪项目进度（进度跟踪）
2. 控制项目变更（变更控制）
3. 报告项目状态（状态报告）
4. 调整项目计划（计划调整）

**检查清单**：
- [ ] 项目进度已跟踪
- [ ] 项目变更已控制
- [ ] 项目状态已报告
- [ ] 项目计划已调整

### 步骤5：项目收尾

**目标**：收尾项目

**操作方法**：
1. 验收项目交付（交付验收）
2. 总结项目经验（经验总结）
3. 释放项目资源（资源释放）
4. 关闭项目（项目关闭）

**检查清单**：
- [ ] 项目交付已验收
- [ ] 项目经验已总结
- [ ] 项目资源已释放
- [ ] 项目已关闭

## 协作协议

### 向谁汇报

- **producer**：项目计划状态
- **executive-producer**：项目里程碑
- **stakeholders**：项目进度

### 协调对象

- **progress-tracking**：进度跟踪
- **resource-manager**：资源分配
- **risk-manager**：风险管理

## 常见错误

### 错误1：不确定项目目标

**问题**：不确定项目目标，直接规划

**修正**：必须先确定项目目标，定义项目范围

### 错误2：不分解任务

**问题**：不分解项目任务，无法估算

**修正**：必须分解项目任务，估算任务时间

### 错误3：不分配资源

**问题**：不分配项目资源

**修正**：必须分配项目资源（人员、设备、预算、时间）

### 错误4：不监控进度

**问题**：不监控项目进度

**修正**：必须监控项目进度，管理项目风险

### 错误5：不调整计划

**问题**：不调整项目计划

**修正**：必须跟踪项目进度，控制项目变更，调整项目计划
