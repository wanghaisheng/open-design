# Retrospective

通过分析已完成工作、速度、阻塞和模式生成冲刺或里程碑回顾。产生可操作的洞察以用于下一次迭代。

## 回顾要素

### 1. 指标分析 (Metrics Analysis)
- 任务完成率：计划vs实际任务
- 速度趋势：速度变化趋势
- 错误统计：发现和修复的错误
- 未计划任务：添加的未计划任务
- 提交统计：代码提交统计

### 2. 速度分析 (Velocity Analysis)
- 计划vs完成：每个冲刺的计划vs完成
- 完成率：任务完成率
- 趋势：改善/稳定/下降
- 调整估算：当前速度下的调整估算

### 3. 估算准确性 (Estimation Accuracy)
- 高估任务：最被高估的任务
- 低估任务：最被低估的任务
- 方差：估算方差
- 原因分析：方差原因分析
- 整体准确性：整体估算准确性

### 4. 遗留分析 (Carryover Analysis)
- 遗留任务：未完成的任务
- 原始冲刺：任务的原始冲刺
- 遗留次数：任务被遗留的次数
- 原因：未完成的原因
- 行动：完成/削减/重新设计

### 5. 技术债务状态 (Technical Debt Status)
- TODO计数：当前TODO计数
- FIXME计数：当前FIXME计数
- HACK计数：当前HACK计数
- 趋势：增长/稳定/减少
- 关注领域：需要关注的领域

## 回顾原则

### 1. 诚实具体
- 诚实回顾
- 具体观察
- 使用数据和示例
- 避免模糊陈述

### 2. 系统性
- 关注系统性问题
- 不指责个人
- 识别根本原因
- 提供系统性解决方案

### 3. 可操作性
- 限制行动项到3-5个
- 每个行动项有负责人
- 每个行动项有截止日期
- 可衡量和可验证

### 4. 持续改进
- 检查之前行动项
- 识别重复问题
- 跟踪速度趋势
- 持续改进流程

## 回顾流程

### 1. 确定范围
- 读取参数确定范围
- 冲刺回顾或里程碑回顾
- 确定时间范围

### 2. 读取计划
- 读取冲刺计划或里程碑定义
- 提取计划任务
- 提取估算工作量
- 提取负责人和目标

### 3. 读取Git日志
- 读取时间范围内的Git日志
- 理解实际提交
- 理解提交时间

### 4. 扫描任务完成情况
- 比较计划vs实际交付
- 识别完成的任务
- 识别修改的任务
- 识别遗留的任务
- 识别添加的任务
- 识别删除的任务

### 5. 扫描技术债务趋势
- 计数当前TODO/FIXME/HACK
- 与之前冲刺计数比较
- 评估技术债务趋势

### 6. 读取之前回顾
- 检查之前行动项是否已处理
- 识别重复问题
- 评估速度趋势

### 7. 生成回顾
- 生成指标分析
- 生成速度分析
- 生成估算准确性
- 生成遗留分析
- 生成技术债务状态
- 生成行动项
- 生成流程改进

### 8. 保存回顾
- 保存到适当位置
- 创建目录（如果不存在）

### 9. 输出摘要
- 输出完成率
- 输出速度趋势
- 输出主要阻塞
- 输出重要行动项
