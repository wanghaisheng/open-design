# Concept Creation

创建视觉概念，将游戏设计转化为视觉概念，为游戏提供视觉方向和灵感。

## 适用场景

- 需要将游戏设计转化为视觉概念
- 需要探索视觉方向
- 需要收集灵感和参考
- 需要创建情绪板
- 需要提供视觉方向指导

## 操作步骤

### 步骤1：需求理解

**目标**：理解游戏设计和视觉目标

**操作方法**：
1. 理解游戏设计（读取GDD）
2. 理解角色/环境功能
3. 确定视觉目标

**检查清单**：
- [ ] 游戏设计已理解
- [ ] 角色/环境功能已理解
- [ ] 视觉目标已确定

### 步骤2：资料收集

**目标**：收集视觉参考和灵感

**操作方法**：
1. 收集参考图（使用 PureRef/Pinterest）
2. 收集灵感（竞品分析、艺术参考）
3. 建立情绪板（使用 Miro/Figma）

**检查清单**：
- [ ] 参考图已收集
- [ ] 灵感已收集
- [ ] 情绪板已建立

### 步骤3：草图创作

**目标**：绘制概念草图

**操作方法**：
1. 绘制概念草图（使用平板/手写板）
2. 探索不同方案
3. 确定方向

**检查清单**：
- [ ] 概念草图已绘制
- [ ] 不同方案已探索
- [ ] 方向已确定

### 步骤4：概念细化

**目标**：细化和完善概念

**操作方法**：
1. 细化概念（使用 Photoshop/Procreate/Krita）
2. 添加细节
3. 完善设计

**检查清单**：
- [ ] 概念已细化
- [ ] 细节已添加
- [ ] 设计已完善

### 步骤5：概念交付

**目标**：交付概念图和文档

**操作方法**：
1. 交付概念图（草图、渲染图、细节图）
2. 编写说明文档
3. 提供技术规格

**检查清单**：
- [ ] 概念图已交付
- [ ] 说明文档已编写
- [ ] 技术规格已提供

## 输出格式

### 概念图
- 概念草图
- 概念渲染图
- 概念细节图

### 文档
- 概念说明文档
- 参考图集
- 技术规格文档

## 操作注意事项

### 创意性

1. **独特创意**：确保概念有独特创意
2. **创新思维**：使用创新思维
3. **视觉冲击**：创造视觉冲击

### 可行性

1. **技术可行**：确保技术可实现
2. **资源可行**：确保资源可行
3. **时间可行**：确保时间可行

### 一致性

1. **风格一致**：确保风格一致
2. **色彩一致**：确保色彩一致
3. **氛围一致**：确保氛围一致

### 沟通性

1. **清晰表达**：清晰表达概念
2. **易于理解**：易于理解
3. **便于实现**：便于实现

## 协作协议

### 向谁汇报

- **art-director**：概念创作和视觉方向
- **game-designer**：概念与设计的一致性
- **technical-artist**：技术可行性

### 协调对象

- **concept-artist**：概念艺术创作
- **character-designer**：角色概念需求
- **environment-designer**：环境概念需求

## 常见错误

### 错误1：不理解游戏设计

**问题**：不读取GDD，不理解游戏设计

**修正**：必须读取GDD，理解游戏设计

### 错误2：不收集参考

**问题**：不收集参考图和灵感

**修正**：必须收集参考图和灵感，建立情绪板

### 错误3：只探索单一方案

**问题**：只绘制单一草图，不探索不同方案

**修正**：应探索不同方案，确定最佳方向

### 错误4：不检查可行性

**问题**：不检查概念的技术可行性

**修正**：必须检查技术可行性、资源可行性、时间可行性

### 错误5：风格不一致

**问题**：概念风格与游戏整体风格不一致

**修正**：确保风格一致，符合游戏整体风格
