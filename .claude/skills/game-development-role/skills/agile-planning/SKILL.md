# Agile Planning

敏捷规划技能，使用敏捷方法论进行项目规划和迭代管理。

## 适用场景

- 制定敏捷开发计划
- 管理迭代和冲刺
- 规划用户故事和任务
- 进行敏捷估算
- 管理敏捷看板

## 操作步骤

### 步骤1：制定敏捷计划

**目标**：创建敏捷开发计划

**操作方法**：
1. 确定项目愿景和目标
2. 创建产品待办列表
3. 定义发布计划
4. 确定迭代周期
5. 设置冲刺目标

**检查清单**：
- [ ] 项目愿景已确定
- [ ] 产品待办列表已创建
- [ ] 发布计划已定义
- [ ] 迭代周期已确定
- [ ] 冲刺目标已设置

### 步骤2：规划冲刺

**目标**：规划单个冲刺

**操作方法**：
1. 选择冲刺待办事项
2. 进行故事点估算
3. 分配任务到团队成员
4. 确定冲刺目标
5. 识别风险和依赖

**检查清单**：
- [ ] 冲刺待办事项已选择
- [ ] 故事点已估算
- [ ] 任务已分配
- [ ] 冲刺目标已确定
- [ ] 风险已识别

### 步骤3：管理迭代

**目标**：管理迭代过程

**操作方法**：
1. 每日站会同步进度
2. 跟踪任务完成情况
3. 处理阻碍和问题
4. 调整计划（如需要）
5. 保持团队沟通

**检查清单**：
- [ ] 每日站会已进行
- [ ] 进度已跟踪
- [ ] 阻碍已处理
- [ ] 计划已调整
- [ ] 团队沟通已保持

### 步骤4：回顾和改进

**目标**：回顾迭代并改进流程

**操作方法**：
1. 进行冲刺回顾
2. 收集团队反馈
3. 识别改进机会
4. 制定改进行动计划
5. 跟踪改进实施

**检查清单**：
- [ ] 冲刺回顾已完成
- [ ] 反馈已收集
- [ ] 改进机会已识别
- [ ] 行动计划已制定
- [ ] 改进已跟踪

### 步骤5：敏捷估算

**目标**：进行敏捷估算

**操作方法**：
1. 使用故事点估算
2. 进行规划扑克
3. 考虑复杂度和不确定性
4. 调整估算（基于历史数据）
5. 记录估算依据

**检查清单**：
- [ ] 故事点已估算
- [ ] 规划扑克已使用
- [ ] 复杂度已考虑
- [ ] 估算已调整
- [ ] 依据已记录

- 开发C: 性能优化

## 操作注意事项

### 敏捷原则

1. **响应变化**：欢迎变化，即使在开发后期
2. **频繁交付**：定期交付可工作的软件
3. **团队协作**：业务人员和开发人员必须天天合作
4. **可持续开发**：保持可持续的开发速度

### 估算技巧

1. **相对估算**：使用相对估算而非绝对时间
2. **团队共识**：通过讨论达成共识
3. **历史数据**：参考历史数据调整估算
4. **不确定性**：考虑不确定性因素

### 冲刺管理

1. **保护冲刺**：避免在冲刺中添加新任务
2. **每日同步**：保持每日站会简短有效
3. **透明沟通**：保持进度和问题透明
4. **持续改进**：每个迭代都要改进流程

## 协作协议

### 向谁汇报

- producer：项目进度和计划
- technical-director：技术可行性

### 协调对象

- team-lead：团队协调
- project-manager：项目管理
- qa-lead：质量保证计划

## 常见错误

### 错误1：冲刺目标不明确

**问题**：冲刺目标模糊，难以衡量

**修正**：确保冲刺目标具体、可衡量

### 错误2：过度承诺

**问题**：在冲刺中承诺过多任务

**修正**：基于团队能力合理规划

### 错误3：忽略回顾

**问题**：不进行冲刺回顾，失去改进机会

**修正**：每个冲刺都要进行回顾和改进

### 错误4：估算不准确

**问题**：估算偏差大，影响计划

**修正**：使用历史数据，持续改进估算

### 错误5：缺乏沟通

**问题**：团队沟通不足，进度不透明

**修正**：保持每日站会，确保信息同步
