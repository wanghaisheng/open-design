# Prototype

快速原型工作流程。跳过正常标准快速验证游戏概念或机制。产生可丢弃代码和结构化原型报告。

## 适用场景

- 需要验证游戏概念
- 需要验证游戏机制
- 需要测试设计假设
- 需要测试技术可行性
- 需要快速迭代

## 操作步骤

### 步骤1：读取概念描述

**目标**：读取概念描述

**操作方法**：
1. 读取概念描述（概念文档、设计文档）
2. 识别核心问题（验证目标、验证范围）
3. 明确问题陈述（问题定义、成功标准）

**检查清单**：
- [ ] 概念描述已读取
- [ ] 核心问题已识别
- [ ] 问题陈述已明确

### 步骤2：读取项目上下文

**目标**：读取项目上下文

**操作方法**：
1. 读取 CLAUDE.md（技术栈、引擎配置）
2. 理解技术栈（引擎、语言、工具）
3. 理解引擎和语言（引擎版本、语言特性）

**检查清单**：
- [ ] CLAUDE.md 已读取
- [ ] 技术栈已理解
- [ ] 引擎和语言已理解

### 步骤3：创建原型计划

**目标**：创建原型计划

**操作方法**：
1. 定义最小可行原型（最小范围、最小功能）
2. 定义核心问题（要回答的问题）
3. 定义最小代码需求（必需代码、可跳过代码）
4. 定义可跳过内容（占位符、简化）

**检查清单**：
- [ ] 最小可行原型已定义
- [ ] 核心问题已定义
- [ ] 最小代码需求已定义
- [ ] 可跳过内容已定义

### 步骤4：创建原型目录

**目标**：创建原型目录

**操作方法**：
1. 创建目录：prototypes/[概念名称]/
2. 隔离环境（独立目录、独立文件）
3. 独立文件（不依赖生产代码）

**检查清单**：
- [ ] 原型目录已创建
- [ ] 隔离环境已设置
- [ ] 独立文件已创建

### 步骤5：实现原型

**目标**：实现原型

**操作方法**：
1. 在隔离目录实现（独立开发）
2. 添加原型标记（原型标识、可丢弃标记）
3. 放宽编码标准（硬编码、占位符、跳过错误处理）
4. 快速开发（最简单方法、快速迭代）

**检查清单**：
- [ ] 原型已实现
- [ ] 原型标记已添加
- [ ] 编码标准已放宽
- [ ] 快速开发已完成

### 步骤6：测试概念

**目标**：测试概念

**操作方法**：
1. 运行原型（原型执行）
2. 观察行为（行为观察、数据收集）
3. 收集数据（可测量数据、感受评估）
4. 感受评估（主观评估、具体描述）

**检查清单**：
- [ ] 原型已运行
- [ ] 行为已观察
- [ ] 数据已收集
- [ ] 感受已评估

### 步骤7：生成原型报告

**目标**：生成原型报告

**操作方法**：
1. 生成报告（假设、方法、结果）
2. 保存到 prototypes/[概念名称]/REPORT.md
3. 记录结果（具体观察、可测量数据）
4. 提供建议（继续/转向/终止）

**检查清单**：
- [ ] 报告已生成
- [ ] 报告已保存
- [ ] 结果已记录
- [ ] 建议已提供

### 步骤8：输出摘要

**目标**：输出原型摘要

**操作方法**：
1. 输出核心问题（验证目标）
2. 输出结果（实际结果）
3. 输出建议（继续/转向/终止）
4. 链接到完整报告（报告路径）

**检查清单**：
- [ ] 核心问题已输出
- [ ] 结果已输出
- [ ] 建议已输出
- [ ] 完整报告已链接

## 输出格式

### 原型报告
```markdown
## 原型报告：[概念名称]

### 假设
[我们期望为真——我们打算回答的问题]

### 方法
[我们构建了什么，花了多长时间，我们采取了什么捷径]

### 结果
[实际发生了什么——具体观察，而非意见]

### 指标
[测试期间收集的任何可测量数据]
- 帧时间：[如果相关]
- 感受评估：[主观但具体——"200ms延迟时响应感觉迟缓"而非"感觉不好"]
- 玩家操作计数：[如果相关]
- 迭代计数：[多少次尝试才使其工作]

### 建议：[继续 / 转向 / 终止]

[一段解释建议并提供证据的段落]

### 如果继续
[生产质量实现需要改变什么]
- 架构需求
- 性能目标
- 从原始设计的范围调整
- 预估生产工作量

### 如果转向
[结果建议的替代方向]

### 如果终止
[为什么这个概念不起作用，我们应该做什么]

### 经验教训
[影响其他系统或未来工作的发现]
```

## 操作注意事项

### 速度优先

1. **快速开发**：确保快速开发
2. **快速验证**：确保快速验证
3. **快速迭代**：确保快速迭代

### 隔离性

1. **隔离目录**：确保隔离目录
2. **不依赖生产**：确保不依赖生产
3. **可丢弃代码**：确保可丢弃代码

### 最小化

1. **最小可行**：确保最小可行
2. **最小代码**：确保最小代码
3. **最小范围**：确保最小范围

### 可测量

1. **可测量结果**：确保可测量结果
2. **可验证假设**：确保可验证假设
3. **可量化数据**：确保可量化数据

## 常见错误

### 错误1：不读取概念描述

**问题**：不读取概念描述、不识别核心问题

**修正**：必须读取概念描述，识别核心问题，明确问题陈述

### 错误2：不创建原型计划

**问题**：不定义最小可行原型、不定义核心问题

**修正**：必须定义最小可行原型，核心问题，最小代码需求

### 错误3：不隔离原型环境

**问题**：不创建隔离目录、不设置独立文件

**修正**：必须创建隔离目录，设置隔离环境，创建独立文件

### 错误4：不放宽编码标准

**问题**：不添加原型标记、不放宽编码标准

**修正**：必须添加原型标记，放宽编码标准，快速开发

### 错误5：不测试概念

**问题**：不运行原型、不收集数据

**修正**：必须运行原型，观察行为，收集数据
