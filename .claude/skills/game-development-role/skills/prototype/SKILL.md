# Prototype

快速原型工作流程。跳过正常标准快速验证游戏概念或机制。产生可丢弃代码和结构化原型报告。

## 原型要素

### 1. 核心问题 (Core Question)
- 概念验证：验证游戏概念
- 机制验证：验证游戏机制
- 假设测试：测试设计假设
- 可行性测试：测试技术可行性

### 2. 原型计划 (Prototype Plan)
- 最小可行原型：定义最小可行原型
- 核心问题：回答的核心问题
- 最小代码：回答问题所需的最小代码
- 可跳过内容：可以跳过的内容

### 3. 原型实现 (Prototype Implementation)
- 隔离目录：在隔离目录中实现
- 放宽标准：有意放宽编码标准
- 快速开发：使用最简单的有效方法
- 占位符资产：使用占位符资产

### 4. 原型测试 (Prototype Testing)
- 运行原型：运行原型
- 观察行为：观察原型行为
- 收集数据：收集可测量数据
- 感受评估：感受评估

## 原型标准

### 放宽的标准
- 硬编码值：自由硬编码值
- 占位符资产：使用占位符资产
- 跳过错误处理：跳过错误处理
- 最简单方法：使用最简单的有效方法
- 复制代码：复制代码而非从生产导入

### 重要约束
- 不从生产导入：原型代码绝不能从生产源文件导入
- 不导入原型：生产代码绝不能从原型目录导入
- 从头实现：如果建议继续，生产实现必须从头编写
- 时间框定：原型总工作量应时间框定为 1-3 天等效工作

## 原型原则

### 1. 速度优先
- 快速开发
- 快速验证
- 快速迭代

### 2. 隔离性
- 隔离目录
- 不依赖生产
- 可丢弃代码

### 3. 最小化
- 最小可行
- 最小代码
- 最小范围

### 4. 可测量
- 可测量结果

## 原型流程

### 1. 读取概念描述
- 读取概念描述
- 识别核心问题
- 明确问题陈述

### 2. 读取项目上下文
- 读取 CLAUDE.md
- 理解技术栈
- 理解引擎和语言

### 3. 创建原型计划
- 定义最小可行原型
- 定义核心问题
- 定义最小代码需求
- 定义可跳过内容

### 4. 创建原型目录
- 创建目录：prototypes/[概念名称]/
- 隔离环境
- 独立文件

### 5. 实现原型
- 在隔离目录实现
- 添加原型标记
- 放宽编码标准
- 快速开发

### 6. 测试概念
- 运行原型
- 观察行为
- 收集数据
- 感受评估

### 7. 生成原型报告
- 生成报告
- 保存到 prototypes/[概念名称]/REPORT.md
- 记录结果
- 提供建议

### 8. 输出摘要
- 输出核心问题
- 输出结果
- 输出建议
- 链接到完整报告

## 输出格式

### 原型报告
```markdown
## 原型报告：[概念名称]

### 假设
[我们期望为真——我们打算回答的问题]

### 方法
[我们构建了什么，花了多长时间，我们采取了什么捷径]

### 结果
[实际发生了什么——具体观察，而非意见]

### 指标
[测试期间收集的任何可测量数据]
- 帧时间：[如果相关]
- 感受评估：[主观但具体——"200ms延迟时响应感觉迟缓"而非"感觉不好"]
- 玩家操作计数：[如果相关]
- 迭代计数：[多少次尝试才使其工作]

### 建议：[继续 / 转向 / 终止]

[一段解释建议并提供证据的段落]

### 如果继续
[生产质量实现需要改变什么]
- 架构需求
- 性能目标
- 从原始设计的范围调整
- 预估生产工作量

### 如果转向
[结果建议的替代方向]

### 如果终止
[为什么这个概念不起作用，我们应该做什么]

### 经验教训
[影响其他系统或未来工作的发现]
```
