# Hybrid Casual Design

混合休闲游戏设计技能，设计超休闲核心玩法与RPG/策略深度系统的融合。

## 适用场景

- 设计混合休闲游戏
- 融合超休闲和深度系统
- 提升用户留存和LTV
- 降低获客成本（CPI）
- 构建长期生态系统

## 操作步骤

### 步骤1：设计超休闲核心玩法

**目标**：设计低门槛、高CPI潜力的核心玩法

**操作方法**：
1. **核心循环设计**
   - 定义简单直观的核心操作
   - 确保即时反馈和满足感
   - 控制单局时长（1-3分钟）
   - 设计清晰的胜负条件

2. **关卡设计**
   - 短平快关卡（1分钟内）
   - 损失厌恶设计（中途失败清零）
   - 难度渐进曲线
   - 可量产关卡设计

3. **UI/UX设计**
   - 极简主义UI
   - 减少用户选择
   - 无缝衔接关卡
   - 及时反馈（进度条）

**检查清单**：
- [ ] 核心循环已设计
- [ ] 关卡系统已设计
- [ ] UI/UX已设计
- [ ] 低门槛已验证

### 步骤2：设计RPG/策略深度系统

**目标**：设计支撑长期留存的深度系统

**操作方法**：
1. **收集系统**
   - 设计丰富的收集对象（英雄、武器、皮肤、卡牌）
   - 确保收集目标清晰
   - 设计稀有度系统
   - 设计获取途径

2. **成长系统**
   - 设计成长路径（升级、技能树、装备）
   - 可视化成长反馈
   - 设计成长节奏
   - 平衡成长曲线

3. **社交/策略系统**
   - **PVP设计**：实时或异步对抗
   - **PVE设计**：联盟、公会合作
   - **策略深度**：克制系统（剪刀石头布）
   - 社交驱动设计

**检查清单**：
- [ ] 收集系统已设计
- [ ] 成长系统已设计
- [ ] 社交系统已设计
- [ ] 策略深度已设计

### 步骤3：设计经济循环

**目标**：设计支撑变现的经济循环

**操作方法**：
1. **基础循环**
   - 游戏产出货币
   - 货币购买道具/角色/皮肤
   - 使用新物品继续游戏获得更多货币

2. **进阶循环**
   - 游戏产出货币
   - 货币购买宝箱/礼包
   - 随机开出道具和角色
   - 利用随机性延长循环

3. **广告设计**
   - 视频激励广告作为最大货币产出口
   - 设计广告场景（复活、增益、试用）
   - 平衡广告和内购

**检查清单**：
- [ ] 基础循环已设计
- [ ] 进阶循环已设计
- [ ] 广告设计已完成
- [ ] 经济平衡已验证

### 步骤4：融合与平衡

**目标**：确保超休闲核心和深度系统自然融合

**操作方法**：
1. **渐进引入**
   - 早期专注超休闲核心
   - 逐步引入深度系统
   - 设计引导教程
   - 控制学习曲线

2. **分工明确**
   - 超休闲核心：降低CPI、提供即时乐趣
   - 深度系统：提升LTV、提供长线追求
   - 确保两者不冲突

3. **数据验证**
   - 监控CPI数据
   - 监控留存数据
   - 监控LTV数据
   - 迭代优化

**检查清单**：
- [ ] 引入节奏已设计
- [ ] 分工已明确
- [ ] 数据监控已设置
- [ ] 融合已验证

## 操作注意事项

### 融合原则

1. **自然融合**：深度系统不应破坏超休闲核心体验
2. **渐进引入**：深度系统逐步引入，避免过早复杂化
3. **明确分工**：超休闲和深度系统各司其职

### 平衡设计

1. **经济平衡**：确保经济循环健康可持续
2. **难度平衡**：确保难度曲线合理
3. **时间平衡**：确保单局时长合理

### 数据驱动

1. **CPI监控**：持续监控获客成本
2. **留存监控**：持续监控用户留存
3. **LTV监控**：持续监控用户价值

## 常见错误

### 错误1：过早引入深度系统

**问题**：在早期就引入复杂系统，吓跑休闲玩家

**修正**：渐进引入深度系统，先让玩家体验超休闲核心

### 错误2：系统冲突

**问题**：深度系统破坏超休闲核心体验

**修正**：确保深度系统不干扰核心玩法

### 错误3：经济失衡

**问题**：经济循环设计不当，导致通货膨胀或通货紧缩

**修正**：设计平衡的产出和消耗机制

### 错误4：忽视CPI

**问题**：只关注深度系统，忽视获客成本

**修正**：始终关注CPI，确保低门槛

### 错误5：过度复杂

**问题**：系统过于复杂，失去混合休闲的简洁优势

**修正**：保持系统简洁，避免过度设计
