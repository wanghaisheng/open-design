# Asset List Generation

从GDD或分镜剧本中提取游戏资产清单，包括角色、背景、道具、特效、声音等元素，生成可执行的生产计划，帮助美术团队明确任务、估算预算和追踪进度。

## 适用场景

- 新项目启动时从GDD生成初始资产清单
- 外包项目需要提供详细的资产需求清单
- 美术团队需要明确的任务分配和进度追踪
- 需要估算美术预算和制作周期
- 从分镜剧本提取视觉元素需求

## 操作步骤

### 步骤1：准备和扫描文档

**目标**：全面理解GDD或分镜剧本，标记所有视觉和听觉元素

**操作方法**：
1. 收集GDD（概念、关卡设计、美术参考）和分镜PDF/图像
2. 全局扫描，标记"高频元素"（如主角出现率>50%场景）
3. 使用工具辅助：
   - Adobe Acrobat 高亮标记
   - AI工具（ChatGPT）上传文档提取元素
4. 记录文档来源页码或帧号

**检查清单**：
- [ ] GDD/分镜文档已收集
- [ ] 高频元素已标记
- [ ] 来源位置已记录
- [ ] 工具辅助已使用

### 步骤2：逐页/逐帧元素提取

**目标**：像看电影一样，从每页GDD/分镜中提取可见/可闻元素

**操作方法**：
1. 视觉元素提取：
   - 角色：主角、敌人（动作：闲置、攻击）
   - 背景：环境（森林、城市，日夜变体）
   - 道具：武器、家具（可交互）
   - 灯光/特效：火光、爆炸、粒子
2. 听觉元素提取：
   - 声音：脚步、爆炸音
   - 音乐：背景音乐、战斗音乐
   - 语音：角色对话
3. 界面元素提取：
   - UI按钮、血条、地图
4. 使用普通表达分类（角色、背景、道具、特效、声音、屏幕显示）
5. **资产格式决策（Asset Format Decision）**
   - 适合video（视频）：过场/展示、背景循环、复杂合成
   - 适合image_sequence（序列帧）：透明背景动画、引擎内混合、插值/节奏需精细控制
   - 适合image（图片）：静态视觉、纹理贴图
   - 常见场景映射：过场/剧情片段→video，战斗FX→image_sequence，UI图标→image
   - 记录元数据：分辨率、FPS/时长、格式、依赖关系

**检查清单**：
- [ ] 视觉元素已提取
- [ ] 听觉元素已提取
- [ ] 界面元素已提取
- [ ] 使用普通表达分类
- [ ] 资产格式已决策
- [ ] 元数据已记录

### 步骤3：分类与量化

**目标**：将提取的元素分类整理，估算数量和规格

**操作方法**：
1. 用普通表达分组：
   - 角色模型+动画
   - 背景纹理/场景
   - 道具静态模型
   - 特效粒子/着色器
   - 声音音频剪辑
2. 估算变体：
   - 角色：不同动作（站立、跑步、攻击）
   - 背景：日夜变体、不同季节
   - 道具：LOD变体（低/高细节）
   - 特效：慢镜头变体
3. 量化规格：
   - 分辨率（4K、2K、1K）
   - 多边形数量（10k、50k）
   - 动画帧数
   - 音频时长

**检查清单**：
- [ ] 元素已分类
- [ ] 变体已估算
- [ ] 规格已量化
- [ ] 专业术语对应明确

### 步骤4：优先级与依赖排序

**目标**：确定制作顺序，优化资源分配

**操作方法**：
1. 定义优先级：
   - P0（高）：核心关卡资产（如主角）
   - P1（中）：重要关卡资产
   - P2（低）：可选资产
2. 标记依赖关系：
   - 剑需英雄手部骨骼匹配
   - 特效需角色动作先完成
3. 评估复用率：
   - 通用背景多场景用（复用率>70%）
   - 通用道具可复用
4. 识别关键路径（最长的依赖链）

**检查清单**：
- [ ] 优先级已定义
- [ ] 依赖关系已标记
- [ ] 复用率已评估
- [ ] 关键路径已识别

### 步骤5：生成资产清单表格

**目标**：输出结构化的资产清单，便于团队使用

**操作方法**：
1. 设计标准列结构：
   - ID（唯一标识）
   - 类别（普通表达）
   - 描述/名称
   - 数量
   - 变体/规格（分辨率/细节）
   - 优先级（P0/P1/P2）
   - 依赖资产
   - 状态（规划/制作/审核）
   - 来源（GDD页/分镜帧）
   - 备注（LOD/平台）
2. 填充表格数据
3. 导出为Excel/Google Sheets格式
4. 添加公式自动合计数量/预算

**检查清单**：
- [ ] 表格结构完整
- [ ] 数据已填充
- [ ] 格式已导出
- [ ] 公式已添加

### 步骤6：分享与迭代

**目标**：与团队确认清单，持续更新

**操作方法**：
1. 分享清单给美术团队
2. 收集反馈和补充
3. 迭代优化清单
4. 建立更新机制（每周更新）
5. 与GDD变更同步

**检查清单**：
- [ ] 清单已分享
- [ ] 反馈已收集
- [ ] 清单已迭代
- [ ] 更新机制已建立

## 操作注意事项

### 普通人视角

1. 使用电影一帧类比：
   - 角色 → 演员
   - 背景 → 布景
   - 道具 → 道具
   - 灯光 → 灯光师设置
   - 特效 → 特效师添加
   - 声音 → 配乐和音效
   - 屏幕 → 字幕/HUD
2. 提供专业术语对照表，便于团队沟通

### 游戏与电影区别

1. 游戏重点：可交互资产（LOD变体、多动画）
2. 电影重点：线性序列（特效、镜头匹配）
3. 游戏输出：动态清单（热更新支持）
4. 电影输出：静态清单（渲染农场提交）

### 规模适应

1. 小项目：手动提取（1人1天）
2. 大项目：自动化+外包
3. 结合命名规范，直通美术管道

- 阶段划分（概念→建模→上色→动作→测试）
- 时间线规划（Gantt图）
- 资源分配
- 里程碑设置

## 协作协议

### 向谁汇报

- lead-artist：资产清单与美术能力对齐
- producer：清单与预算和周期对齐
- asset-planner：清单与生产计划对接

### 协调对象

- concept-artist：概念设计需求
- character-artist：角色制作任务
- environment-artist：环境制作任务
- ui-designer：UI资产需求

## 常见错误

### 错误1：文档不详尽

**问题**：GDD或分镜缺少细节，导致清单不完整

**修正**：补充访谈策划，迭代2-3轮；参考Moodboard添加细节

### 错误2：数量爆炸

**问题**：清单项数过多，超出团队承受能力

**修正**：优先核心关卡，复用率>70%；分阶段制作

### 错误3：跨团队冲突

**问题**：不同团队对资产需求理解不一致

**修正**：共享在线表格，版本控制；定期同步会议

### 错误4：电影线性 vs 游戏分支

**问题**：用线性思维处理游戏分支玩法

**修正**：分镜树状结构，GDD分支玩法估算
