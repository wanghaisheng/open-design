# Script Review

评审剧本质量，检查剧本的逻辑性、一致性和表达力，确保剧本质量达标。

## 适用场景

- 需要检查剧本逻辑性
- 需要检查剧本一致性
- 需要检查剧本表达力
- 需要检查剧本可玩性
- 需要跟踪改进进度

## 操作步骤

### 步骤1：剧本阅读

**目标**：阅读和理解剧本

**操作方法**：
1. 阅读完整剧本（从头到尾完整阅读）
2. 理解故事结构（故事结构分析）
3. 理解角色设定（角色设定分析）

**检查清单**：
- [ ] 完整剧本已阅读
- [ ] 故事结构已理解
- [ ] 角色设定已理解

### 步骤2：评审检查

**目标**：执行剧本评审

**操作方法**：
1. 检查逻辑性（故事逻辑、角色逻辑、情节逻辑、对话逻辑）
2. 检查一致性（角色一致性、叙事一致性、风格一致性、世界观一致性）
3. 检查表达力（对话表达、情感表达、冲突表达、主题表达）
4. 检查可玩性（游戏整合、互动设计、玩家体验、技术实现）

**检查清单**：
- [ ] 逻辑性已检查
- [ ] 一致性已检查
- [ ] 表达力已检查
- [ ] 可玩性已检查

### 步骤3：问题记录

**目标**：记录评审问题

**操作方法**：
1. 记录逻辑问题（问题描述、问题位置）
2. 记录一致性问题（问题描述、问题位置）
3. 记录表达问题（问题描述、问题位置）
4. 记录可玩性问题（问题描述、问题位置）

**检查清单**：
- [ ] 逻辑问题已记录
- [ ] 一致性问题已记录
- [ ] 表达问题已记录
- [ ] 可玩性问题已记录

### 步骤4：反馈建议

**目标**：提供评审反馈

**操作方法**：
1. 提供评审报告（评审概述、问题列表）
2. 提供改进建议（改进方法、优化方向）
3. 提供优先级排序（高/中/低优先级）

**检查清单**：
- [ ] 评审报告已提供
- [ ] 改进建议已提供
- [ ] 优先级已排序

### 步骤5：跟踪改进

**目标**：跟踪改进进度

**操作方法**：
1. 跟踪改进进度（改进状态、改进时间）
2. 验证改进效果（改进验证、质量确认）
3. 关闭问题（问题关闭、问题归档）

**检查清单**：
- [ ] 改进进度已跟踪
- [ ] 改进效果已验证
- [ ] 问题已关闭

## 输出格式

### 评审报告
- 评审概述
- 问题列表
- 改进建议
- 优先级排序
- 评审总结

## 常见错误

### 错误1：不阅读完整剧本

**问题**：不阅读完整剧本、不理解故事结构

**修正**：必须阅读完整剧本，理解故事结构，理解角色设定

### 错误2：不检查逻辑性

**问题**：不检查故事逻辑、不检查角色逻辑

**修正**：必须检查故事逻辑，角色逻辑，情节逻辑，对话逻辑

### 错误3：不记录问题

**问题**：不记录逻辑问题、不记录一致性问题

**修正**：必须记录逻辑问题，一致性问题，表达问题，可玩性问题

### 错误4：不提供改进建议

**问题**：不提供评审报告、不提供改进建议

**修正**：必须提供评审报告，改进建议，优先级排序

### 错误5：不跟踪改进进度

**问题**：不跟踪改进进度、不验证改进效果

**修正**：必须跟踪改进进度，验证改进效果，关闭问题
