## 适用场景

- 设计新游戏的 VIP 等级体系
- 优化现有游戏的 VIP 系统
- 设计用户分层和付费激励体系
- 平衡付费玩家和非付费玩家体验
- 设计 VIP 特权权益和积分获取机制

---

## 操作步骤

### 步骤1：读取参数和PRD

**目标**：读取游戏PRD和VIP系统设计参数

**操作方法**：
1. 读取游戏PRD文档
2. 读取游戏类型（MMORPG、MOBA、卡牌收集、模拟经营、休闲养成等）
3. 读取核心玩法和主要循环
4. 读取目标用户群体特征、付费意愿和能力预估
5. 读取主要商业化目标和核心付费点
6. 读取游戏对平衡性的要求（强竞技游戏 vs PVE/养成）
7. 读取游戏世界观或主题

**检查清单**：
- [ ] PRD文档已读取
- [ ] 游戏类型已确定
- [ ] 核心玩法已理解
- [ ] 目标用户已分析
- [ ] 商业化目标已明确
- [ ] 平衡性要求已确认
- [ ] 游戏世界观已了解

---

### 步骤2：分析设计目标

**目标**：明确VIP系统的设计目标和定位

**操作方法**：
1. 基于商业化目标确定VIP系统主要目标（提升LTV、鼓励持续消费、提供身份认同、提供额外价值）
2. 确定VIP特权定位（虚荣/身份类、便利/效率类、概率/结果类、功能/内容类）
3. 评估游戏平衡性要求对特权设计的限制
4. 分析目标用户付费能力和消费层级
5. 确定VIP系统与核心玩法的关联方式

**检查清单**：
- [ ] 设计目标已明确
- [ ] 特权定位已确定
- [ ] 平衡性限制已评估
- [ ] 用户付费能力已分析
- [ ] 与核心玩法关联已明确

---

### 步骤3：设计VIP等级体系

**目标**：设计VIP等级数量和积分要求

**操作方法**：
1. 确定VIP等级数量（建议5-10个等级）
2. 设计VIP积分获取方式（累计消费金额或累计消费付费货币）
3. 设定付费货币与VIP积分的兑换比例
4. 设计每个等级的累计积分要求
5. 确保积分阶梯合理分布（初期较低，后期指数或跳跃式增长）
6. 估算达到最高等级所需的真实人民币消费
7. 验证消费金额与目标用户付费能力和商业目标的匹配度

**检查清单**：
- [ ] 等级数量已确定
- [ ] 积分获取方式已设计
- [ ] 兑换比例已设定
- [ ] 各等级积分要求已设计
- [ ] 积分阶梯分布合理
- [ ] 最高等级消费已估算
- [ ] 与目标用户匹配度已验证

---

### 步骤4：设计VIP特权权益

**目标**：设计各VIP等级的特权权益

**操作方法**：
1. **特权分类设计**：基于游戏平衡性要求确定特权主要偏向类别
   - **虚荣/身份类**：专属头像框、聊天气泡、特殊徽章、称号、皮肤
   - **便利/效率类**：体力上限提升、资源产出加成、扫荡功能、加速建造
   - **功能/内容类**：专属副本、商店折扣、额外背包空间、自动战斗
2. 基于特权定位设计特权类别（身份展示类、每日福利类、效率加成类、功能解锁类、商店折扣类、额外资源获取类、特殊服务类）
3. 为每个VIP等级设计具体特权内容
4. 确保特权随着等级提升，价值感和吸引力同步提升
5. 设计特权价值梯度（低等级基础特权，高等级稀有特权）
6. 区分一次性奖励和持续性加成特权
7. 评估特权对游戏平衡性的影响
8. 确保特权不破坏核心玩法公平性（特别是竞技游戏）
9. **特权分配与价值梯度**
   - 将设计好的特权分配到不同的VIP等级上
   - 确保每个等级的提升都有明确可感知的价值
   - 设计里程碑式的特权只在较高VIP等级解锁
10. **鲸鱼用户考量**
    - 最高等级VIP的特权是否能满足"鲸鱼用户"的识别和激励需求

**检查清单**：
- [ ] 特权类别已设计
- [ ] 各等级特权已设计
- [ ] 价值梯度已确保
- [ ] 一次性/持续性特权已区分
- [ ] 平衡性影响已评估
- [ ] 核心玩法公平性已保障

---

### 步骤5：设计VIP积分规则

**目标**：设计VIP积分的获取和清零规则

**操作方法**：
1. 设计VIP积分的获取规则（付费即获得或最低消费门槛）
2. 确定VIP积分是否累计永久有效
3. 设计VIP积分的清零规则（建议不清零，驱动累计消费）
4. 设计VIP积分的显示方式（当前等级、当前积分、离下一级所需积分）
5. 设计VIP等级的展示方式（玩家头像旁、排行榜、聊天频道等）

**检查清单**：
- [ ] 积分获取规则已设计
- [ ] 积分有效期已确定
- [ ] 清零规则已设计
- [ ] 积分显示方式已设计
- [ ] 等级展示方式已设计

---

### 步骤6：评估经济平衡

**目标**：评估VIP系统对游戏经济的影响

**操作方法**：
1. 分析效率加成特权对货币/资源总产出的影响
2. 分析资源赠送特权对经济循环的影响
3. 评估VIP系统如何驱动持续、阶梯式消费
4. 分析VIP系统对免费玩家和付费玩家体验的影响
5. 评估最高等级VIP对鲸鱼用户的吸引力
6. 设计经济调控策略（如调整积分要求、修改特权内容）

**检查清单**：
- [ ] 效率加成影响已分析
- [ ] 资源赠送影响已分析
- [ ] 消费驱动机制已评估
- [ ] 免费付费玩家体验已平衡
- [ ] 鲸鱼用户吸引力已评估
- [ ] 经济调控策略已设计

## 操作注意事项

### 平衡性原则

1. **避免Pay-to-Win**：在竞技游戏中，特权不应直接影响核心战斗平衡
2. **区分特权类型**：优先使用虚荣/身份类特权，谨慎使用效率/概率类特权
3. **保护免费玩家**：确保非付费玩家能够获得完整的游戏体验
4. **透明度**：特权效果和获取方式应清晰透明

### 用户体验

1. **价值感知**：每个等级的提升都应有明确可感知的价值
2. **激励梯度**：特权价值应随着等级提升而增强
3. **奖励感**：获得VIP等级和领取特权时应有强烈的奖励感
4. **清晰展示**：VIP状态和特权应清晰展示在游戏内

### 经济平衡

1. **产出控制**：效率加成特权应有合理的产出限制
2. **通胀预防**：设计货币回收点防止经济通胀
3. **分层激励**：设计不同付费层级的消费激励
4. **数据监控**：建立完善的数据监控体系

### 商业化协同

1. **目标一致**：VIP系统应与游戏商业化目标一致
2. **付费驱动**：设计持续、阶梯式的消费驱动机制
3. **鲸鱼满足**：最高等级应满足鲸鱼用户的身份和消费需求
4. **长期留存**：通过VIP系统提升用户长期留存

## 常见错误

### 错误1：特权设计影响游戏平衡性

**问题**：在竞技游戏中设计影响核心战斗平衡的特权

**修正**：严格区分特权类型，在竞技游戏中只使用虚荣/身份类特权，避免使用影响战斗结果的效率/概率类特权

### 错误2：等级划分不合理

**问题**：等级数量过多或过少，积分阶梯分布不合理

**修正**：合理设计等级数量（5-10个），确保积分阶梯初期较低、后期指数或跳跃式增长，形成清晰的价值梯度

### 错误3：积分要求与目标用户不匹配

**问题**：最高等级的积分要求与目标用户付费能力不匹配

**修正**：基于目标用户付费能力和商业目标，合理设计积分要求和消费金额估算

### 错误4：特权价值梯度不清晰

**问题**：等级提升时特权价值感不明显，缺乏激励

**修正**：确保每个等级的提升都有明确可感知的价值，特权价值随着等级提升而增强

### 错误5：缺乏经济平衡评估

**问题**：未评估VIP系统对游戏经济的影响

**修正**：全面评估效率加成和资源赠送特权对经济循环的影响，设计经济调控策略

### 错误6：UI/UX展示不清晰

**问题**：VIP状态和特权展示不清晰，影响用户体验

**修正**：设计清晰的VIP界面和游戏内展示方式，确保玩家能够直观了解VIP系统和特权内容
