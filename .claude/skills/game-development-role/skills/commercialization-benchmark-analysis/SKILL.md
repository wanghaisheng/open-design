## 适用场景

- 分析成功产品的商业化/货币体系设计
- 提取商业化设计原则和最佳实践
- 为新产品提供商业化设计参考
- 竞品商业化策略分析
- 货币体系设计原理分析

---

## 操作步骤

### 步骤1：选择标杆产品

**目标**：选择要分析的标杆产品

**操作方法**：
1. 确定分析目标（货币体系、商业化模式、付费设计等）
2. 选择行业内的成功产品作为标杆
3. 收集标杆产品的基本信息
   - 产品名称
   - 上线时间
   - 目标用户
   - 商业化模式
   - 市场表现
4. 确定分析的深度和范围

**检查清单**：
- [ ] 分析目标已确定
- [ ] 标杆产品已选择
- [ ] 产品基本信息已收集
- [ ] 分析深度已确定

---

### 步骤2：分析货币体系设计

**目标**：分析标杆产品的货币体系设计原理

**操作方法**：
1. 识别货币类型
   - 硬通货（付费货币）
   - 软通货（游戏内免费货币）
   - 特殊货币（限定用途货币）
2. 分析货币获取途径
   - 免费获取途径
   - 付费获取途径
   - 活动获取途径
3. 分析货币消耗途径
   - 主要消耗场景
   - 消耗频率
   - 消耗价值
4. 分析货币间兑换关系
   - 兑换比例
   - 兑换限制
   - 兑换时机
5. 分析多货币体系的好处
   - 区分获取途径与货币性质
   - 满足不同消费需求
   - 促进玩家留存活跃
   - 提供经济平衡调控灵活性
   - 利用心理效应

**检查清单**：
- [ ] 货币类型已识别
- [ ] 获取途径已分析
- [ ] 消耗途径已分析
- [ ] 兑换关系已分析
- [ ] 多货币体系好处已分析

---

### 步骤3：分析商业化设计原则

**目标**：提取标杆产品的商业化设计原则

**操作方法**：
1. 分析付费点设计
   - 付费点的位置
   - 付费点的自然度
   - 付费点的价值感
2. 分析付费模式
   - 一次性付费
   - 订阅制
   - 内购制
   - 混合模式
3. 分析付费转化策略
   - 新手引导
   - 首充优惠
   - 限时活动
   - 社交驱动
4. 分析付费用户分层
   - 免费用户
   - 小额付费用户
   - 中等付费用户
   - 大额付费用户
5. 分析免费和付费体验平衡
   - 非付费玩家体验
   - 付费玩家优势
   - 公平性考虑

**检查清单**：
- [ ] 付费点设计已分析
- [ ] 付费模式已分析
- [ ] 转化策略已分析
- [ ] 用户分层已分析
- [ ] 体验平衡已分析

---

### 步骤4：分析货币体系优缺点

**目标**：分析标杆产品货币体系的优缺点和适用场景

**操作方法**：
1. 分析优点
   - 经济平衡性
   - 玩家留存
   - 商业化效果
   - 用户体验
2. 分析缺点
   - 复杂度
   - 理解门槛
   - 潜在问题
3. 分析适用场景
   - 适合的游戏类型
   - 适合的目标用户
   - 适合的市场环境
4. 分析可复制性
   - 哪些设计可以复制
   - 哪些设计需要调整
   - 哪些设计不适合复制

**检查清单**：
- [ ] 优点已分析
- [ ] 缺点已分析
- [ ] 适用场景已分析
- [ ] 可复制性已分析

---

### 步骤5：提炼可复用的设计模式

**目标**：提炼可复用的商业化设计模式

**操作方法**：
1. 提炼货币体系设计模式
   - 多货币体系设计原则
   - 货币获取/消耗平衡
   - 货币兑换设计
2. 提炼付费转化模式
   - 首充转化模式
   - 活动转化模式
   - 社交转化模式
3. 提炼用户分层模式
   - 用户分层标准
   - 分层服务设计
   - 分层激励设计
4. 提炼长期留存模式
   - 长期目标设计
   - 持续激励设计
   - 社区驱动设计

**检查清单**：
- [ ] 货币体系模式已提炼
- [ ] 付费转化模式已提炼
- [ ] 用户分层模式已提炼
- [ ] 长期留存模式已提炼

---

### 步骤6：为新产品提供设计建议

**目标**：基于标杆分析，为新产品提供商业化设计建议

**操作方法**：
1. 分析新产品特点
   - 游戏类型
   - 目标用户
   - 商业化目标
   - 市场环境
2. 选择适用的设计模式
   - 从标杆产品中选择适用的模式
   - 根据新产品特点进行调整
3. 设计货币体系
   - 确定货币类型
   - 设计获取/消耗途径
   - 设计兑换关系
4. 设计付费转化策略
   - 设计付费点
   - 设计转化漏斗
   - 设计激励措施
5. 设计用户分层服务
   - 确定分层标准
   - 设计分层服务
   - 设计分层激励

**检查清单**：
- [ ] 新产品特点已分析
- [ ] 适用模式已选择
- [ ] 货币体系已设计
- [ ] 转化策略已设计
- [ ] 分层服务已设计

## 操作注意事项

### 分析原则

1. **客观公正**：基于实际数据进行分析，避免主观偏见
2. **全面性**：从多个维度分析商业化设计，避免遗漏重要方面
3. **具体性**：分析要具体，避免泛泛而谈
4. **可操作性**：提炼的设计模式要具有可操作性

### 标杆产品选择原则

1. **成功性**：选择市场上成功的标杆产品
2. **相关性**：选择与新产品类型相关的标杆产品
3. **代表性**：选择具有代表性的标杆产品
4. **可分析性**：选择有足够公开信息的标杆产品

### 设计模式提炼原则

1. **普适性**：提炼的设计模式要具有普适性
2. **可调整性**：设计模式要能够根据不同产品进行调整
3. **有效性**：设计模式要经过市场验证，具有有效性
4. **创新性**：在复制的基础上进行创新，避免完全照搬

### 新产品设计建议原则

1. **适配性**：设计建议要适配新产品的特点
2. **可行性**：设计建议要具有可行性
3. **差异化**：在借鉴的基础上形成差异化
4. **迭代性**：设计建议要支持后续迭代优化

## 常见错误

### 错误1：标杆产品选择不当

**问题**：选择了不相关或不成功的标杆产品

**修正**：选择市场上成功的、与新产品类型相关的标杆产品

### 错误2：分析过于表面

**问题**：只分析表面现象，没有深入分析设计原理

**修正**：深入分析设计原理，理解设计背后的逻辑

### 错误3：忽视优缺点分析

**问题**：只分析优点，忽视缺点和适用场景

**修正**：全面分析优缺点和适用场景，避免盲目复制

### 错误4：设计模式提炼不准确

**问题**：提炼的设计模式不准确或不具有普适性

**修正**：确保提炼的设计模式准确、具有普适性和可操作性

### 错误5：新产品设计建议不适配

**问题**：设计建议没有考虑新产品的特点，直接照搬标杆产品

**修正**：根据新产品特点调整设计建议，确保适配性

### 错误6：忽视数据验证

**问题**：分析和建议没有数据支持

**修正**：基于实际数据进行分析，确保分析和建议有数据支持
