# Setup Engine

配置项目游戏引擎和版本。在 CLAUDE.md 中固定引擎，检测知识差距，当版本超出 LLM 训练数据时通过 WebSearch 填充引擎参考文档。

## 配置要素

### 1. 引擎选择 (Engine Selection)
- 完整规格：引擎和版本都提供
- 仅引擎：提供引擎，版本将查找
- 引导模式：完全引导模式（引擎推荐+版本）

### 2. 引擎推荐 (Engine Recommendation)
- 游戏类型：2D、3D 或两者
- 目标平台：PC、移动、主机、Web
- 团队规模和经验：单人新手、单人经验丰富、小团队
- 语言偏好：GDScript、C#、C++、可视化脚本
- 引擎许可预算：仅免费或商业许可可接受

### 3. 版本查找 (Version Lookup)
- 最新稳定版本：通过 WebSearch 查找
- 版本确认：与用户确认版本
- 版本固定：在 CLAUDE.md 中固定版本

### 4. 知识差距检测 (Knowledge Gap Detection)
- 训练数据截止：LLM 知识截止日期
- 版本覆盖：各引擎版本覆盖范围
- 风险评估：低风险、中风险、高风险

## 引擎决策矩阵

| 因素 | Godot 4 | Unity | Unreal Engine 5 |
|------|---------|-------|-----------------|
| **最适合** | 2D游戏、小型3D、单人/小团队 | 移动端、中范围3D、跨平台 | AAA 3D、照片级真实感、大团队 |
| **语言** | GDScript (+ C#, C++ 通过扩展) | C# | C++ / Blueprint |
| **成本** | 免费，MIT许可 | 收入阈值下免费 | 收入阈值下免费，5%版税 |
| **学习曲线** | 温和 | 中等 | 陡峭 |
| **2D支持** | 优秀（原生） | 良好（但3D优先引擎） | 可能但不理想 |
| **3D质量上限** | 良好（快速改进） | 非常好 | 顶级 |
| **Web导出** | 是（原生） | 是（有限） | 否 |
| **主机导出** | 通过第三方 | 是（需许可） | 是 |
| **开源** | 是 | 否 | 源代码可用 |

## 配置原则

### 1. 准确性
- 不猜测引擎版本
- 通过 WebSearch 验证
- 用户确认版本

### 2. 可追溯性
- 记录配置决策
- 记录配置日期
- 记录版本信息

### 3. 可更新性
- 支持版本更新
- 支持参考文档刷新
- 支持配置修改

### 4. 知识管理
- 检测知识差距
- 填充参考文档
- 更新代理指令

## 配置流程

### 1. 解析参数
- 检查参数格式
- 确定配置模式
- 设置配置目标

### 2. 引导模式（无参数）
- 检查现有游戏概念
- 提问引导选择
- 提供引擎推荐
- 用户选择引擎

### 3. 查找当前版本
- 使用 WebSearch 查找
- 确认最新稳定版本
- 用户确认版本

### 4. 更新技术栈
- 读取 CLAUDE.md
- 更新技术栈部分
- 填充实际值

### 5. 填充技术偏好
- 创建技术偏好文档
- 填充引擎默认值
- 填充命名约定
- 用户确认保存

### 6. 确定知识差距
- 检查训练数据截止
- 比较版本覆盖
- 评估风险级别
- 通知用户风险

### 7. 填充引擎参考文档
- 低风险：创建最小参考
- 中高风险：创建完整参考
- 搜索迁移指南
- 搜索破坏性更改
- 创建参考目录

### 8. 更新 CLAUDE.md 导入
- 更新引擎版本引用
- 指向正确引擎
- 更新代理指令

### 9. 刷新子命令
- 检查新版本
- 更新参考文档
- 报告更改

### 10. 输出摘要
- 输出配置摘要
- 提供后续步骤
- 完成配置

## 输出格式

### 配置摘要
```
引擎设置完成
=====================
引擎:          [名称] [版本]
知识风险:      [低/中/高]
参考文档:      [已创建/跳过]
CLAUDE.md:     [已更新]
技术偏好:     [已创建/更新]
代理配置:     [已验证]

后续步骤:
1. 查阅 docs/engine-reference/<engine>/VERSION.md
2. 运行 /design-systems 分解概念为独立系统
3. 运行 /prototype [核心机制] 测试核心循环
4. 运行 /sprint-plan new 创建第一个里程碑
```
