# Core Gameplay Design

核心玩法设计技能，设计游戏的核心循环和玩家目标。

## 适用场景

- 设计游戏核心玩法
- 定义核心循环
- 设计玩家目标
- 构建游戏基础
- 确保游戏可玩性

## 操作步骤

### 步骤1：定义核心循环

**目标**：定义游戏的核心循环机制

**操作方法**：
1. **核心循环识别**
   - 识别玩家主要操作
   - 识别反馈机制
   - 识别奖励机制
   - 识别循环路径

2. **循环类型选择**
   - 角色扮演（RPG）：任务→战斗→奖励→成长
   - 动作执行：操作→反馈→挑战→进步
   - 策略组合：决策→执行→结果→学习
   - 资源管理：采集→管理→发展→扩张

3. **循环设计**
   - 设计操作步骤
   - 设计反馈机制
   - 设计奖励系统
   - 设计循环节奏

**检查清单**：
- [ ] 核心循环已识别
- [ ] 循环类型已选择
- [ ] 循环已设计
- [ ] 节奏已确定

### 步骤2：设计玩家目标

**目标**：设计清晰明确的玩家目标

**操作方法**：
1. **目标类型设计**
   - 短期目标：单局目标、关卡目标
   - 中期目标：章节目标、阶段目标
   - 长期目标：游戏目标、成就目标

2. **目标层次设计**
   - 主目标：游戏最终目标
   - 次目标：阶段性目标
   - 辅助目标：可选目标
   - 隐藏目标：探索目标

3. **目标反馈设计**
   - 设计目标提示
   - 设计进度显示
   - 设计完成反馈
   - 设计奖励机制

**检查清单**：
- [ ] 目标类型已设计
- [ ] 目标层次已设计
- [ ] 反馈已设计
- [ ] 目标已验证

### 步骤3：设计核心机制

**目标**：设计支撑核心循环的核心机制

**操作方法**：
1. **机制识别**
   - 识别核心操作机制
   - 识别核心反馈机制
   - 识别核心奖励机制
   - 识别核心挑战机制

2. **机制设计**
   - 设计操作规则
   - 设计反馈规则
   - 设计奖励规则
   - 设计挑战规则

3. **机制平衡**
   - 平衡难度
   - 平衡奖励
   - 平衡节奏
   - 平衡体验

**检查清单**：
- [ ] 机制已识别
- [ ] 机制已设计
- [ ] 平衡已设计
- [ ] 机制已验证

### 步骤4：设计心流体验

**目标**：设计让玩家进入心流状态的体验

**操作方法**：
1. **心流条件设计**
   - 设计清晰目标
   - 设计即时反馈
   - 设计挑战与技能匹配
   - 设计控制感

2. **难度曲线设计**
   - 设计渐进难度
   - 设计难度波动
   - 设计挑战多样性
   - 设计学习曲线

3. **节奏设计**
   - 设计游戏节奏
   - 设计紧张与放松
   - 设计高潮与低谷
   - 设计节奏变化

**检查清单**：
- [ ] 心流条件已设计
- [ ] 难度曲线已设计
- [ ] 节奏已设计
- [ ] 体验已验证

### 步骤5：验证核心玩法

**目标**：验证核心玩法的可玩性和趣味性

**操作方法**：
1. **原型制作**
   - 制作核心玩法原型
   - 实现核心机制
   - 测试核心循环
   - 收集反馈

2. **可玩性测试**
   - 测试玩法可玩性
   - 测试玩法趣味性
   - 测试玩法深度
   - 测试玩法平衡

3. **迭代优化**
   - 分析测试反馈
   - 识别问题点
   - 设计优化方案
   - 迭代优化

**检查清单**：
- [ ] 原型已制作
- [ ] 可玩性已测试
- [ ] 问题已识别
- [ ] 优化已完成

## 操作注意事项

### 核心循环设计

1. **简洁清晰**：核心循环要简洁清晰
2. **即时反馈**：提供即时反馈
3. **合理奖励**：设计合理奖励
4. **循环完整**：确保循环完整

### 目标设计

1. **清晰明确**：目标要清晰明确
2. **层次分明**：目标层次要分明
3. **反馈及时**：目标反馈要及时
4. **奖励合理**：目标奖励要合理

### 机制设计

1. **规则清晰**：机制规则要清晰
2. **平衡合理**：机制平衡要合理
3. **易于理解**：机制要易于理解
4. **易于操作**：机制要易于操作

## 常见错误

### 错误1：核心循环不清晰

**问题**：核心循环不清晰，玩家不知道做什么

**修正**：简化核心循环，确保清晰明确

### 错误2：目标不明确

**问题**：目标不明确，玩家缺乏方向

**修正**：设计清晰明确的目标

### 错误3：反馈不及时

**问题**：反馈不及时，玩家缺乏即时满足

**修正**：设计即时反馈机制

### 错误4：难度不合理

**问题**：难度不合理，玩家体验差

**修正**：设计合理的难度曲线

### 错误5：不验证

**问题**：不验证核心玩法就投入开发

**修正**：必须验证核心玩法的可玩性
