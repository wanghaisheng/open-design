# Visual QA

检查游戏视觉质量，使用静态、动态和问题模式检测视觉缺陷、渲染错误和运动异常。

## QA模式

### 1. 静态模式 (Static Mode)
- 参考图像：提供参考图像
- 单个截图：检查单个截图
- 像素对比：像素级对比
- 视觉一致性：检查视觉一致性

### 2. 动态模式 (Dynamic Mode)
- 参考图像：提供参考图像
- 帧序列：检查帧序列
- 运动分析：分析运动
- 渲染一致性：检查渲染一致性

### 3. 问题模式 (Question Mode)
- 自由问题：自由提问
- 截图分析：分析截图
- 缺陷检测：检测缺陷
- 问题诊断：诊断问题

## 检测类型

### 1. 视觉缺陷 (Visual Defects)
- Z-fighting：Z轴冲突
- 纹理拉伸：纹理拉伸问题
- 法线错误：法线计算错误
- 阴影错误：阴影渲染错误
- 光照错误：光照计算错误

### 2. 渲染错误 (Rendering Bugs)
- 渲染伪影：渲染伪影
- 帧率问题：帧率不稳定
- 加载问题：加载闪烁
- 内存泄漏：内存泄漏
- 崩溃问题：渲染崩溃

### 3. 运动异常 (Motion Anomalies)
- 动画错误：动画播放错误
- 物理错误：物理模拟错误
- 碰撞错误：碰撞检测错误
- 同步错误：同步问题
- 卡顿问题：运动卡顿

## QA原则

### 1. 全面性
- 覆盖所有视觉元素
- 覆盖所有场景
- 覆盖所有平台

### 2. 准确性
- 准确识别问题
- 准确描述问题
- 准确分类问题

### 3. 及时性
- 及时发现问题
- 及时报告问题
- 及时跟踪问题

### 4. 可追溯性
- 记录问题历史
- 记录修复过程
- 记录验证结果

## QA流程

### 1. QA计划
- 制定QA计划
- 确定QA范围
- 确定QA标准

### 2. QA执行
- 执行静态QA
- 执行动态QA
- 执行问题QA

### 3. 问题报告
- 记录问题
- 分类问题
- 优先级排序

### 4. 问题跟踪
- 跟踪修复进度
- 验证修复结果
- 关闭问题

### 5. QA报告
- 生成QA报告
- 分析QA数据
- 提出改进建议

## 输出格式

### QA报告
- QA范围
- QA标准
- 问题列表
- 问题分类
- 优先级排序
- 修复建议
- QA总结

### 问题报告
- 问题ID
- 问题描述
- 问题截图
- 问题严重性
- 修复建议
- 修复状态
