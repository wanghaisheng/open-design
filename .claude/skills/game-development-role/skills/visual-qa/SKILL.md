# Visual QA

检查游戏视觉质量，使用静态、动态和问题模式检测视觉缺陷、渲染错误和运动异常。

## 适用场景

- 需要执行静态QA
- 需要执行动态QA
- 需要执行问题QA
- 需要检测视觉缺陷
- 需要检测渲染错误

## 操作步骤

### 步骤1：QA计划

**目标**：制定QA计划

**操作方法**：
1. 制定QA计划（QA目标、QA时间、QA资源）
2. 确定QA范围（视觉元素、场景、平台）
3. 确定QA标准（质量标准、验收标准）

**检查清单**：
- [ ] QA计划已制定
- [ ] QA范围已确定
- [ ] QA标准已确定

### 步骤2：QA执行

**目标**：执行QA

**操作方法**：
1. 执行静态QA（参考图像、单个截图、像素对比、视觉一致性）
2. 执行动态QA（参考图像、帧序列、运动分析、渲染一致性）
3. 执行问题QA（自由问题、截图分析、缺陷检测、问题诊断）

**检查清单**：
- [ ] 静态QA已执行
- [ ] 动态QA已执行
- [ ] 问题QA已执行

### 步骤3：问题报告

**目标**：记录和分类问题

**操作方法**：
1. 记录问题（问题描述、问题截图、问题严重性）
2. 分类问题（视觉缺陷、渲染错误、运动异常）
3. 优先级排序（高/中/低优先级）

**检查清单**：
- [ ] 问题已记录
- [ ] 问题已分类
- [ ] 优先级已排序

### 步骤4：问题跟踪

**目标**：跟踪问题修复

**操作方法**：
1. 跟踪修复进度（修复状态、修复时间）
2. 验证修复结果（回归测试、验证测试）
3. 关闭问题（问题关闭、问题归档）

**检查清单**：
- [ ] 修复进度已跟踪
- [ ] 修复结果已验证
- [ ] 问题已关闭

### 步骤5：QA报告

**目标**：生成QA报告

**操作方法**：
1. 生成QA报告（QA范围、QA标准、问题列表）
2. 分析QA数据（问题分布、问题趋势）
3. 提出改进建议（改进措施、预防措施）

**检查清单**：
- [ ] QA报告已生成
- [ ] QA数据已分析
- [ ] 改进建议已提出

## 常见错误

### 错误1：不制定QA计划

**问题**：不制定QA计划、不确定QA范围

**修正**：必须制定QA计划，确定QA范围，确定QA标准

### 错误2：不执行静态QA

**问题**：不执行静态QA、不进行像素对比

**修正**：必须执行静态QA，进行像素对比，检查视觉一致性

### 错误3：不记录问题

**问题**：不记录问题、不分类问题

**修正**：必须记录问题，分类问题，优先级排序

### 错误4：不跟踪修复进度

**问题**：不跟踪修复进度、不验证修复结果

**修正**：必须跟踪修复进度，验证修复结果，关闭问题

### 错误5：不生成QA报告

**问题**：不生成QA报告、不分析QA数据

**修正**：必须生成QA报告，分析QA数据，提出改进建议
