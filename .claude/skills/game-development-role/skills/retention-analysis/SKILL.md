# Retention Analysis

留存分析技能，分析用户留存数据以优化用户留存。

## 适用场景

- 分析用户留存
- 识别流失原因
- 优化用户生命周期
- 提高留存率
- 优化用户参与度

## 操作步骤

### 步骤1：定义留存指标

**目标**：定义留存分析指标

**操作方法**：
1. 确定留存类型（D1, D7, D30等）
2. 定义留存计算方法
3. 设置留存目标
4. 确定用户群体
5. 设置时间范围

**检查清单**：
- [ ] 留存类型已确定
- [ ] 计算方法已定义
- [ ] 留存目标已设置
- [ ] 用户群体已确定
- [ ] 时间范围已设置

### 步骤2：收集留存数据

**目标**：收集用户留存数据

**操作方法**：
1. 配置用户追踪
2. 收集登录数据
3. 收集行为数据
4. 计算留存率
5. 验证数据质量

**检查清单**：
- [ ] 用户追踪已配置
- [ ] 登录数据已收集
- [ ] 行为数据已收集
- [ ] 留存率已计算
- [ ] 数据质量已验证

### 步骤3：分析留存模式

**目标**：分析用户留存模式

**操作方法**：
1. 分析留存曲线
2. 识别留存拐点
3. 分析留存影响因素
4. 比较不同用户群体
5. 分析时间趋势

**检查清单**：
- [ ] 留存曲线已分析
- [ ] 留存拐点已识别
- [ ] 影响因素已分析
- [ ] 用户群体已比较
- [ ] 时间趋势已分析

### 步骤4：识别流失原因

**目标**：识别用户流失原因

**操作方法**：
1. 分析流失用户行为
2. 分析留存用户行为
3. 对比流失和留存用户
4. 识别关键流失点
5. 验证流失原因

**检查清单**：
- [ ] 流失用户行为已分析
- [ ] 留存用户行为已分析
- [ ] 用户对比已完成
- [ ] 流失点已识别
- [ ] 流失原因已验证

### 步骤5：提出优化建议

**目标**：基于留存分析提出优化建议

**操作方法**：
1. 识别优化机会
2. 制定留存提升方案
3. 评估优化影响
4. 优先级排序
5. 制定实施计划

**检查清单**：
- [ ] 优化机会已识别
- [ ] 提升方案已制定
- [ ] 优化影响已评估
- [ ] 优先级已排序
- [ ] 实施计划已制定

## 操作注意事项

### 留存计算

1. **定义清晰**：留存定义要清晰一致
2. **时间窗口**：选择合适的时间窗口
3. **用户群体**：明确用户群体定义
4. **计算方法**：使用标准计算方法

### 分析方法

1. **多维度**：从多维度分析留存
2. **分段分析**：按用户分段分析
3. **行为分析**：结合行为分析
4. **长期跟踪**：跟踪长期留存

### 优化策略

1. **早期干预**：在早期流失点干预
2. **个性化**：提供个性化体验
3. **参与度**：提高用户参与度
4. **持续优化**：持续优化留存策略

## 常见错误

### 错误1：留存定义不一致

**问题**：留存定义不一致，数据不可比

**修正**：使用一致的留存定义

### 错误2：忽视早期留存

**问题**：只关注长期留存，忽视早期留存

**修正**：同时关注早期和长期留存

### 错误3：不分段分析

**问题**：不分段分析，忽略群体差异

**修正**：进行用户分段分析

### 错误4：归因错误

**问题**：错误归因流失原因

**修正**：通过用户研究和数据验证归因

### 错误5：不跟踪优化效果

**问题**：优化后不跟踪效果

**修正**：持续跟踪优化效果，验证改进
