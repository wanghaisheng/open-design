# Performance Profile

结构化性能分析工作流程。识别瓶颈、根据预算测量性能、生成带优先级排序的优化建议。

## 分析要素

### 1. CPU性能分析 (CPU Profiling)
- 更新函数：_process() / Update() / Tick() 函数
- 嵌套循环：大集合上的嵌套循环
- 字符串操作：热路径中的字符串操作
- 分配模式：每帧代码中的分配模式
- 搜索/排序：游戏实体上的未优化搜索/排序
- 物理查询：每帧的昂贵物理查询

### 2. 内存性能分析 (Memory Profiling)
- 数据结构：大型数据结构及其增长模式
- 纹理/资产：纹理/资产内存占用估计
- 对象池：对象池 vs 实例化/销毁模式
- 泄漏引用：应该释放但未释放的对象
- 缓存大小：缓存大小和驱逐策略

### 3. 渲染性能分析 (Rendering Profiling)
- Draw Call：Draw Call 估计
- 过度绘制：重叠透明对象的过度绘制
- 着色器复杂度：着色器复杂度
- 粒子系统：未优化的粒子系统
- LOD/遮挡：缺失 LOD 或遮挡剔除

### 4. I/O性能分析 (I/O Profiling)
- 保存/加载：保存/加载性能
- 资产加载：资产加载模式（同步 vs 异步）
- 网络消息：网络消息频率和大小

## 性能预算

### 帧时间预算
- 目标 FPS：60fps = 16.67ms 帧预算
- 30fps = 33.33ms 帧预算
- 帧时间预算：根据目标 FPS 计算

### 内存预算
- 总内存预算：总内存目标
- 系统内存预算：每个系统的内存分配
- 资产内存预算：资产内存分配

### 加载时间预算
- 初始加载：初始加载时间目标
- 场景加载：场景加载时间目标
- 资产加载：资产加载时间目标

### 其他预算
- Draw Call：Draw Call 预算
- 网络带宽：网络带宽限制（如果是多人游戏）

## 分析原则

### 1. 测量优先
- 不测量不优化
- 性能直觉不可靠
- 数据驱动决策

### 2. 影响评估
- 估计影响
- 可操作建议
- 风险评估

### 3. 目标硬件
- 在目标硬件上分析
- 不只在开发机器
- 考虑最低规格

### 4. 最坏情况
- 考虑最坏场景
- 最大实体数
- 最差网络条件

## 分析流程

### 1. 确定范围
- 从参数确定范围
- 特定系统分析
- 全面系统分析

### 2. 读取性能预算
- 检查设计文档
- 检查 CLAUDE.md
- 确定性能目标

### 3. 分析代码库
- CPU 分析目标
- 内存分析目标
- 渲染分析目标
- I/O 分析目标

### 4. 生成分析报告
- 性能预算对比
- 识别热点
- 优化建议
- 快速修复
- 需要调查

### 5. 输出报告摘要
- 前 3 个热点
- 预算余量
- 推荐下一步
