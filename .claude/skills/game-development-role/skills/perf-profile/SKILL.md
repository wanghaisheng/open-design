# Performance Profile

结构化性能分析工作流程。识别瓶颈、根据预算测量性能、生成带优先级排序的优化建议。

## 适用场景

- 需要分析CPU性能
- 需要分析内存性能
- 需要分析渲染性能
- 需要分析I/O性能
- 需要生成性能分析报告

## 操作步骤

### 步骤1：确定范围

**目标**：确定分析范围

**操作方法**：
1. 从参数确定范围（特定系统分析、全面系统分析）
2. 确定分析目标（CPU、内存、渲染、I/O）

**检查清单**：
- [ ] 分析范围已确定
- [ ] 分析目标已确定

### 步骤2：读取性能预算

**目标**：读取性能预算

**操作方法**：
1. 检查设计文档（性能目标）
2. 检查 CLAUDE.md（性能配置）
3. 确定性能目标（帧时间、内存、加载时间、Draw Call）

**检查清单**：
- [ ] 设计文档已检查
- [ ] CLAUDE.md已检查
- [ ] 性能目标已确定

### 步骤3：分析代码库

**目标**：分析代码库

**操作方法**：
1. CPU分析目标（更新函数、嵌套循环、字符串操作、分配模式、搜索/排序、物理查询）
2. 内存分析目标（数据结构、纹理/资产、对象池、泄漏引用、缓存大小）
3. 渲染分析目标（Draw Call、过度绘制、着色器复杂度、粒子系统、LOD/遮挡）
4. I/O分析目标（保存/加载、资产加载、网络消息）

**检查清单**：
- [ ] CPU分析已完成
- [ ] 内存分析已完成
- [ ] 渲染分析已完成
- [ ] I/O分析已完成

### 步骤4：生成分析报告

**目标**：生成分析报告

**操作方法**：
1. 性能预算对比（帧时间、内存、加载时间、Draw Call）
2. 识别热点（热点位置、问题、预估影响、修复工作量）
3. 优化建议（按优先级排序）
4. 快速修复（每个 < 1 小时）
5. 需要调查（需要实际运行时分析）

**检查清单**：
- [ ] 性能预算已对比
- [ ] 热点已识别
- [ ] 优化建议已生成
- [ ] 快速修复已列出
- [ ] 需要调查已列出

### 步骤5：输出报告摘要

**目标**：输出报告摘要

**操作方法**：
1. 前 3 个热点
2. 预算余量
3. 推荐下一步

**检查清单**：
- [ ] 前 3 个热点已输出
- [ ] 预算余量已输出
- [ ] 推荐下一步已输出

## 操作注意事项

### 测量优先

1. **不测量不优化**：不测量不优化
2. **性能直觉不可靠**：性能直觉不可靠
3. **数据驱动决策**：数据驱动决策

### 影响评估

1. **估计影响**：估计影响

1. **在目标硬件上分析**：在目标硬件上分析
2. **不只在开发机器**：不只在开发机器

1. **考虑最坏场景**：考虑最坏场景

## 常见错误

### 错误1：不确定分析范围

**问题**：不确定分析范围，直接分析

**修正**：必须先确定分析范围和分析目标

### 错误2：不读取性能预算

**问题**：不读取性能预算，无法对比

**修正**：必须检查设计文档和CLAUDE.md，确定性能目标

### 错误3：不分析所有维度

**问题**：只分析CPU，不分析内存、渲染、I/O

**修正**：必须分析CPU、内存、渲染、I/O所有维度

### 错误4：不识别热点

**问题**：不识别热点，无法提供优化建议

**修正**：必须识别热点位置、问题、预估影响、修复工作量

### 错误5：不输出报告摘要

**问题**：不输出报告摘要

**修正**：必须输出前3个热点、预算余量、推荐下一步
