# Patch Notes

从git历史、冲刺数据和内部变更日志生成面向玩家的补丁说明。将开发者语言翻译为清晰、引人入胜的玩家沟通。

## 补丁说明要素

### 1. 变更分类
- 新内容：新功能、地图、角色、物品、模式
- 游戏玩法变更：平衡调整、机制变更、进度变更
- 生活质量：UI改进、便利功能、可访问性
- 错误修复：按系统分组（战斗、UI、网络等）
- 性能：玩家可能注意到的优化改进
- 已知问题：关于未解决问题的透明度

### 2. 语言翻译
- "重构伤害计算管道" → "改进命中检测准确性"
- "修复库存管理器中的空引用" → "修复打开库存时的崩溃"
- "减少战斗循环中的GC分配" → "改进战斗性能"
- 移除不影响玩家的纯内部变更
- 保留平衡变更的具体数字（伤害：50 → 45）

### 3. 输出风格
- 简洁：要点
- 详细：带上下文
- 完整：带开发者评论

## 补丁说明原则

### 1. 玩家友好
- 避免内部术语
- 专注于玩家体验
- 使用清晰语言

### 2. 透明度
- 诚实面对已知问题
- 提供变通方法
- 解释变更原因

### 3. 准确性
- 基于实际变更
- 验证变更
- 提供具体数值

### 4. 语气匹配
- 匹配游戏声音
- 调整正式程度
- 保持一致

## 补丁说明流程

### 1. 解析参数
- 版本：生成说明的发布版本
- 风格：输出风格——简洁、详细、完整

### 2. 从多个来源收集变更数据
- 读取内部变更日志
- 运行git日志
- 读取冲刺回顾
- 读取平衡变更文档
- 读取QA错误修复记录

### 3. 分类所有变更
- 新内容
- 游戏玩法变更
- 生活质量
- 错误修复
- 性能
- 已知问题

### 4. 翻译开发者语言为玩家语言
- 技术术语转换为玩家友好语言
- 移除不影响玩家的纯内部变更
- 保留平衡变更的具体数字
- 描述玩家体验而非技术原因

### 5. 使用适当风格生成补丁说明
- 简洁风格
- 详细风格
- 完整风格

### 6. 审查输出
- 无内部术语
- 无内部系统引用
- 平衡变更包含前后值
- 错误修复描述玩家体验
- 语气匹配游戏声音

### 7. 保存补丁说明
- 保存到 production/releases/[version]/patch-notes.md
- 创建目录（如需要）

### 8. 输出到用户
- 完整补丁说明
- 文件路径
- 按类别变更计数
- 排除的内部变更（供审查）

## 输出格式

### 简洁风格
```markdown
# 补丁 [版本] — [标题]

**新增**
- [功能1]
- [功能2]

**变更**
- [平衡/机制变更，带前后值]

**修复**
- [错误修复1]
- [错误修复2]

**已知问题**
- [问题1]
```

### 详细风格
```markdown
# 补丁 [版本] — [标题]
*[日期]*

## 亮点
[1-2句最激动人心的变更摘要]

## 新内容
### [功能名称]
[2-3句描述功能和玩家应兴奋的原因]

## 游戏玩法变更
### 平衡
| 变更 | 之前 | 之后 | 原因 |
| ---- | ---- | ---- | ---- |
| [物品/能力] | [旧值] | [新值] | [简要理由] |

### 机制
- **[变更]**：[变更解释和原因]

## 生活质量
- [带上下文的改进]

## 错误修复
### 战斗
- 修复[玩家经历的描述]

### UI
- 修复[描述]

### 网络
- 修复[描述]

## 性能
- [玩家将注意到的改进]

## 已知问题
- [问题和可用变通方法]
```

### 完整风格
包含详细风格的所有内容，加上：
```markdown
## 开发者评论
### [主题]
> [对重大变更的开发者洞察——为什么做出、考虑了什么、团队学到了什么。以第一人称团队声音编写。]
```

## 规则

- 面向玩家补丁说明中无内部术语（用玩家友好语言替换技术术语）
- 无内部系统、票据或冲刺编号引用
- 平衡变更包含前后值
- 错误修复描述玩家体验，而非技术原因
- 语气匹配游戏声音（根据游戏风格调整正式程度）
- 保存到 production/releases/[version]/patch-notes.md
