# Code Review

对指定文件或文件集执行架构和质量代码审查。检查编码标准合规性、架构模式遵循性、SOLID原则、可测试性和性能问题。

## 审查要素

### 1. 编码标准合规性 (Coding Standards Compliance)
- 文档注释：公共方法和类有文档注释
- 圈复杂度：每个方法圈复杂度低于10
- 方法长度：没有方法超过40行（不包括数据声明）
- 依赖注入：依赖被注入（没有游戏状态的静态单例）
- 配置值：配置值从数据文件加载
- 系统接口：系统暴露接口（不依赖具体类）

### 2. 架构合规性 (Architectural Compliance)
- 依赖方向：正确的依赖方向（引擎 <- 游戏玩法，而非反向）
- 循环依赖：模块间没有循环依赖
- 层分离：适当的层分离（UI不拥有游戏状态）
- 事件/信号：使用事件/信号进行跨系统通信
- 模式一致性：与代码库中已建立的模式一致

### 3. SOLID合规性 (SOLID Compliance)
- 单一职责：每个类只有一个变更原因
- 开闭原则：可扩展无需修改
- 里氏替换：子类型可替换基类型
- 接口隔离：没有臃肿接口
- 依赖倒置：依赖抽象而非具体

### 4. 游戏开发常见问题 (Game Development Issues)
- 帧率独立性：帧率独立性（delta time使用）
- 热路径分配：热路径（更新循环）中没有分配
- 空状态处理：适当的空/空状态处理
- 线程安全：需要时线程安全
- 资源清理：资源清理（无泄漏）

## 审查原则

### 1. 全面性
- 全面审查代码
- 检查所有方面
- 避免遗漏

### 2. 建设性
- 提供建设性反馈
- 提供改进建议
- 避免批评

### 3. 可操作性
- 审查结果可操作
- 提供具体建议
- 指导后续工作

### 4. 及时性
- 及时审查
- 及时反馈
- 及时修正

## 审查流程

### 1. 读取目标文件
- 完整读取目标文件
- 理解代码结构
- 理解代码目的

### 2. 读取编码标准
- 读取CLAUDE.md
- 理解项目编码标准
- 理解架构模式

### 3. 识别系统类别
- 识别系统类别（引擎、游戏玩法、AI、网络、UI、工具）
- 应用类别特定标准

### 4. 评估编码标准
- 检查文档注释
- 检查圈复杂度
- 检查方法长度
- 检查依赖注入
- 检查配置值
- 检查系统接口

### 5. 检查架构合规性
- 检查依赖方向
- 检查循环依赖
- 检查层分离
- 检查事件/信号
- 检查模式一致性

### 6. 检查SOLID合规性
- 检查单一职责
- 检查开闭原则
- 检查里氏替换
- 检查接口隔离
- 检查依赖倒置

### 7. 检查游戏开发问题
- 检查帧率独立性
- 检查热路径分配
- 检查空状态处理
- 检查线程安全
- 检查资源清理

### 8. 输出审查报告
- 生成审查报告
- 提供改进建议
- 提供批准决定
