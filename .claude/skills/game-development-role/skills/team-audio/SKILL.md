# Team Audio

协调音频团队：音频总监+声音设计师+技术艺术家+游戏程序员，实现从方向到实施的完整音频管道。

## 适用场景

- 需要协调音频团队
- 需要设计音频方向
- 需要实施音频系统
- 需要集成音频代码
- 需要编译音频文档

## 操作步骤

### 步骤1：读取参数

**目标**：读取音频参数

**操作方法**：
1. 读取目标功能或区域（战斗、主菜单、森林生物群落、Boss遭遇）
2. 收集上下文（设计文档、声音圣经、现有音频资产）
3. 确定音频需求（功能需求、性能需求、资源需求）

**检查清单**：
- [ ] 目标功能已读取
- [ ] 上下文已收集
- [ ] 音频需求已确定

### 步骤2：音频方向

**目标**：定义音频方向

**操作方法**：
1. 定义功能/区域的声音标识
2. 指定情感基调和音频调色板
3. 设置音乐方向（自适应层、茎、过渡）
4. 定义音频优先级和混音目标
5. 建立自适应音频规则

**检查清单**：
- [ ] 声音标识已定义
- [ ] 情感基调已指定
- [ ] 音乐方向已设置
- [ ] 音频优先级已定义
- [ ] 自适应规则已建立

### 步骤3：声音设计

**目标**：设计声音

**操作方法**：
1. 为每个音频事件创建详细SFX规格
2. 定义声音类别（环境、UI、游戏玩法、音乐、对话）
3. 指定每个声音参数（音量范围、音调变化、衰减）
4. 规划带触发条件的音频事件列表
5. 定义混音组和鸭子规则

**检查清单**：
- [ ] SFX规格已创建
- [ ] 声音类别已定义
- [ ] 声音参数已指定
- [ ] 音频事件已规划
- [ ] 混音组已定义

### 步骤4：技术实施

**目标**：实施音频技术

**操作方法**：
1. 设计音频中间件集成（Wwise/FMOD/原生）
2. 定义音频总线结构和路由
3. 指定每个平台音频资产的内存预算
4. 规划流媒体与预加载资产策略
5. 设计音频反应视觉效果

**检查清单**：
- [ ] 音频中间件已设计
- [ ] 总线结构已定义
- [ ] 内存预算已指定
- [ ] 流媒体策略已规划
- [ ] 视觉效果已设计

### 步骤5：代码集成

**目标**：集成音频代码

**操作方法**：
1. 实现音频管理器系统或审查现有
2. 将音频事件连接到游戏触发器
3. 实现自适应音乐系统（如指定）
4. 设置音频遮挡/混响区域
5. 编写音频事件触发器的单元测试

**检查清单**：
- [ ] 音频管理器已实现
- [ ] 音频事件已连接
- [ ] 自适应音乐已实现
- [ ] 遮挡区域已设置
- [ ] 单元测试已编写

### 步骤6：编译文档

**目标**：编译音频设计文档

**操作方法**：
1. 结合所有团队输出
2. 创建综合音频设计文档
3. 保存到design/gdd/audio-[feature].md
4. 输出摘要（音频事件计数、资产计数、实施任务）

**检查清单**：
- [ ] 团队输出已结合
- [ ] 音频设计文档已创建
- [ ] 文档已保存
- [ ] 摘要已输出

## 常见错误

### 错误1：不读取参数

**问题**：不读取目标功能、不收集上下文

**修正**：必须读取目标功能，收集上下文，确定音频需求

### 错误2：不定义音频方向

**问题**：不定义声音标识、不指定情感基调

**修正**：必须定义声音标识，指定情感基调，设置音乐方向

### 错误3：不设计声音

**问题**：不创建SFX规格、不定义声音类别

**修正**：必须创建SFX规格，定义声音类别，指定声音参数

### 错误4：不实施音频技术

**问题**：不设计音频中间件、不定义总线结构

**修正**：必须设计音频中间件，定义总线结构，指定内存预算

### 错误5：不集成音频代码

**问题**：不实现音频管理器、不连接音频事件

**修正**：必须实现音频管理器，连接音频事件，实现自适应音乐
