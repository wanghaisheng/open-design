# Bug Reporting

编写游戏Bug报告，包括Bug描述、重现步骤、严重性评估和修复建议，确保Bug得到有效修复。

## 适用场景

- 发现游戏Bug需要报告
- 需要记录Bug的重现步骤
- 需要评估Bug的严重性
- 需要提供修复建议
- 需要跟踪Bug修复进度

## 操作步骤

### 步骤1：Bug发现和信息收集

**目标**：发现Bug并收集相关信息

**操作方法**：
1. 发现Bug（测试、玩家反馈、内部发现）
2. 记录Bug（初步记录Bug现象）
3. 收集信息（环境信息、截图、视频、日志、存档）

**检查清单**：
- [ ] Bug已发现
- [ ] Bug已记录
- [ ] 信息已收集

### 步骤2：编写Bug描述

**目标**：编写清晰详细的Bug描述

**操作方法**：
1. **Bug标题**：编写简洁明确的标题
2. **Bug描述**：编写详细的Bug描述
3. **预期行为**：描述预期的正确行为
4. **实际行为**：描述实际的错误行为

**检查清单**：
- [ ] Bug标题已编写
- [ ] Bug描述已编写
- [ ] 预期行为已描述
- [ ] 实际行为已描述

### 步骤3：编写重现步骤

**目标**：编写详细的重现步骤

**操作方法**：
1. **前置条件**：描述重现的前置条件
2. **操作步骤**：编写详细的重现步骤（按顺序）
3. **触发条件**：描述触发Bug的条件
4. **重现率**：评估Bug的重现概率（100%/50%/随机）

**检查清单**：
- [ ] 前置条件已描述
- [ ] 操作步骤已编写
- [ ] 触发条件已描述
- [ ] 重现率已评估

### 步骤4：收集环境信息

**目标**：记录环境信息

**操作方法**：
1. **游戏版本**：记录游戏版本号
2. **平台信息**：记录操作系统、硬件信息
3. **设备信息**：记录设备型号、规格
4. **配置信息**：记录游戏配置

**检查清单**：
- [ ] 游戏版本已记录
- [ ] 平台信息已记录
- [ ] 设备信息已记录
- [ ] 配置信息已记录

### 步骤5：评估严重性

**目标**：评估Bug的严重性和优先级

**操作方法**：
1. **严重性级别**：评估严重性（Critical/High/Medium/Low）
2. **优先级评估**：确定优先级（P0/P1/P2/P3）
3. **影响范围**：评估影响范围（影响用户、影响功能、影响平台、影响频率）

**检查清单**：
- [ ] 严重性级别已评估
- [ ] 优先级已确定
- [ ] 影响范围已评估

### 步骤6：提供修复建议

**目标**：分析可能原因并提供修复建议

**操作方法**：
1. **可能原因**：分析可能原因（代码逻辑、资源问题、配置问题、兼容问题）
2. **修复建议**：提供修复方案
3. **修复难度**：评估修复难度
4. **修复时间**：预估修复时间
5. **风险评估**：评估修复风险

**检查清单**：
- [ ] 可能原因已分析
- [ ] 修复建议已提供
- [ ] 修复难度已评估
- [ ] 修复时间已预估
- [ ] 风险已评估

### 步骤7：添加附件

**目标**：添加相关附件

**操作方法**：
1. 添加Bug截图
2. 添加Bug录像（如适用）
3. 添加游戏日志
4. 添加游戏存档（如适用）

**检查清单**：
- [ ] 截图已添加
- [ ] 视频已添加（如适用）
- [ ] 日志已添加
- [ ] 存档已添加（如适用）

### 步骤8：提交Bug报告

**目标**：提交Bug报告到跟踪系统

**操作方法**：
1. 编写完整Bug报告
2. 提交到Bug跟踪系统
3. 分配负责人

**检查清单**：
- [ ] Bug报告已编写
- [ ] Bug报告已提交
- [ ] 负责人已分配

### 步骤9：跟踪Bug修复进度

**目标**：跟踪Bug修复进度并验证

**操作方法**：
1. 跟踪修复进度
2. 验证修复结果
3. 关闭Bug（如已修复）

**检查清单**：
- [ ] 修复进度已跟踪
- [ ] 修复结果已验证
- [ ] Bug已关闭（如适用）

## 输出格式

### Bug报告
- Bug标题
- Bug描述
- 重现步骤
- 环境信息
- 严重性评估
- 修复建议
- 附件

## 操作注意事项

### 清晰性

1. **描述清晰**：Bug描述应清晰明确
2. **步骤明确**：重现步骤应明确详细
3. **信息完整**：所有必要信息应完整

### 可重现性

1. **可重现**：Bug应可重现
2. **可验证**：Bug应可验证
3. **可测试**：Bug应可测试

### 准确性

1. **信息准确**：所有信息应准确
2. **描述准确**：描述应准确无误
3. **评估准确**：严重性评估应准确

### 及时性

1. **及时报告**：Bug应及时报告
2. **及时更新**：Bug状态应及时更新
3. **及时验证**：修复结果应及时验证

## 协作协议

### 向谁汇报

- **qa-lead**：Bug报告和严重性评估
- **technical-lead**：修复建议和风险评估
- **producer**：Bug影响和修复优先级

### 协调对象

- **gameplay-programmer**：游戏玩法Bug
- **engine-programmer**：引擎Bug
- **ui-programmer**：UI Bug

## 常见错误

### 错误1：重现步骤不详细

**问题**：重现步骤过于简单，无法重现

**修正**：必须编写详细的重现步骤，包括前置条件和触发条件

### 错误2：缺少环境信息

**问题**：不记录游戏版本、平台信息

**修正**：必须记录完整的环境信息（游戏版本、平台、设备、配置）

### 错误3：严重性评估不准确

**问题**：将Critical Bug评估为Low

**修正**：必须准确评估严重性，参考严重性级别定义

### 错误4：缺少附件

**问题**：不提供截图、视频、日志

**修正**：必须提供相关附件（截图、视频、日志、存档）

### 错误5：不跟踪修复进度

**问题**：提交Bug后不跟踪修复进度

**修正**：必须跟踪修复进度，验证修复结果
