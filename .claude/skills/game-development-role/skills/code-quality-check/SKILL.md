# Code Quality Check

代码质量检查技能，自动化代码质量分析和改进建议。

## 适用场景

- 自动化代码质量检查
- 静态代码分析
- 代码复杂度分析
- 代码重复检测
- 代码安全扫描

## 操作步骤

### 步骤1：配置质量检查工具

**目标**：设置代码质量检查工具

**操作方法**：
1. 选择质量检查工具（SonarQube, ESLint, flake8等）
2. 配置检查规则
3. 设置质量阈值
4. 配置忽略规则
5. 集成到CI/CD

**检查清单**：
- [ ] 质量检查工具已选择
- [ ] 检查规则已配置
- [ ] 质量阈值已设置
- [ ] 忽略规则已配置
- [ ] CI/CD已集成

### 步骤2：静态代码分析

**目标**：执行静态代码分析

**操作方法**：
1. 运行静态分析工具
2. 分析代码复杂度
3. 检测代码异味
4. 识别潜在bug
5. 生成分析报告

**检查清单**：
- [ ] 静态分析已运行
- [ ] 复杂度已分析
- [ ] 代码异味已检测
- [ ] 潜在bug已识别
- [ ] 分析报告已生成

### 步骤3：代码重复检测

**目标**：检测代码重复

**操作方法**：
1. 运行重复检测工具
2. 识别重复代码块
3. 计算重复率
4. 生成重复报告
5. 提取重复代码

**检查清单**：
- [ ] 重复检测已运行
- [ ] 重复代码已识别
- [ ] 重复率已计算
- [ ] 重复报告已生成
- [ ] 重复代码已提取

### 步骤4：安全扫描

**目标**：扫描代码安全问题

**操作方法**：
1. 运行安全扫描工具
2. 检测安全漏洞
3. 识别敏感信息
4. 检查依赖安全
5. 生成安全报告

**检查清单**：
- [ ] 安全扫描已运行
- [ ] 安全漏洞已检测
- [ ] 敏感信息已识别
- [ ] 依赖安全已检查
- [ ] 安全报告已生成

### 步骤5：质量报告和改进

**目标**：生成质量报告并提供改进建议

**操作方法**：
1. 汇总质量指标
2. 生成质量报告
3. 提供改进建议
4. 设置质量目标
5. 跟踪改进进度

**检查清单**：
- [ ] 质量指标已汇总
- [ ] 质量报告已生成
- [ ] 改进建议已提供
- [ ] 质量目标已设置
- [ ] 改进进度已跟踪

## 操作注意事项

### 质量标准

1. **明确标准**：定义清晰的质量标准
2. **合理阈值**：设置合理的质量阈值
3. **持续改进**：持续提高质量标准
4. **团队共识**：团队对质量标准达成共识

### 工具配置

1. **规则定制**：根据项目定制检查规则
2. **性能优化**：优化检查工具性能
3. **误报处理**：处理工具误报
4. **定期更新**：定期更新工具和规则

### 改进策略

1. **优先级排序**：按优先级处理质量问题
2. **小步改进**：小步持续改进
3. **团队协作**：团队协作解决质量问题
4. **知识分享**：分享质量改进经验

## 协作协议

### 向谁汇报

- lead-programmer：代码质量状态
- technical-director：技术质量决策

### 协调对象

- team-members：质量问题修复
- code-reviewer：代码审查

## 常见错误

### 错误1：规则过于严格

**问题**：质量检查规则过于严格，影响开发效率

**修正**：根据实际情况调整规则，平衡质量和效率

### 错误2：忽略误报

**问题**：工具误报被忽略，错过真正的问题

**修正**：定期审查误报，调整规则减少误报

### 错误3：不跟踪改进

**问题**：质量问题被识别但不跟踪改进

**修正**：建立问题跟踪机制，确保问题得到解决

### 错误4：依赖安全检查不足

**问题**：不检查依赖安全，引入已知漏洞

**修正**：定期检查依赖安全，及时更新依赖

### 错误5：质量报告不行动

**问题**：生成质量报告但不采取行动

**修正**：根据质量报告制定改进计划并执行
