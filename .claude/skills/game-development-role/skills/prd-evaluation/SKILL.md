name: prd-evaluation
description: 基于"好玩+商业化+长期运营"三维度评估游戏PRD，识别冲突点和短板，提供优化建议。
tags: [游戏, 设计, PRD, 评估]

---

## 适用场景

- 评估游戏PRD的质量和可行性
- 从"好玩"、"商业化"、"长期运营"三个维度评估游戏
- 识别PRD中的冲突点和短板
- 提供PRD优化建议
- 迭代优化PRD文档

---

## 操作步骤

### 步骤1：读取PRD文档

**目标**：读取游戏PRD的核心内容

**操作方法**：
1. 读取游戏PRD文档
2. 提取游戏愿景和目标
3. 提取目标用户画像
4. 提取核心玩法循环
5. 提取主要游戏系统（成长、战斗、经济）
6. 提取商业化模式草案
7. 提取社交系统和社区设计

**检查清单**：
- [ ] PRD文档已读取
- [ ] 游戏愿景已提取
- [ ] 目标用户已提取
- [ ] 核心玩法已提取
- [ ] 主要系统已提取
- [ ] 商业化模式已提取
- [ ] 社交系统已提取

---

### 步骤2：评估好玩度

**目标**：评估游戏的核心玩法循环是否清晰、有吸引力

**操作方法**：
1. 评估玩法独特性
   - 核心玩法是否有创新点
   - 与同类游戏的差异化
   - 是否有独特的"意志投射"或创作巧思
2. 评估内在驱动力
   - 玩家是否因为玩法本身而愿意玩
   - 是否过度依赖数值奖励
   - 是否有足够的内在激励
3. 评估心流体验可能性
   - 难度曲线是否合理
   - 是否有挑战与能力的平衡
   - 是否有沉浸感
4. 评估玩家技巧成长空间
   - 是否有技巧深度
   - 是否有学习曲线
   - 是否有长期成长空间
5. 给出1-10分的好玩度评分
6. 提供详细的评分理由

**检查清单**：
- [ ] 玩法独特性已评估
- [ ] 内在驱动力已评估
- [ ] 心流体验已评估
- [ ] 技巧成长空间已评估
- [ ] 好玩度评分已给出
- [ ] 评分理由已提供

---

### 步骤3：评估商业化潜力

**目标**：评估游戏是否能与目标用户建立价值共鸣，并设计出合理的付费模式

**操作方法**：
1. 评估目标用户画像清晰度
   - 目标用户是否明确
   - 用户需求是否清晰
   - 用户付费能力是否合理
2. 评估付费点与核心玩法结合度
   - 付费点是否自然融入玩法
   - 是否有强制付费
   - 付费是否影响核心体验
3. 评估付费项目价值感
   - 付费是否提供便利
   - 付费是否提供美学价值
   - 付费是否提供实力提升
   - 价值感是否合理
4. 评估付费设计对非付费玩家的影响
   - 非付费玩家体验是否受损
   - 是否有公平性问题
   - 是否有Pay-to-Win问题
5. 给出1-10分的商业化潜力评分
6. 提供详细的评分理由

**检查清单**：
- [ ] 目标用户画像已评估
- [ ] 付费点结合度已评估
- [ ] 付费价值感已评估
- [ ] 非付费玩家影响已评估
- [ ] 商业化评分已给出
- [ ] 评分理由已提供

---

### 步骤4：评估长期运营潜力

**目标**：评估游戏的系统框架是否逻辑自洽，且具备可扩展性和深度

**操作方法**：
1. 评估系统可扩展性
   - 是否能轻松加入新角色
   - 是否能轻松加入新关卡
   - 是否能轻松加入新玩法
2. 评估玩家长线目标和追求
   - 是否有长期目标
   - 是否有持续激励
   - 是否有重玩价值
3. 评估社交系统与社区构建
   - 是否有公会系统
   - 是否有UGC工具
   - 是否有社区互动机制
4. 评估Live Ops接口
   - 是否为线上活动预留设计接口
   - 是否有活动系统
   - 是否有运营工具
5. 给出1-10分的长期运营潜力评分
6. 提供详细的评分理由

**检查清单**：
- [ ] 系统可扩展性已评估
- [ ] 长线目标已评估
- [ ] 社交系统已评估
- [ ] Live Ops接口已评估
- [ ] 长期运营评分已给出
- [ ] 评分理由已提供

---

### 步骤5：识别最大冲突点

**目标**：识别"好玩"、"商业化"、"长期运营"三者之间的最大冲突或短板

**操作方法**：
1. 分析三个维度之间的冲突
   - 商业化是否破坏好玩度
   - 商业化是否影响长期运营
   - 好玩度是否影响商业化
   - 长期运营是否影响好玩度
2. 识别最大冲突点
   - 确定最严重的冲突
   - 分析冲突的根本原因
   - 评估冲突的影响程度
3. 识别核心短板
   - 确定最薄弱的维度
   - 分析短板的根本原因
   - 评估短板的影响程度

**检查清单**：
- [ ] 维度间冲突已分析
- [ ] 最大冲突点已识别
- [ ] 冲突根本原因已分析
- [ ] 核心短板已识别
- [ ] 短板根本原因已分析

---

### 步骤6：识别核心优势

**目标**：识别PRD最大的亮点或长板

**操作方法**：
1. 分析三个维度的优势
   - 好玩度的优势
   - 商业化的优势
   - 长期运营的优势
2. 识别核心优势
   - 确定最突出的优势
   - 分析优势的独特性
   - 评估优势的可持续性
3. 提出放大核心优势的建议
   - 如何进一步强化优势
   - 如何利用优势带动其他维度
   - 如何将优势转化为竞争力

**检查清单**：
- [ ] 各维度优势已分析
- [ ] 核心优势已识别
- [ ] 优势独特性已分析
- [ ] 优势可持续性已评估
- [ ] 放大建议已提出

---

### 步骤7：提供优化行动计划

**目标**：提供3-5条最高优先级的、具体的PRD修改建议

**操作方法**：
1. 基于冲突点和短板，提出优化建议
2. 每条建议包含：
   - **要修改什么**：清晰描述要修改的PRD部分
   - **为什么修改**：解释这个修改如何解决冲突点或补足短板
   - **预期效果**：说明修改后，预计会对哪个维度的得分产生正面影响
3. 按优先级排序建议
4. 以"制作人备忘录"的形式输出

**检查清单**：
- [ ] 优化建议已提出
- [ ] 建议包含修改内容
- [ ] 建议包含修改原因
- [ ] 建议包含预期效果
- [ ] 建议已按优先级排序
- [ ] 建议已以备忘录形式输出

---

3. 整理核心优势和放大建议
4. 整理优化行动计划

## 操作注意事项

### 评估原则

1. **客观公正**：基于PRD内容进行客观评估，避免主观偏见
2. **全面性**：从三个维度全面评估，避免遗漏重要方面
3. **具体性**：评估理由要具体，避免泛泛而谈
4. **建设性**：提供具体可操作的优化建议

### 评分标准

1. **好玩度评分标准**：
   - 9-10分：核心玩法独特，有强烈内在驱动力，心流体验优秀
   - 7-8分：核心玩法有特色，有一定内在驱动力，心流体验良好
   - 5-6分：核心玩法中规中矩，内在驱动力一般，心流体验一般
   - 3-4分：核心玩法缺乏特色，内在驱动力弱，心流体验差
   - 1-2分：核心玩法有严重问题，无内在驱动力，无心流体验

2. **商业化潜力评分标准**：
   - 9-10分：目标用户清晰，付费点自然，价值感强，非付费体验良好
   - 7-8分：目标用户较清晰，付费点较自然，价值感较强，非付费体验较好
   - 5-6分：目标用户一般，付费点一般，价值感一般，非付费体验一般
   - 3-4分：目标用户不清晰，付费点不自然，价值感弱，非付费体验差
   - 1-2分：目标用户混乱，付费点强制，无价值感，严重损害非付费体验

3. **长期运营潜力评分标准**：
   - 9-10分：系统可扩展性强，长线目标丰富，社交系统完善，Live Ops接口完备
   - 7-8分：系统可扩展性较强，长线目标较丰富，社交系统较完善，Live Ops接口较完备
   - 5-6分：系统可扩展性一般，长线目标一般，社交系统一般，Live Ops接口一般
   - 3-4分：系统可扩展性弱，长线目标少，社交系统差，Live Ops接口差
   - 1-2分：系统不可扩展，无长线目标，无社交系统，无Live Ops接口

### 迭代优化

1. **迭代评估**：根据优化建议修改PRD后，再次进行评估
2. **对比分析**：对比前后评分变化，评估优化效果
3. **持续改进**：根据评估结果持续优化PRD

## 常见错误

### 错误1：评估过于主观

**问题**：评估基于个人喜好而非客观标准

**修正**：严格按照评估框架和评分标准进行评估，提供具体的评估理由

### 错误2：评估理由不具体

**问题**：评估理由泛泛而谈，缺乏具体分析

**修正**：提供具体的评估理由，引用PRD中的具体内容进行分析

### 错误3：优化建议不可操作

**问题**：优化建议过于抽象，缺乏可操作性

**修正**：提供具体可操作的优化建议，明确修改内容、修改原因和预期效果

### 错误4：忽视维度间冲突

**问题**：只评估单个维度，忽视维度间的冲突

**修正**：深入分析三个维度之间的冲突，识别最大冲突点

### 错误5：忽视核心优势

**问题**：只关注短板，忽视核心优势

**修正**：识别核心优势，提出放大优势的建议

### 错误6：建议优先级不明确

**问题**：优化建议没有明确的优先级

**修正**：按优先级排序优化建议，优先解决最严重的冲突和短板
