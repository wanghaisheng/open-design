# Launch Checklist

完整启动准备度验证，涵盖每个部门：代码、内容、商店、营销、社区、基础设施、法律和通过/不通过签署。

## 适用场景

- 需要验证代码准备度
- 需要验证内容准备度
- 需要验证质量保证
- 需要验证商店和分发
- 需要验证启动准备度

## 操作步骤

### 步骤1：读取参数

**目标**：读取启动参数

**操作方法**：
1. 读取启动日期或dry-run模式
2. Dry-run模式生成检查清单而不创建签署条目
3. 确定检查清单范围

**检查清单**：
- [ ] 启动参数已读取
- [ ] 模式已确定
- [ ] 检查清单范围已确定

### 步骤2：收集项目上下文

**目标**：收集项目上下文

**操作方法**：
1. 读取 CLAUDE.md（项目配置）
2. 读取最新里程碑（里程碑定义）
3. 读取现有发布检查清单（发布状态）
4. 读取内容日历（如存在）

**检查清单**：
- [ ] CLAUDE.md 已读取
- [ ] 最新里程碑已读取
- [ ] 发布检查清单已读取
- [ ] 内容日历已读取

### 步骤3：扫描代码库健康

**目标**：扫描代码库健康

**操作方法**：
1. 计数TODO、FIXME、HACK注释
2. 检查生产代码中的调试输出
3. 检查占位符资产
4. 检查硬编码测试/开发值

**检查清单**：
- [ ] TODO/FIXME/HACK已计数
- [ ] 调试输出已检查
- [ ] 占位符资产已检查
- [ ] 硬编码值已检查

### 步骤4：生成启动检查清单

**目标**：生成启动检查清单

**操作方法**：
1. 代码准备度（构建健康、代码质量、安全性）
2. 内容准备度（资产、文本和本地化、游戏内容）
3. 质量保证（测试、平台认证、性能）
4. 商店和分发（商店页面、法律）
5. 基础设施（服务器、分析和监控）
6. 社区和营销（社区准备度、营销）
7. 运营（团队准备度、第一天计划）

**检查清单**：
- [ ] 代码准备度已检查
- [ ] 内容准备度已检查
- [ ] 质量保证已检查
- [ ] 商店分发已检查
- [ ] 基础设施已检查
- [ ] 社区营销已检查
- [ ] 运营已检查

### 步骤5：保存检查清单

**目标**：保存检查清单

**操作方法**：
1. 保存到 production/releases/
2. 创建目录（如需要）
3. 使用标准命名格式

**检查清单**：
- [ ] 检查清单已保存
- [ ] 目录已创建（如需要）
- [ ] 命名格式正确

### 步骤6：输出摘要

**目标**：输出检查清单摘要

**操作方法**：
1. 输出总项目（总检查项数量）
2. 输出阻塞项目计数（阻塞项数量）
3. 输出条件项目计数（条件项数量）
4. 输出不完整部门（未完成部门）
5. 输出文件路径（检查清单路径）

**检查清单**：
- [ ] 总项目已输出
- [ ] 阻塞项目已输出
- [ ] 条件项目已输出
- [ ] 不完整部门已输出
- [ ] 文件路径已输出

## 3. 质量保证

### 测试
- [ ] 完整回归测试套件通过

### 性能
- [ ] 最低规格硬件上达到目标FPS

- [ ] 所有平台内存使用在预算内

- [ ] 关键游戏时刻无帧卡顿

## 常见错误

### 错误1：不扫描代码库健康

**问题**：不计数TODO/FIXME、不检查调试输出

**修正**：必须计数TODO/FIXME/HACK，检查调试输出，检查占位符资产

### 错误2：不收集项目上下文

**问题**：不读取CLAUDE.md、不读取最新里程碑

**修正**：必须读取CLAUDE.md，最新里程碑，发布检查清单

### 错误3：不生成完整检查清单

**问题**：不检查代码准备度、不检查内容准备度

**修正**：必须检查代码准备度，内容准备度，质量保证，商店分发

### 错误4：不检查基础设施

**问题**：不检查服务器配置、不检查分析监控

**修正**：必须检查服务器配置，分析监控，社区营销

### 错误5：不检查运营准备

**问题**：不检查团队准备度、不检查第一天计划

**修正**：必须检查团队准备度，第一天计划，待命团队
