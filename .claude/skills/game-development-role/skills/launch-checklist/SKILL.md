# Launch Checklist

完整启动准备度验证，涵盖每个部门：代码、内容、商店、营销、社区、基础设施、法律和通过/不通过签署。

## 启动检查清单要素

### 1. 代码准备度 (Code Readiness)
- 构建健康
- 代码质量
- 安全性

### 2. 内容准备度 (Content Readiness)
- 资产
- 文本和本地化
- 游戏内容

### 3. 质量保证 (Quality Assurance)
- 测试
- 平台认证
- 性能

### 4. 商店和分发 (Store and Distribution)
- 商店页面
- 法律

### 5. 基础设施 (Infrastructure)
- 服务器（如多人游戏/在线）
- 分析和监控

### 6. 社区和营销 (Community and Marketing)
- 社区准备度
- 营销

### 7. 运营 (Operations)
- 团队准备度
- 第一天计划

## 启动检查清单原则

### 1. 全面性
- 覆盖所有部门
- 不遗漏关键检查项
- 考虑所有方面

### 2. 严格性
- 零容忍关键问题
- 严格质量标准
- 明确通过/不通过决策

### 3. 可操作性
- 每个检查项可验证
- 明确验收标准
- 明确负责人

### 4. 协调性
- 跨部门协调
- 统一标准
- 统一决策

## 启动检查清单流程

### 1. 读取参数
- 启动日期或dry-run模式
- Dry-run模式生成检查清单而不创建签署条目

### 2. 收集项目上下文
- 读取 CLAUDE.md
- 读取最新里程碑
- 读取现有发布检查清单
- 读取内容日历（如存在）

### 3. 扫描代码库健康
- 计数TODO、FIXME、HACK注释
- 检查生产代码中的调试输出
- 检查占位符资产
- 检查硬编码测试/开发值

### 4. 生成启动检查清单
- 代码准备度
- 内容准备度
- 质量保证
- 商店和分发
- 基础设施
- 社区和营销
- 运营

### 5. 保存检查清单
- 保存到 production/releases/
- 创建目录（如需要）

### 6. 输出摘要
- 总项目
- 阻塞项目计数
- 条件项目计数
- 不完整部门
- 文件路径

## 输出格式

### 启动检查清单
```markdown
# 启动检查清单：[游戏标题]
目标启动：[日期或DRY RUN]
生成日期：[日期]

---

## 1. 代码准备度

### 构建健康
- [ ] 所有目标平台干净构建
- [ ] 零编译器警告
- [ ] 所有单元测试通过
- [ ] 所有集成测试通过
- [ ] 性能基准在目标内
- [ ] 无内存泄漏（通过扩展浸泡测试验证）
- [ ] 构建大小在平台限制内
- [ ] 构建版本正确设置并在源代码控制中标记

### 代码质量
- [ ] TODO计数：[N]（启动需要零，或文档化例外）
- [ ] FIXME计数：[N]（需要零）
- [ ] HACK计数：[N]（每个必须有文档化理由）
- [ ] 生产代码中无调试输出
- [ ] 无硬编码开发/测试值
- [ ] 所有功能标志设置为生产值
- [ ] 错误处理覆盖所有关键路径
- [ ] 崩溃报告集成并验证

### 安全性
- [ ] 源代码中无暴露的API密钥或凭据
- [ ] 保存数据加密
- [ ] 网络通信安全（TLS/DTLS）
- [ ] 反作弊措施激活（如多人游戏）
- [ ] 所有服务器端点输入验证（如多人游戏）
- [ ] 隐私政策合规验证

---

## 2. 内容准备度

### 资产
- [ ] 所有占位符艺术替换为最终资产
- [ ] 所有占位符音频替换为最终音频
- [ ] 音频混音最终确定并经音频总监批准
- [ ] 所有VFX打磨并性能验证
- [ ] 无缺失或损坏的资产引用
- [ ] 资产命名约定强制执行

### 文本和本地化
- [ ] 所有面向玩家的文本校对
- [ ] 无硬编码字符串（所有外部化用于本地化）
- [ ] 所有支持的语言翻译并验证
- [ ] 文本在所有语言中适合UI（文本适合性完成）
- [ ] 所有支持语言的字体覆盖验证
- [ ] 演职员表完整、准确且最新

### 游戏内容
- [ ] 所有关卡/地图从头到尾可游玩
- [ ] 教程流程完成并与新玩家测试
- [ ] 所有成就/奖杯实现并测试
- [ ] 保存/加载对所有游戏状态正确工作
- [ ] 难度设置平衡并测试
- [ ] 终局/演职员表序列完成

---

## 3. 质量保证

### 测试
- [ ] 完整回归测试套件通过
- [ ] 零S1（关键）错误开放
- [ ] 零S2（主要）错误开放（或文档化例外）
- [ ] 浸泡测试通过（8+小时连续游玩）
- [ ] 多人游戏压力测试通过（如适用）
- [ ] 每个平台测试所有关键用户路径
- [ ] 边缘情况测试（全存储、无网络、挂起/恢复）

### 平台认证
- [ ] PC：Steam/Epic/GOG SDK要求满足
- [ ] Console：TRC/TCR/Lotcheck提交准备
- [ ] Mobile：App Store/Play Store指南合规
- [ ] 可访问性：满足最低标准（重新映射、文本缩放、色盲）
- [ ] 获得年龄评级（ESRB、PEGI、区域）

### 性能
- [ ] 最低规格硬件上达到目标FPS
- [ ] 所有平台加载时间在预算内
- [ ] 所有平台内存使用在预算内
- [ ] 网络带宽在目标内（如多人游戏）
- [ ] 关键游戏时刻无帧卡顿

---

## 4. 商店和分发

### 商店页面
- [ ] 商店页面副本最终确定并校对
- [ ] 截图当前且每平台分辨率
- [ ] 预告片当前并批准
- [ ] 关键艺术和胶囊图像最终确定
- [ ] 系统要求准确（PC）
- [ ] 所有区域定价配置
- [ ] 预购/愿望清单活动激活（如适用）

### 法律
- [ ] EULA最终确定并经法律批准
- [ ] 隐私政策发布并链接
- [ ] 第三方许可归属完成
- [ ] 音乐/音频许可验证
- [ ] 商标/IP清理确认
- [ ] GDPR/CCPA合规验证（数据收集、同意、删除）

---

## 5. 基础设施

### 服务器（如多人游戏/在线）
- [ ] 生产服务器配置并负载测试
- [ ] 自动缩放配置并测试
- [ ] 数据库备份配置
- [ ] CDN配置用于内容交付
- [ ] DDoS保护激活
- [ ] 监控和警报配置

### 分析和监控
- [ ] 分析管道验证并接收数据
- [ ] 崩溃报告激活且仪表板可访问
- [ ] 服务器监控仪表板活跃
- [ ] 跟踪关键指标：DAU、会话长度、留存、崩溃
- [ ] 为关键阈值配置警报

---

## 6. 社区和营销

### 社区准备度
- [ ] 社区指南发布
- [ ] 审核团队简报且工具就绪
- [ ] Discord/论坛/社交频道设置
- [ ] FAQ和已知问题页面准备
- [ ] 支持电子邮件/工单系统激活

### 营销
- [ ] 启动预告片发布
- [ ] 媒体/影响者审查密钥分发
- [ ] 社交媒体启动帖子计划
- [ ] 启动日博客文章/开发者更新起草
- [ ] 启动版本的补丁说明发布

---

## 7. 运营

### 团队准备度
- [ ] 启动后72小时待命计划设置
- [ ] 事件响应剧本由团队审查
- [ ] 回滚计划文档化并测试
- [ ] 热修复管道测试（可在4小时内紧急修复）
- [ ] 启动问题沟通计划（谁发布、在哪里、多快）

### 第一天计划
- [ ] 第一天补丁准备（如需要）
- [ ] 服务器解锁/上线流程文档化
- [ ] 启动监控仪表板由所有负责人书签
- [ ] 启动日作战室/频道建立

---

## 通过/不通过决策

**整体状态**：[准备就绪 / 未准备就绪 / 条件]

### 阻塞项目
[列出启动前必须解决的任何项目]

### 条件项目
[列出具文档化变通方法或接受风险的项目]

### 所需签署
- [ ] 创意总监——内容和体验质量
- [ ] 技术总监——技术健康和稳定性
- [ ] QA负责人——质量和测试覆盖
- [ ] 制作人——时间表和整体准备度
- [ ] 发布经理——构建和部署准备度
```

## 规则

- 仅明确调用：此技能应仅在用户明确请求 /launch-checklist 时运行
- 不基于上下文匹配自动调用
- Dry-run模式生成检查清单而不创建签署条目
- 保存到 production/releases/launch-checklist-[date].md
