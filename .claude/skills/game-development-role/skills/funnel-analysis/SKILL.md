# Funnel Analysis

漏斗分析技能，分析用户转化漏斗以优化转化率。

## 适用场景

- 分析用户转化
- 优化用户旅程
- 识别流失点
- 提高转化率
- 优化关键流程

## 操作步骤

### 步骤1：定义漏斗

**目标**：定义分析漏斗

**操作方法**：
1. 确定分析目标
2. 定义漏斗步骤
3. 设置转化目标
4. 确定用户群体
5. 设置时间范围

**检查清单**：
- [ ] 分析目标已确定
- [ ] 漏斗步骤已定义
- [ ] 转化目标已设置
- [ ] 用户群体已确定
- [ ] 时间范围已设置

### 步骤2：收集漏斗数据

**目标**：收集漏斗各步骤数据

**操作方法**：
1. 配置事件追踪
2. 收集各步骤数据
3. 验证数据完整性
4. 计算转化率
5. 识别异常数据

**检查清单**：
- [ ] 事件追踪已配置
- [ ] 步骤数据已收集
- [ ] 数据完整性已验证
- [ ] 转化率已计算
- [ ] 异常数据已识别

### 步骤3：分析漏斗表现

**目标**：分析漏斗各步骤表现

**操作方法**：
1. 计算各步骤转化率
2. 识别流失率最高的步骤
3. 分析流失原因
4. 比较不同用户群体
5. 分析时间趋势

**检查清单**：
- [ ] 转化率已计算
- [ ] 高流失步骤已识别
- [ ] 流失原因已分析
- [ ] 用户群体已比较
- [ ] 时间趋势已分析

### 步骤4：分段分析

**目标**：对不同用户群体进行分段分析

**操作方法**：
1. 按用户属性分段
2. 按行为分段
3. 按来源分段
4. 比较各段表现
5. 识别高价值群体

**检查清单**：
- [ ] 用户属性分段已完成
- [ ] 行为分段已完成
- [ ] 来源分段已完成
- [ ] 各段表现已比较
- [ ] 高价值群体已识别

### 步骤5：提出优化建议

**目标**：基于漏斗分析提出优化建议

**操作方法**：
1. 识别优化机会
2. 制定优化方案
3. 优先级排序
4. 评估优化影响
5. 制定实施计划

**检查清单**：
- [ ] 优化机会已识别
- [ ] 优化方案已制定
- [ ] 优先级已排序
- [ ] 优化影响已评估
- [ ] 实施计划已制定

## 输出格式

### 漏斗分析报告
```markdown
## 漏斗分析报告

### 漏斗定义
1. 访问游戏
2. 完成教程
3. 创建角色
4. 完成首局游戏
5. 第2天登录

### 转化率
- 访问游戏: 100%
- 完成教程: 80%
- 创建角色: 70%
- 完成首局: 60%
- 第2天登录: 40%

### 关键发现
1. 教程完成流失率最高(20%)
2. 新用户留存率低
3. 移动端转化率低于PC端

### 优化建议
1. 简化教程流程
2. 优化新手引导
3. 改进移动端体验
```

## 操作注意事项

### 漏斗设计

1. **步骤清晰**：漏斗步骤定义清晰
2. **目标明确**：每个步骤有明确目标
3. **可测量**：每个步骤可测量
4. **合理长度**：漏斗长度合理

### 数据分析

1. **绝对值**：关注绝对值和相对值
2. **时间维度**：考虑时间维度变化
3. **用户分段**：进行用户分段分析
4. **归因分析**：进行归因分析

### 优化策略

1. **优先级**：按影响程度排序优化
2. **小步迭代**：小步迭代优化
3. **A/B测试**：通过A/B测试验证
4. **持续监控**：持续监控优化效果

## 常见错误

### 错误1：漏斗步骤过多

**问题**：漏斗步骤过多，难以分析

**修正**：简化漏斗，聚焦关键步骤

### 错误2：忽视绝对值

**问题**：只关注转化率，忽视绝对值

**修正**：同时关注转化率和绝对值

### 错误3：不分段分析

**问题**：不分段分析，忽略群体差异

**修正**：进行用户分段分析

### 错误4：归因错误

**问题**：错误归因流失原因

**修正**：通过用户研究和数据验证归因

### 错误5：不跟踪优化效果

**问题**：优化后不跟踪效果

**修正**：持续跟踪优化效果，验证改进
