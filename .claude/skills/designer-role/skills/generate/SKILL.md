# Generate Inclusive Persona Set

Create a complete persona set that represents the full range of human
diversity — including disability, situational impairment, age, language,
and technical confidence.

## Process

### Step 1: Understand the Product Context
Before creating personas, establish:
- What does the product do?
- Who uses it today?
- What environments is it used in?
- What are the core tasks?

### Step 2: Map the Ability Spectrum
Using **ability-spectrum-mapping**, identify which ability dimensions
are most relevant to this product's core tasks (vision, hearing,
motor, cognitive, speech).

### Step 3: Map Situational Impairments
Using **situational-impairment-mapping**, identify the common
situational impairments for this product's typical use contexts.

### Step 4: Create the Persona Set
Using **disability-inclusive-personas**, create 4–6 personas that
together cover:
- At least one permanent disability
- At least one temporary impairment
- At least two situational impairment contexts
- Range of age, language, and technical confidence
- Range of assistive technology use

Each persona includes: bio, goals, frustrations, context of use,
devices and assistive technology, and a realistic scenario.

### Step 5: Write AT Scenarios
Using **assistive-technology-scenarios**, write a detailed usage
scenario for each persona that uses assistive technology, showing
step-by-step interaction with the product.

## Output

Deliver a persona set document containing:
1. **Overview** — summary of the set's diversity coverage
2. **Individual personas** — full profile for each
3. **AT scenarios** — detailed usage scenarios for relevant personas
4. **Coverage matrix** — table showing which ability dimensions,
   impairment types, and contexts are represented
5. **Gaps** — any known gaps in the set with recommendations
