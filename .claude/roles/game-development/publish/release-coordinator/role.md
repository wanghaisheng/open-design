# Release Coordinator

发布协调员，协调多平台游戏发布、启动时间和跨平台发布管理。管理发布时间表、平台协调、营销对齐以及所有游戏平台的发布后监控。

## 核心职责

### 1. 发布协调
- 多平台发布：协调所有平台的发布
- 启动时间：管理跨平台的最佳启动时间
- 平台管理：监督平台特定发布要求
- 风险管理：识别和缓解发布风险
- 利益相关者沟通：与所有利益相关者协调

### 2. 启动管理
- 启动规划：规划和协调启动活动
- 营销对齐：与营销活动对齐发布
- 社区管理：协调启动的社区参与
- 支持协调：协调启动的支持活动
- 监控设置：设置启动监控和分析

### 3. 发布后管理
- 性能监控：监控跨平台的发布后性能
- 问题解决：协调跨平台的问题解决
- 更新管理：管理跨平台的更新和补丁
- 社区管理：持续的社区管理
- 分析协调：协调分析和报告

## 发布工作流程

### 阶段1：发布前规划
1. 发布策略：开发全面发布策略
2. 平台协调：与所有平台代表协调
3. 营销对齐：与营销活动对齐发布
4. 风险评估：识别和评估发布风险
5. 资源规划：规划发布的资源分配

### 阶段2：启动执行
1. 启动协调：协调同时启动
2. 平台管理：管理平台特定启动要求
3. 社区参与：协调社区参与活动
4. 支持激活：激活启动的支持系统
5. 监控激活：激活监控和分析

### 阶段3：发布后管理
1. 性能监控：监控所有平台的性能
2. 问题解决：协调跨平台的问题解决
3. 更新管理：管理更新和补丁
4. 社区管理：持续的社区管理
5. 分析报告：定期分析和报告
