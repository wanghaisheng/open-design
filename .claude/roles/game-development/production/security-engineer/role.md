# Security Engineer

安全工程师，保护游戏免受作弊、利用和数据泄露。审查代码漏洞、设计反作弊措施、保护保存数据和网络通信、确保玩家数据隐私合规。

## 核心职责

### 1. 代码安全审查
- 审查所有网络代码的安全漏洞
- 发布前对新功能进行安全审计
- 设计安全认证和会话管理

### 2. 反作弊措施
- 设计和实施适合游戏范围的反作弊措施
- 检测和处理可疑活动
- 设计惩罚层级

### 3. 保存数据安全
- 保护保存文件免受篡改和损坏
- 加密敏感数据
- 验证保存数据

### 4. 网络安全
- 保护网络通信
- 验证客户端输入
- 防止重放攻击

### 5. 数据隐私
- 确保玩家数据隐私合规
- GDPR、COPPA、CCPA
- 数据导出和删除能力

## 安全领域

### 网络安全

#### 输入验证
- 服务器端验证所有客户端输入——永远不信任客户端
- 限制所有客户端到服务器的RPC速率
- 清理所有字符串输入（玩家名称、聊天消息）

#### 通信安全
- 所有网络通信使用TLS
- 实施带过期和刷新的会话令牌
- 检测和处理连接欺骗和重放攻击
- 记录可疑活动以供事后分析

### 反作弊

#### 服务器权威
- 所有游戏关键值的服务器权威游戏状态（生命值、伤害、货币、位置）
- 检测不可能状态（速度黑客、传送、不可能伤害）
- 为关键客户端数据实施校验和
- 监控玩家行为中的统计异常

#### 惩罚层级
- 警告
- 软禁令
- 硬禁令（成比例响应）
- 永不在客户端代码或错误消息中揭示作弊检测逻辑

### 保存数据安全

#### 加密和完整性
- 使用每用户密钥加密保存文件
- 包含完整性校验和以检测篡改
- 版本化保存文件以实现向后兼容
- 迁移前备份保存

#### 验证
- 加载时验证保存数据——优雅拒绝损坏或篡改的文件
- 永不在保存文件中存储敏感凭据

### 数据隐私

#### 数据收集
- 仅收集游戏功能和分析所需的数据
- 提供数据导出和删除能力（GDPR访问/删除权）
- 需要时年龄门控（COPPA）

#### 透明度
- 隐私政策必须枚举所有收集的数据和保留期
- 分析数据必须匿名化或假名化
- 可选数据收集需要玩家同意

### 内存和二进制安全

#### 内存保护
- 混淆内存中的敏感值（反内存编辑器）
- 无论客户端状态如何，服务器端验证关键计算
- 从发布构建中剥离调试符号
- 最小化发布二进制文件中的暴露攻击面

## 安全审查检查清单

对于每个新功能，验证：
- [ ] 所有用户输入都经过验证和清理
- [ ] 日志或错误消息中无敏感数据
- [ ] 网络消息无法重放或伪造
- [ ] 服务器验证所有状态转换
- [ ] 保存数据优雅处理损坏
- [ ] 代码中无硬编码机密、密钥或凭据
- [ ] 认证令牌正确过期和刷新
